## 它是什么
一个可复用的Next.js项目模板，内置AI代理指令，只需一条命令即可反向工程任何网站，生成干净的TypeScript/React代码。

## 为什么火
它解决了网站重建中最耗时的手工环节——分析布局、提取样式、重写组件。通过多阶段管道和并行构建，将数周的工作压缩到几分钟，特别适合平台迁移、源代码丢失等场景。

## 技术栈
- Next.js 16 (App Router) + React 19 + TypeScript strict
- shadcn/ui (Radix + Tailwind CSS v4)
- Lucide React 图标，克隆时替换为提取的SVG
- 支持 Claude Code、Copilot、Cursor 等 10+ 种 AI 代理

## 核心能力
- **多阶段管道**：截图/设计令牌提取 → 交互扫描 → 字体/颜色/资源下载 → 组件规范编写 → 并行构建 → 组装与QA
- **精确提取**：通过 getComputedStyle() 获取实际 CSS 值，避免猜测
- **并行构建**：在 git worktrees 中派发独立 Builder 智能体，每个负责一个部分
- **多平台支持**：通过 sync 脚本将核心指令同步到各种 AI 代理的配置格式

## 适用场景
- 从 WordPress/Webflow/Squarespace 等平台迁移到现代前端
- 恢复丢失源代码的网站
- 学习生产级网站的布局与动画实现

## 同类对比
与传统网页抓取工具（如 HTTrack）相比，此模板不仅抓取静态资源，更生成可维护的组件代码；与纯手工重写相比，自动化程度极高。本项目并非直接竞品，而是为 AI 代理提供指令与脚手架，属于新兴的 AI 辅助开发范式。
