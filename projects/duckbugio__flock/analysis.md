## 它是什么
Flock 是一个在自有服务器上运行的多代理 AI 开发团队机器人。通过 Telegram 或 VK 聊天接收自然语言需求，自动进行规划、编码、测试、审查并打开 PR。每个聊天拥有独立的沙盒工作区，支持跨多个微服务仓库的协同。

## 为什么火
它让开发者可以在聊天中调用一个完整的 AI 开发团队，而非单次代码补全。利用 Claude 订阅绕开 token 计费，同时提供从需求到 PR 的闭环流水线，对已订阅 Claude 并希望自托管 AI 编码助手的团队很有吸引力。

## 技术栈
- 核心与适配器使用 Go 语言编写
- 代理基于 Claude Code 子代理（planner / coder / tester / reviewer / arbiter）
- 通过预构建 Docker 镜像部署，非 root 用户运行
- 支持 Git 托管平台（Gitea/GitHub/GitLab），通过轮询获取 PR 评论
- 可选 dind sidecar 用于容器化测试，语音转录支持 Mistral/OpenAI

## 核心能力
- **多代理流水线**：规划→编码→测试→审查→仲裁，仲裁者打破循环防止无限挂起
- **聊天驱动物理隔离**：每个聊天独立 `/workspace/chat_<id>`，支持并行且互不干扰
- **无入站 webhook 的 PR 审查反馈**：通过轮询 Git 主机，将审查评论路由回原聊天
- **微服务跨仓库协同**：一个功能可跨多个服务，自动命名分支 `duck/<chatid>/<slug>` 并创建跨仓库 PR
- **可扩展适配器**：Telegram 和 VK 已可用，新平台仅需薄适配层
- **订阅模式集成**：支持 Claude Pro/Max token 或 API key，不按 token 计费

## 适用场景
- 个人或小团队想用聊天直接驱动代码变更和 PR 自动化
- 已订阅 Claude，希望将 AI 编码能力融入自托管 CI 流程
- 对隐私敏感、不能将代码送第三方 API 的项目（一切在自有服务器完成）
- 需要多服务协同的微服务项目，自动管理跨仓库分支和 PR

## 同类对比
相比于 Copilot Chat、Cursor 等单次问答或 IDE 嵌入方案，Flock 提供了自托管、多代理协作的完整开发流程。与 Devin、Sweep 等早期 AI 工程师工具相比，它的自托管和订阅计费模式在成本控制和数据隐私上更有优势，但需要使用者具备容器化部署和 Git 运维能力。
