## 它是什么
Kilo Code 是一个开源的 AI 编码代理平台，提供 VS Code 扩展、JetBrains 插件和 CLI 工具。它能从自然语言生成代码、提供行内补全、自我审查、控制终端与浏览器，并通过 MCP 市场扩展能力。

## 为什么火
开源许可（MIT）加零加价定价策略，开发者只需支付所用模型的供应商费用，没有中间商差价；社区星数达 23k+，活跃度高；支持 500+ 模型且任务中可切换，契合追求灵活性和成本控制的工程团队需求。

## 技术栈
TypeScript 编写，核心三端统一：VS Code 扩展、JetBrains 插件、Node.js CLI。配备云端代理和代码审查服务，但自托管架构细节未在 README 中展开。

## 核心能力
- **多代理模式**：Code（默认编码）、Plan（架构规划）、Ask（问答）、Debug（排错）、Review（审查），并可自定义。
- **内联自动补全**：幽灵文本建议，Tab 接受。
- **自检查**：代理自我审查并修正产出。
- **终端与浏览器控制**：可在 IDE 内运行命令和自动化网页操作。
- **MCP 市场**：扩展代理能力的服务器市场。
- **500+ 模型与中任务切换**：自由匹配延迟、成本和推理能力。
- **自主模式**：`--auto` 参数用于 CI/CD，无需人工确认，完全自动化完成指令。

## 适用场景
- 日常编码辅助（跨 IDE 的代码生成与补全）
- CI/CD 流水线中的自动化测试和修复（`kilo run --auto "run tests and fix any failures"`）
- 代码审查自动化（线上 PR 审查服务）
- 架构设计与调试（使用专用 Plan/Debug 代理）
- 终端重度用户通过 CLI 直接调用

## 同类对比
对比 GitHub Copilot、Cursor、Amazon CodeWhisperer，Kilo Code 的优势在于：完全开源、定价透明（无加价）、支持 500+ 模型且一键导入个人密钥（BYOK）、自主模式原生支持 CI/CD、跨三大平台。势势在于需自行管理模型供应商账户（若不用 Kilo Gateway），且生态成熟度可能不及微软系产品。
