## 它是什么
Nubase 是一个开源、AI 原生的后端平台，专为 AI 编码代理和现代产品团队设计。它把数据库、认证、存储、静态资产、边缘函数、AI 网关、记忆和定时任务八个模块打包成一个可自托管的服务。与 Supabase 类似，但强调多项目隔离、内置记忆（Mem0 风格）和面向 AI 代理的 MCP 工具链。

## 为什么火
- **AI 编码的“最后一公里”**：传统 AI 生成代码后仍需手动部署、配置后端，Nubase 让代理通过 MCP 工具直接创建表、部署函数、发布前端、设置定时任务，实现“生成即上线”。
- **记忆系统开箱即用**：不像大多数后端把向量存储当作外挂，Nubase 将用户记忆、实体提取、混合检索作为核心原语，非常适合构建个性化 Agent 应用。
- **真正的多项目自托管**：Supabase 自托管版通常模拟单项目，Nubase 的一个控制平面可管理多个孤立项目数据库，对 AI 公司或自由职业者自建基础设施更友好。

## 技术栈
- 后端：Java 17 + Spring Boot + Maven
- 数据库：PostgreSQL 15 + pgvector（每个项目独立数据库）
- REST API：PostgREST 兼容层（Java 实现）
- 存储：兼容 S3（支持 Cloudflare R2、MinIO、AWS S3）
- 边缘函数：本地执行器或 Cloudflare Workers for Platforms
- Studio：Next.js + pnpm
- CLI：npm 包 `nubase_cli`，通过 MCP 连接 Claude Code、Codex 等代理
- 部署：Docker 一体化镜像（内置 PostgreSQL + Redis + 后端 + Studio）

## 核心能力
- **数据库与认证**：每个项目独立 PostgreSQL，Supabase 风格的用户管理、JWT、RLS、角色和 OAuth/SAML 支持。
- **记忆（Memory）**：支持 LLM 事实提取（新增/更新/删除）、混合检索（pgvector + 全文搜索 + 实体加权）、追加历史记录。
- **静态资源与函数**：能直接通过 MCP 部署前端到公共 CDN（`/assets/v1`），部署边缘函数（`/functions/v1`）并设置定时 cron 作业。
- **AI 网关**：兼容 OpenAI/Anthropic 的端点，提供项目级 API 密钥和用量追踪。
- **代理集成**：一行命令 `npx nubase_cli install-skills` 为 Claude Code 或 Codex 安装技能和 MCP 配置，授权后代理即可全面操控平台。

## 适用场景
- **AI 生成应用的快速交付**：从自然语言描述直接生成全栈应用并上线，适合 PoC 或内部工具。
- **个性化 AI Agent 开发**：需要长期记忆、用户画像的聊天机器人、教练、助手等应用。
- **多租户 SaaS 或项目工作室**：需要在单一基础设施上为多个客户或项目提供完全隔离的后端服务。
- **自建 Supabase 替代品**：希望保留开源、自托管，但需要多项目管理和 AI 特色功能（记忆、MCP）的团队。

## 同类对比
| 特性 | Supabase 自托管 | Nubase |
|------|----------------|--------|
| 多项目仪表板 | 无（仅模拟单项目） | 有 |
| 项目数据库隔离 | 仅一个本地项目 | 每个项目独立 PG 库 |
| 内置记忆 | 无（需自己搭建） | 内置混合检索记忆原语 |
| 面向 AI 代理的 MCP | 无 | 完整 MCP 工具（建表、部署函数、发布资产等） |
| 部署生成应用 | 需要额外配置托管、函数、定时任务 | 内置资产 CDN + 函数 + cron |
| 边缘函数 | 支持 | 支持（本地或 Cloudflare WfP） |
| 实时订阅 | 可用 | 暂未提供 |
