## 它是什么
codebase-memory-mcp 是一个用纯C编写的MCP服务器，能将任意代码库索引为由函数、类、调用链、HTTP路由等构成的知识图谱。它面向AI编码代理（如Claude Code、Gemini CLI、Cursor等），旨在用最少的token和工具调用完成代码理解任务。

## 为什么火
- **Star数快速增长**：7763颗星，社区认可度高。
- **击中痛点**：传统agent通过grep/read文件探索代码，消耗大量上下文窗口；该项目将代码结构预计算为图谱，单次查询即可替代数十次文件读取，实测token减少99%。
- **零摩擦集成**：单二进制，无Python、Node、Docker等运行时依赖，一条命令即可安装并自动配置11种主流编码代理。

## 技术栈
- **语言**：纯C，单二进制，最小化内存占用。
- **解析引擎**：内置158个tree-sitter语法，离线工作。
- **存储**：SQLite（内存/磁盘），LZ4压缩，知识图谱持久化。
- **高级分析**：Hybrid LSP（C实现）进行类型推断，Louvain社区发现，MinHash重复检测，BM25/FTS5全文搜索，向量语义搜索（内嵌Nomic嵌入）。
- **分发**：macOS/Linux/Windows静态二进制，SLSA 3级供应链安全，每个发布包均经70+杀毒引擎扫描。

## 核心能力
- **14个MCP工具**：架构概览、图谱搜索、Cypher查询、死代码检测、跨服务HTTP/gRPC链接、变更影响分析、ADR管理等。
- **多语言支持**：158种语言，9种支持Hybrid LSP语义解析（Python、TS/JS/JSX/TSX、PHP、C#、Go、C/C++、Java、Kotlin、Rust）。
- **IaC索引**：Dockerfile、Kubernets清单、Kustomize也被建模为节点和边。
- **可视化**：内置3D图UI，可在localhost:9749探索图谱。
- **自动同步**：后台监听文件变更并重新索引。

## 适用场景
- AI编码代理查询项目结构、调用关系、架构边界。
- 大型遗留系统快速理解与重构。
- 微服务间API依赖梳理与变更影响分析。
- 多仓库架构可视化与跨服务死代码检测。

## 同类对比
- **vs. 传统grepping**：图谱查询返回结构化关系，而非扁平文本，消除了agent对结果的手动关联和二次查询。
- **vs. 单一LSP工具**：不仅限于单个文件或符号，构建全局调用图、跨模块依赖、通道连接，且不依赖于外部语言服务器运行。
- **vs. 云托管代码智能服务**：100%本地运行，代码永不离机，零信任风险。
