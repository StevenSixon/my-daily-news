## 它是什么
codebase-memory-mcp 是一个面向 AI 编程代理的高性能代码智能 MCP 服务器。它用 Tree-sitter 解析 158 种语言，构建持久化知识图谱（函数、类、调用链、HTTP 路由等），将整个代码库索引为可秒级查询的 SQLite 数据库。单个 C 语言二进制，无运行时依赖，可一键接入 11 种代理。

## 为什么火
在 GitHub 上已获 13k+ Star。它抓住了 LLM 编码助手反复读取文件、消耗大量 token 和时间的痛点。通过一次索引、毫秒级图查询替代数十次文件抓取，显存/账单节省显著。全本地运行也打消了代码外泄顾虑。

## 技术栈
- 语言：纯 C（生成单静态二进制）
- 解析器：vendored 158 个 tree-sitter 语法
- 存储：SQLite（内存+磁盘），LZ4 压缩
- 查询引擎：自定义 Cypher 风格模式，FTS5+BM25，11 信号语义搜索（捆绑 Nomic 嵌入模型）
- 增量索引：基于 git 的文件监视
- 分发：macOS/Linux/Windows，签名+校验+杀毒扫描

## 核心能力
- **极速索引**：Linux 内核（28M LOC）3 分钟完成
- **结构查询**：14 个 MCP 工具（搜索、追踪、架构、死代码、影响分析、ADR 管理）
- **混合 LSP**：为 Python/TS/PHP/C#/Go/C/C++/Java/Kotlin/Rust 等提供类型解析，提升符号解析精度
- **跨服务链接**：HTTP/gRPC/GraphQL/tRPC 路由匹配，消息通道检测
- **语义搜索**：内置嵌入模型，99% token 缩减（3.4k vs 412k）
- **可视化**：内置 3D 图谱 UI（可选）
- **IaC 索引**：Dockerfile/K8s/Kustomize 作为图节点
- **自动同步**：后台监测文件变化自动重索引

## 适用场景
- 使用 Claude Code、Codex CLI、Gemini CLI、Aider 等 AI 编码代理的开发者，希望给代理全仓结构感知
- 大型单体或多仓库项目，需要快速理解架构、调用链、路由关系
- 代码审计、死代码发现、架构决策记录管理

## 同类对比
与 Sourcegraph（服务化、需部署）相比，它是本地单二进制，零配置；与 ast-grep / Semgrep 相比，它侧重构建可查询的全仓图谱而非一次性规则匹配；与基于文件的 MCP 工具相比，它大幅减少 token 消耗和往返次数。
