## 它是什么

Matt Pocock 从自己日常使用的 `.claude` 目录中提炼出的一组小型、可组合的代理技能（Skills）。这些技能不是“氛围编码”的快捷方式，而是基于数十年工程经验设计的重复性工作流，帮助开发者在与编码代理（如 Claude Code、Codex）协作时保持对齐、质量和设计纪律。项目包含十几个技能，分为工程、生产力、杂项三类，且分为用户主动调用（如 `/grill-me`）和模型自动触发两种模式。

## 为什么火

Star 数高达 139k，因为精准击中了 AI 辅助编码的四大常见失败模式：
1. **代理不理解用户真正想要什么** → 通过深入的“烧烤会话（grilling）”对齐意图；
2. **代理输出过于啰嗦** → 建立共享语言（领域术语表、`CONTEXT.md`），让沟通与代码命名高度一致；
3. **生成的代码不工作** → 引入红-绿-重构的 TDD 循环和系统化调试流程；
4. **代码库迅速变成泥球** → 主动投资于设计，用“改进代码架构”技能定期重构，强调深模块和清晰接缝。

这些理念源自《程序员修炼之道》《领域驱动设计》等经典，以极低成本给予开发者控制权，因此迅速获得大量关注。

## 技术栈

- 技能实质为 Markdown 文本文件，通过 `npx skills@latest` CLI 工具分发和安装
- 依赖 Node.js 运行时（使用 npx，需预先安装）
- 官方支持与 Claude Code 深度集成，但设计上宣称“与任何模型兼容”，推测通过将技能文件加载到代理的上下文中工作
- 仓库本身采用 Shell 脚本辅助安装，语言标签为 Shell

## 核心能力

- **意愿对齐**：`/grill-me` 和 `/grill-with-docs` 系统化追问需求，同时构建领域模型和共享语言
- **共享语言构建**：`/domain-modeling` 主动挑战术语，更新 `CONTEXT.md` 和架构决策记录，减少多义词和冗长表述
- **TDD 循环**：`/tdd` 强制执行红-绿-重构，指导如何写出有效的测试
- **结构化调试**：`/diagnosing-bugs` 提供复现→最小化→假设→插装→修复→回归测试的闭环
- **架构演进**：`/improve-codebase-architecture` 扫描代码库，生成可视化报告并深度讨论改进点
- **任务拆分**：`/to-issues` 将计划按垂直切片分解为独立可领取的议题
- **教学与知识传递**：`/teach` 支持多会话教学，强调组件复用

## 适用场景

适用于使用 AI 编码代理进行严肃软件开发的工程师，尤其关注长期可维护性和团队协作的中大型项目。无论是新项目启动前的需求讨论，还是老代码库的架构救火，都能找到对应技能。也适合希望提升代理输出质量、减少沟通成本的个人开发者。

## 同类对比

- **vs. GSD/BMAD/Spec-Kit**：后者是企图掌控整个开发流程的重量级方法论，常剥夺开发者控制权且难以定位 bug；本技能集则保持“小而美”，可任意组合、修改，与现有流程无缝嫁接。
- **vs. 零散提示词**：提供经过验证的结构化循环，而非一次性提示，强调“每日投资设计”，效果更持久。
- **独特优势**：将经典软件工程原则（领域驱动设计、持续重构、TDD）适配到人机协作新范式，可视为“AI 时代的工程纪律工具箱”。
