## 它是什么

gstack 是 Y Combinator CEO Garry Tan 开源的 Claude Code 工具集，内含 23 个专业“角色”（如 CEO、设计师、工程经理、QA、安全官）的斜杠指令。它把 AI 辅助开发从零散提示词升级为结构化的软件交付流程（思考→规划→构建→审查→测试→交付→回顾），旨在让单人开发者获得一个虚拟工程团队的协作能力。

## 为什么火

- 作者身份与背书：Garry Tan 是 YC 总裁，项目源于其个人的超高产出实践（声称 2026 年生产效率是 2013 年的 810 倍，并提供了可复现的计量方法）。
- 痛点共鸣：许多技术创始人、独立开发者渴望用 AI 加速，但缺少将 AI 从代码补全升级为项目级协作的方法。
- 开源与易接入：MIT 许可，30 秒安装，无需购买任何服务，支持 Claude Code 及 OpenClaw、Codex CLI 等 10 种 AI 代理。

## 技术栈

- 语言：TypeScript
- 运行环境：需 Node.js（Windows）、Bun v1.0+、Git
- 核心平台：Claude Code（主要）、OpenClaw、OpenAI Codex CLI、Cursor 等多种代理支持
- 脚本和技能均以 Markdown 和 TypeScript 配置驱动，无额外后端服务

## 核心能力

1. **角色化技能集**：/office-hours（产品诊疗）、/plan-ceo-review（CEO 视角审查）、/plan-eng-review（架构锁定）、/plan-design-review（设计评分与交互式改进）、/review（代码审查与自动修复）、/qa（真实浏览器测试）、/ship（集成测试与 PR 生成）、/cso（安全审计）等。
2. **流程编排**：技能间能传递产物（设计文档→CEO审阅→工程计划→QA 脚本），形成完整的 sprint 工作流。
3. **多代理适配**：通过 setup 脚本可安装到 Claude Code、OpenClaw、Codex、Cursor、Factory 等不同代理。
4. **团队模式**：可将 gstack 强制或可选地嵌入仓库，使团队成员自动获得相同工具链和自动更新。

## 适用场景

- 单人技术创始人或独立开发者，想用 AI 辅助从产品构思到发布的全流程。
- 小型团队希望统一 AI 辅助编码的规范，提高代码质量和交付节奏。
- 初次使用 Claude Code 的用户，想要一套“开箱即用”的角色化指令代替空白对话。

## 同类对比

- **Claude Code 默认用法**：自主规划能力虽有，但缺乏结构化的角色和步骤。gstack 为其注入工程最佳实践（强制审查、安全审计、测试流程）。
- **其他 AI 编码工具（如 GitHub Copilot、Cursor）**：多为代码补全或单轮对话，缺少贯穿全周期的流程。gstack 可与其互补，甚至在这些工具上安装部分技能。
- **DevOps 平台（如 GitLab CI）**：侧重自动化构建/部署，而 gstack 侧重 AI 驱动的设计与质量决策环节。
