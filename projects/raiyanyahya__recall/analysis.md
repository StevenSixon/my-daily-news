## 它是什么
Recall 是一个 Claude Code 插件，用来给 Claude Code 提供**持久记忆**。每次会话结束时，它会自动捕获对话、文件操作、命令等信息，并用一个**纯本地算法（TF-IDF + TextRank）**生成一份简洁的上下文摘要（`context.md`），下次启动时 Claude 可以直接从这里续上，不再需要重新解释项目。

## 为什么火
- **零成本、零联网**：摘要完全在本地 CPU 上跑，无需 API key、不调用外部模型、不消耗 token，对订阅用户免费，对 API 用户省钱。
- **隐私优先**：所有会话记录和摘要均不离开本机，没有遥测、没有账户、没有第三方服务。
- **无缝集成**：作为 Claude Code 官方插件格式，安装即用，无需 `pip install`，无额外依赖（numpy 仅作为可选加速）。

## 技术栈
- **语言**：Python（标准库为主，无第三方模型依赖）
- **摘要算法**：TF-IDF 向量化 + 余弦相似图 + TextRank（PageRank 变体），可选 numpy 加速
- **集成方式**：Claude Code 插件钩子（`SessionStart`、`Stop`、`SessionEnd`）
- **数据存储**：纯文本 Markdown（`.recall/history.md` 和 `context.md`）
- **安全措施**：内置简易密钥脱敏、Git 命令防注入、输出目录沙箱

## 核心能力
- **自动会话日志**：实时追加每次提问、回复、文件操作到 `history.md`。
- **本地上下文摘要**：`/recall:save` 或配置自动触发，用 TextRank 提取最关键的句子，生成 `context.md`。
- **无缝续接**：下个会话启动时自动加载摘要，Claude 会询问是否从上次上下文继续。
- **与原生记忆互补**：不同于 `CLAUDE.md`（人工规则）或 `--continue`（重放全部对话），Recall 提供一种**自动化、结构化的会话回顾**，token 占用极低（~1-2K）。
- **可配置**：摘要长度、Git 上下文包含、自动保存时机等均可定制。
- **隐私硬承诺**：无网络调用，无密钥，无第三方模型，且会话记录仅限当前项目。

## 适用场景
- 需要频繁在多个项目间切换的 Claude Code 用户。
- 不想每次会话重新解释项目背景的开发者。
- 重视隐私、不愿将工作内容上传到任何服务器的个人或团队。
- 用订阅计划且想省 token 的用户（摘要本地生成，续接上下文也很省 token）。
- 团队共享记忆：可提交 `.recall/` 作为共享上下文（需注意注入风险）。

## 同类对比
- **`CLAUDE.md` / `#` 规则**：需要手动维护，固定指令，不记录会话历史。
- **`--continue` / `--resume`**：能重放会话但 token 消耗高，不够便携，也不提供结构化摘要。
- **其他“记忆”工具**：通常通过 API 调用外部模型对上下文进行总结，有隐私泄露风险和额外费用，Recall 是少数**完全本地运行**的选项，且专为 Claude Code 插件生态设计。
