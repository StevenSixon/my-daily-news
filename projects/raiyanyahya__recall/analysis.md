## 它是什么

Recall 是 Claude Code 的一个离线插件，在项目根目录 `.recall/` 下自动记录会话历史（`history.md`）并用纯本地算法生成紧凑的状态摘要（`context.md`），下次启动时让 Claude 直接恢复上次会话，无需重新解释项目。

## 为什么火

因为 Claude Code 的冷启动问题严重——每次新会话都要花大量 token 解释项目背景，既费钱又费时。Recall 用古典 TextRank 摘要算法在本地完成所有工作，不调任何外部模型，不产生任何费用，且代码和会话记录完全留在用户机器上，击中隐私、成本和效率三个痛点。

## 技术栈

- **语言**: Python (插件逻辑 + CLI 命令)
- **核心算法**: TF-IDF + TextRank 提取式摘要（自包含在 `scripts/summarizer.py`）
- **依赖**: 纯 Python 标准库，numpy 可选（加速用）
- **部署**: 作为 Claude Code 插件，无 `pip install`，无外部服务，无 API 调用
- **安全**: 内置简陋秘密红化，Git 操作时禁用钩子防注入

## 核心能力

- **自动捕获**：SessionStart/Stop 钩子把每轮对话追加到 `history.md`
- **本地摘要**：用户运行 `/recall:save` 或设置自动保存，用 TextRank 生成 `context.md`（约 1–2K token），包含目标、文件修改、命令记录、下一步
- **会话恢复**：新会话开始时展示摘要并询问是否从上次继续
- **灵活配置**：通过 `recall.config.json` 调整摘要句数、保存触发时机、是否包含 git diff 等
- **隐私安全**：完全离线，不联网，不读取密钥，自动红化常见机密格式

## 适用场景

- 频繁使用 Claude Code 进行迭代开发的个人开发者
- 同一项目多人协作，想通过 `.recall/` 共享项目记忆的团队（可提交仓库）
- 重视隐私、不能将会话记录上传到第三方模型端点的用户
- 使用 Claude 订阅制，希望节省 token 用量的用户

## 同类对比

与官方内置记忆对比：
- `CLAUDE.md`/`#` 是手动编写的规则指令，不自动记录会话
- `--continue`/`--resume` 重放完整会话，消耗大量 token，且不生成精炼摘要
- 上下文压缩只在单次会话内有效，不跨会话持久化

与其他第三方记忆工具对比：大多数工具将历史记录发送给外部模型进行总结，Recall 是唯一一个完全在本地运行、不消耗 API 调用、且无网络请求的方案。
