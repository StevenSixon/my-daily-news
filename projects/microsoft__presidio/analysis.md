## 它是什么
Presidio 是微软开源的敏感数据自动识别与匿名化框架，覆盖文本、图像和结构化数据。核心思路是将 PII 检测抽象为“分析器+匿名化器”的管道，支持预置实体类型、正则、校验和及上下文增强，并提供图像编辑模块。

## 为什么火
- **企业级背景**：微软背书，MIT 许可，活跃社区（9.5k+ Star），频繁的安全修复和识别器扩增。
- **可组合性**：识别器（Analyzer）与脱敏策略（Anonymizer）解耦，支持华夫饼式自定义。
- **多模态**：不仅处理文本，还能对图片（包括 DICOM 医学影像）进行 PII 擦除。
- **部署灵活**：单机 pip 即可，也提供 Docker 镜像和 Kubernetes 部署方案，甚至支持 Spark 批处理。

## 技术栈
- **语言**：Python
- **核心 NLP**：spaCy（可选用 transformer 模型）
- **可选增强**：Hugging Face Transformers（用于定制 NER）、校验和算法（信用卡、身份证号等）
- **打包**：Docker、Kubernetes、PySpark
- **依赖库**：pyyaml、pillow（图像编辑）等

## 核心能力
1. **文本识别**：预置 30+ 种实体类型（姓名、电话、信用卡、货币、地址等），支持自定义正则/规则识别器。
2. **上下文增强**：利用周围文本提升准确率，减少误报。
3. **匿名化策略**：替换、屏蔽、加密、哈希、假名化等。
4. **图像编辑**：使用 OCR 检测文字后模糊/黑色覆盖，支持 DICOM 医学格式。
5. **结构化数据**：通过 `presidio-structured` 处理表格或数据库查询结果。
6. **多语言**：已支持英语、西班牙语等多种语言（通过 spaCy 模型）。

## 适用场景
- 日志脱敏、客服对话记录隐私处理
- 医疗数据去标识化（PHI）和临床实体识别（新增 MedicalNER）
- 金融合规：信用卡号、社保号等自动匿名
- 数据科学工作流中的隐私预处理器
- 图像文档（如证件扫描件）的自动打码

## 同类对比
- **spaCy 直接使用**：缺乏匿名化层和图像处理，需大量额外开发。
- **Google Cloud DLP**：托管服务，灵活度低且无法离线。Presidio 适合自有基础设施、需要精细控制且不依赖云 API 的企业。
- **其他开源（如 redact-pii）**：社区规模小，规则集简单，缺乏模块化流水线和微软的安全维护。
