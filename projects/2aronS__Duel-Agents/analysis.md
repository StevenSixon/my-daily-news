## 它是什么
Duel Agents 是一个面向开发者的模块化工具包，核心是一个**多模型路由代理层**：它接收提示词，并行发送给多个大语言模型，并自动选择成本最低但仍然“获胜”的答案。项目以 CLI 安装器、TypeScript SDK 和 IDE 插件的形式提供，让开发者无需离开熟悉的工作环境即可使用该路由功能。

## 为什么火
项目在短时间内获得近 1000 星，可能源于 AI 编码工具爆炸式增长带来的**成本优化刚需**。它从 IDE 切入，直接解决开发者用 AI 助手时无法自动选择性价比最高模型的问题，门槛极低。附带推出的 $DUEL 代币也增加了加密社区的关注度。

## 技术栈
- **语言**：TypeScript（全栈）
- **运行环境**：Node.js 20/22
- **包管理**：npm（monorepo 结构，但截至 README 撰写时包未公开发布）
- **核心依赖**：所有请求通过 `https://duelagents.com/v1` 代理转发，不直接使用各模型原厂 API。
- **集成对象**：Claude Code（插件）、Cursor（Skill）、Codex CLI、OpenClaw
- **Python 辅助**：官方提供 langchain-duel 和 llama-index-llms-duel 包

## 核心能力
- **多模型路由**：同时调用多个模型，按“最便宜且质量可靠”准则返回结果，模型列表由后端控制。
- **IDE 原生集成**：为 Claude Code、Cursor、Codex、OpenClaw 提供一条命令安装，自动写入环境变量和配置文件。
- **统一 SDK**：`@duel-agents/sdk` 同时提供 OpenAI 和 Anthropic 兼容的 API 客户端，一个密钥通吃两种接口。
- **环境模板与诊断工具**：内置 `doctor` 命令验证连接和 API 密钥有效性。

## 适用场景
- 使用 Claude Code 或 Cursor 等 AI IDE 插件、希望降低 Token 费用同时保持输出质量的开发者。
- 无需自建复杂模型选择逻辑，只需一个密钥即享“多模型打擂台，选性价比最高”的团队。
- 对加密代币经济（$DUEL）有参与兴趣，愿意早期尝试新 AI 基础设施的用户。

## 同类对比
- **自建路由/代理**：灵活但开发维护成本高，且难实现复杂的质量评判；Duel 开箱即用，专注 IDE 场景。
- **OpenRouter 等第三方多模型网关**：OpenRouter 提供广而全的模型列表，但未深度嵌入 IDE，且路由策略不同。Duel 强调“最便宜赢家”且与 IDE 工作流深度绑定。
- **原生多模型键**：一些平台支持同时调用多个模型，但缺乏智能优选，通常需用户手动切换。
