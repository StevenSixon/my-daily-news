## 它是什么
eve 是 Vercel 推出的文件系统优先的持久化 AI 代理框架。项目的核心目标是让代理能力的定义回归到文件系统结构：指令、工具、技能、频道、调度等都按约定目录放置，开发者只需编写 Markdown 和 TypeScript 文件，无需理解复杂的链式 API。代理运行时读取这些文件并拼接成一个可交互的代理。

## 为什么火
由 Vercel 背书，贴合其“前端即产品”的哲学。将代理构建简化为类似 Next.js 的文件约定，踩中开发者对低认知负担框架的渴望。虽然 Star 数尚在早期，但其“文件即配置”的理念在众多代码式代理框架中令人耳目一新，引发讨论。

## 技术栈
- **语言 & 运行时**：TypeScript，Node.js
- **核心依赖**：Zod（工具参数校验）、可选沙箱引擎（支持 workspace 提升安装）
- **多渠道支持**：内置 HTTP、Slack、Discord 等频道集成
- **客户端**：`eve/react` 包支持 React 集成，使用 Zod 校验 `/eve/v1/info` 接口
- **模型接入**：通过字符串标识选择模型（如 `anthropic/claude-sonnet-4.6`），模型调用细节由框架封装
- **文档内嵌**：`eve` npm 包包含完整 `docs`，方便编码代理在本地读取

## 核心能力
1. **指令层**：`instructions.md` 作为始终激活的系统提示
2. **工具**：`tools/*.ts` 通过 `defineTool` 定义可调用函数，参数以 Zod schema 声明
3. **技能**：`skills/*.md` 按需加载的规程，扩展代理能力
4. **频道**：`channels/*.ts` 设定消息接入点，支持 Slack、Discord 等
5. **调度**：`schedules/*.ts` 定义 Cron 作业，实现定时任务
6. **子代理**：支持在代理内调用子代理（文档含 `subagents.mdx`）
7. **人机交互**：可在工作流中插入人工审批环节
8. **沙箱**：可选的沙箱引擎，提供安全执行环境

## 适用场景
- 需要快速搭建面向用户或团队的 AI 助手（如客服、内部问答）
- 多平台同步部署的代理（Slack + Discord 等）
- 定时报告、周期工作流
- TypeScript 生态内的原型或生产级代理开发

## 同类对比
- **vs LangChain**：LangChain 以链式调用和抽象类为核心，学习曲线陡峭；eve 只有文件和函数约定，项目结构更扁平，更适合偏好约定优于配置的团队。
- **vs AutoGPT / CrewAI**：后者偏重自主任务分解和 agent 协作，配置复杂；eve 聚焦单代理的基础能力组件化，更可控和易调试。
- **vs 传统脚本**：eve 提供了频道、调度等生产必需模块，同时保留文件系统的透明性，比自写胶水代码更健壮。
