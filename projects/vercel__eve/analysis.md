## 它是什么

eve（Vercel 出品）是一个以文件系统为首要创作界面的持久化 AI 代理框架。代理的所有核心能力——系统提示、工具、技能、频道、定时任务——都按约定放在特定的文件与目录中，使得项目结构即代理逻辑，便于检查、扩展和运维。它用 TypeScript 编写，提供 `npx eve@latest init` 脚手架，内置交互式终端 UI，并支持快速接入 Anthropic 等模型。

## 为什么火

- **极简约定**：代理的入口、工具、指令一目了然，无需复杂的对象组装或隐式配置。
- **Vercel 背书**：由 Vercel 团队维护，与 Vercel 生态（如 AI SDK）存在潜在协同。
- **持久化与协作**：文件天然可版本控制，适用于需要长期维护、多人参与的代理项目。
- **beta 阶段热度**：Star 增长较快，社区在 GitHub Discussions 活跃。

## 技术栈

- 语言：TypeScript
- 运行时：Node.js
- 模型接入：通过 `defineAgent` 配置模型 ID（如 `anthropic/claude-sonnet-4.6`），需外部 API 密钥
- 工具校验：使用 Zod 定义输入 schema
- 交互频道：内置 HTTP、Slack、Discord 等频道支持
- 调度：基于 cron 的 jobs
- 沙箱：主题标签提及 sandbox，但本文档未展开说明，可能指执行环境隔离

## 核心能力

1. **指令即文件**：`instructions.md` 是常驻系统提示，修改即更新行为。
2. **工具定义**：`tools/` 下的 TypeScript 文件导出 `defineTool`，带 Zod schema 与异步执行逻辑。
3. **技能**：`skills/` 目录存放按需加载的过程性 Markdown。
4. **频道**：`channels/` 支持多种消息渠道接入（HTTP、Slack 等）。
5. **定时调度**：`schedules/` 定义周期性任务。
6. **人工干预**：支持 human-in-the-loop 提示。
7. **子代理**：可创建子代理协调复杂任务。

## 适用场景

- 快速原型：想用最短路径得到一个可交互的 AI 助手。
- 团队开发：需要可审查、可版本化的代理配置，避免黑盒。
- 渠道集成：需要同时对接 Slack、Discord 等消息平台的机器人。
- 工作流自动化：结合定时任务和工具调用，实现定期报告、数据抓取等。

## 同类对比

对比 LangChain Agent 或 Semantic Kernel，eve 的差异化在于：
- **文件系统优先**：将代理逻辑显式化为文件结构，而非全部代码内组装。
- **轻量约定**：不引入复杂的链式抽象，学习曲线较低。
- **Vercel 原生**：可能深度集成 Vercel 部署和 AI SDK（具体整合待确认）。
当前它更偏向个人与团队的“文件即代理”哲学，而 LangChain 等更偏库式编排。
