## 它是什么

eve 是 Vercel 推出的文件系统优先的 TypeScript 框架，用于构建持久化的 AI 代理（agent）。它强制将代理的核心能力——指令、工具、技能、通道、定时任务——落入约定的目录结构，使项目可直接通过文件树理解行为，降低认知负担。

## 为什么火

- **Vercel 背书**：依托 Vercel 的开发者生态和设计哲学，与 Next.js 一脉相承的“约定优于配置”体验。
- **零胶水起步**：一个 `instructions.md` 加可选工具函数即可运行，初始化命令自动搭建项目。
- **开发友好**：本地 `eve dev` 提供交互式终端，支持动态模块热重载，模型与工具变更即时生效。
- **可扩展性**：内置通道（HTTP、Slack、Discord）、定时任务、子代理等机制，覆盖常见 AI 应用场景。

## 技术栈

- 语言：TypeScript（Node.js 运行环境）
- 核心依赖：Zod（输入校验）、Vercel AI SDK（模型集成，推测）
- 执行模型：本地进程执行工具/skill，可选沙箱隔离（文档提及 sandbox）
- 分发方式：npm 包，脚手架通过 `npx eve@latest init` 使用

## 核心能力

1. **指令系统**：`instructions.md` 作为持久化系统提示词，支持 Markdown 语法，模型始终可见。
2. **工具定义**：TypeScript 函数通过 `defineTool` 暴露给模型，使用 Zod 声明输入格式，同步或异步执行。
3. **技能**：按需加载的 Markdown 过程描述或程序模块，适合多步工作流。
4. **通道**：声明式接入 Slack、Discord 等外部消息接口，统一消息路由。
5. **定时任务**：类似 Cron 的周期性动作，用 TypeScript 定义。
6. **模型灵活**：通过 `agent.ts` 切换提供商（如 Anthropic Claude），支持代码级模型解析与动态导入。
7. **子代理与动态映射**：可嵌套代理，动态工具映射支持命名空间隔离，避免冲突。

## 适用场景

- 个人助理或聊天机器人（支持多通道）
- 自动化工作流（定时任务 + 工具调用）
- 需要快速原型的多步骤 AI 应用
- 希望代码与配置分离、利于团队协作的项目

## 同类对比

- **vs. LangChain/LlamaIndex**：eve 不提供庞大的工具链和 RAG 抽象，而是聚焦文件和约定，更轻量、侵入性更低。
- **vs. AutoGen/ CrewAI**：后者侧重多代理对话编排，eve 以单代理内嵌工具、技能和子代理方式实现，更贴近传统应用开发。
- **独特性**：全文件驱动，开发体验极简，天然适合作为 AI 能力嵌入现有 Node.js 项目。

## 版本动态

- **最新版本**：0.12.0（2026-06-21）
- **关键变更**：动态工具映射的 key 不再自动添加文件标识前缀，改为使用原始名称，若冲突需手动命名空间；连接器工具按 `<connection>__<tool>` 格式重命名。修复了 `eve dev` 下代码定义模型的解析问题，子代理可复用父级模块映射以避免重复构建。
- **发展阶段**：仍处 Beta，API 和行为可能变动，遵循 Vercel Beta 条款。
