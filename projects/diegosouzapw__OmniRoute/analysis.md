## 它是什么
OmniRoute 是一个本地运行的免费 AI 网关，它把 236 个模型提供商（其中 50+ 有免费额度）聚合到一个 `/v1` 端点。只需在 IDE 或 CLI 工具（Claude Code、Cursor、Cline、Copilot 等）中配置为 OpenAI 兼容端点，就能自动在订阅额度、API 密钥、廉价和免费模型之间按策略故障转移。

## 为什么火
- **解决真痛点**：开发者常面对多个 AI 订阅、每月配额用不完或不够用、速率限制打断编码流程。OmniRoute 让所有工具共用同一个配额池，并透明压缩 token，最高可省 95%。
- **零成本起步**：无需信用卡，直接聚合免费模型（如 Kiro、Qoder、Pollinations），首月增量额度可达 ~2.1B token。
- **极客友好**：本地优先（localhost:20128），MIT 开源，可自建可 npm 全局安装，适合把 AI 接入当作基础设施的硬核用户。

## 技术栈
- 语言：TypeScript
- 运行环境：Node.js ≥22，支持 Docker、Electron 桌面应用
- 核心依赖：未列出，但从代码结构看有压缩流水线、多提供商适配器、路由引擎、MCP/A2A 工具集
- 测试：14,965 个测试

## 核心能力
- **智能路由**：4 层自动回退（订阅→API密钥→廉价→免费），17 种路由策略可选，实现毫秒级切换
- **双重压缩**：RTK（重复工具知识） + Caveman（类穴居人缩写），堆叠式压缩，内置膨胀保护，避免压缩后比原文更长
- **广兼容**：兼容 OpenAI、Claude、Gemini、Responses API，支持 MCP（87 个工具）、A2A 代理
- **可靠与安全**：熔断器、TLS 指纹伪装、代理支持，可突破地区封锁
- **全功能仪表板**：Web/PWA 界面，多语言（41+），实时免费额度追踪（/dashboard/free-tiers）

## 适用场景
- 想最大化多个 AI 工具免费额度、避免订阅浪费的个人开发者
- 需要在组织内统一 AI 接口，并按成本/策略路由的团队
- 网络受限地区需要代理和伪装使用 AI 的用户
- 频繁使用编码代理（Claude Code、Cline、Aider 等），希望降低 token 成本和速率失败率

## 同类对比
- 与 LiteLLM、Portkey、OpenRouter 等服务相比，OmniRoute 更侧重本地运行与免费额度聚合，而非单纯的多模型代理或付费 API 路由。其内置的压缩引擎和免费层实时计数是差异化亮点。README 未提供性能对比或基准数据，实际高频吞吐下的延迟表现待验证。
