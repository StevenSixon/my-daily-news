## 它是什么
Superpowers 是一套面向编码代理（如 Claude Code、Codex、Cursor 等）的技能框架与软件开发方法论。它并非一个独立工具，而是通过安装在各种编码代理中的“技能包”，在对话中自动触发**TDD（测试驱动开发）红绿重构**、**子代理驱动开发（SDD）**、**系统化调试**、**代码审查**等严格流程，强制代理遵循设计、计划、实施、审查的闭环。重点在于将模糊的需求转化为可执行的任务列表，并由新起的子代理执行每个小任务，每完成一个任务即审查，避免跑偏或直接堆砌代码。

## 为什么火
- **明星作者和团队**：由 Prime Radiant 的 Jesse Vincent（原 DuckDuckGo 搜索负责人）创建，社区活跃，Discord 群和定期邮箱公告。
- **解决 AI 编码的痛点**：完全针对实际痛点——“跳过设计直接写代码”“不写测试”“计划执行半途而废”，通过强制性工作流（不是建议）规范代理行为。
- **子代理驱动开发 (SDD)**：突破单代理上下文限制，每个任务用全新上下文执行，类似人类开发者的“清空大脑、专注小任务”，效果显著。
- **广泛平台支持**：覆盖几乎全部主流编程代理（Claude Code、Codex、Cursor、Copilot CLI、Gemini CLI 等），形成生态，降低使用门槛。
- **社区反馈**：Star 数高达 23.3 万（截止 2026 年），远超一般开发工具。

## 技术栈
- **主体**：纯文本技能文件（`SKILL.md`）及元数据，在各代理中通过自带的 Skill/Plugin 机制加载。
- **集成方式**：为每个目标代理提供各自的 marketplace/插件包或安装脚本（Bash、Shell）。
- **SDD 基础设施**：在 workspace 中维护 `.superpowers/sdd/` 目录（存放任务简报、执行报告、审查差异和进度台账），通过共享的 `sdd-workspace` 辅助脚本为各个子代理提供隔离上下文。
- **测试框架无关**：依赖项目本身的测试运行器；技能定义的是测试驱动的行为逻辑，不绑定具体框架。
- **部署前提**：仅需目标编码代理的安装。

## 核心能力
1. **脑暴与设计审批**：在编写代码前，先通过 Socratic 对话提取需求，输出分块设计文档供人类确认。
2. **工作树隔离**：利用 git worktree 创建独立分支和干净环境，确保测试基线。
3. **计划生成**：将设计拆分为 2–5 分钟的小任务，包含精确文件路径、完整代码和验证步骤。
4. **子代理驱动执行**：每个任务用新子代理执行，过程中进行两阶段审查（规范合规、代码质量），关键问题阻塞进度。
5. **强制 TDD**：红-绿-重构循环，先写失败测试，再写最少代码使测试通过，提交前删除未经测试的代码。
6. **自动代码审查**：审查结果按严重性分级，严重问题阻塞前进。
7. **结项流程**：完成后验证全部测试通过，提供合并/创建 PR/保留/丢弃分支的选项，清理工作树。
8. **可组合的技能库**：涵盖测试、调试、协作、元技能，可扩展（`writing-skills` 指导创建新技能）。

## 适用场景
- **个人或团队使用 AI 编码代理进行严肃软件开发**，特别是不满足于“一次性代码生成”，希望确保代码可测试、可审查、可维护的项目。
- **原型到产品的全流程**：从需求探索、架构设计到功能交付，尤其适合中小型功能迭代。
- **多代理协作探索**：通过并行代理技能（`dispatching-parallel-agents`）调查多个方案。
- **企业级定制**：提供了商业支持和托管消费选项。

## 同类对比
- **传统代码生成提示词/教程**：多为静态建议，无强制执行和状态管理。Superpowers 本质是一个运行时方法论引擎。
- **Cline/Open Interpreter/Claude Code 原生功能**：这些工具允许自由编写提示，但缺乏内置的结构化开发流程（脑暴→计划→TDD→子代理审查→结项）。Superpowers 像“给代理的操作系统”。
- **LangChain/其他代理框架**：侧重于搭建代理流水线，而 Superpowers 无需编程，直接作为现有代理的“驾驶指引”。
- **SWE-bench 风味的自动修复工具**：专注于修复或短任务，Superpowers 偏重长链路功能开发方法论。
