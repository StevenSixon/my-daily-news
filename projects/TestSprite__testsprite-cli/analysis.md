## 它是什么
TestSprite CLI 是 TestSprite 平台的官方命令行工具，为AI编程智能体（如Claude Code、Cursor、Codex）提供**真实环境下的全自动端到端测试验证循环**。测试在云端浏览器或真实API上运行，而非模拟；智能体创建测试 → 运行 → 失败时拉取一份自洽的失败定位包（含截图、DOM、代码、根因推断）→ 修复后重跑，所有步骤在终端内闭环完成。

## 为什么火
随着AI编码的普及，代码产出速度远超人工验证速度，传统E2E测试和mock环境难以跟上。TestSprite CLI 直接嵌入智能体的工作流：**智能体在提交代码前就能自测真实行为，避免破坏现有功能**。其在CoderCup公开排行榜上，以最便宜的模型搭配该CLI取得了89%最高正确率，且成本仅为最贵模型的一半，因此迅速获得关注。

## 技术栈
- 语言：TypeScript
- 运行环境：Node.js ≥ 20
- 底层自动化：Playwright（CLI内部不直接暴露，但主题标签含playwright）
- 通信：与TestSprite云端API交互，使用`TESTSPRITE_API_KEY`鉴权
- 输出格式：支持`--output json`，提供稳定的机器可读契约

## 核心能力
- **一键集成**：`testsprite setup` 完成API密钥配置、验证及向智能体安装验证技能文件（如claude、cursor等）
- **真实环境测试**：在前端测试中启动真实浏览器进行点击、导航和断言；后端测试直接调用真实API
- **面向智能体的失败定位**：`test failure get` 输出单一快照的失败步骤、相邻步骤、截图、DOM、测试源码、根因假设和修复目标，杜绝跨运行数据拼接
- **持久化测试套件**：每次测试运行都累积到项目中，形成持续增长的验证覆盖
- **批量操作与脚本化**：提供批量创建、项目级全部运行/重跑等命令，稳定的退出码和`--dry-run`模式便于CI集成

## 适用场景
- 使用AI编程助手（Claude Code、Cursor、Cline等）快速迭代的项目，需要即时反馈代码是否破坏现有功能
- 希望从手动E2E测试转向自动化、且不想维护测试基础设施的团队
- 需要将测试验证融入CI/CD流水线，并让AI智能体自主修复失败

## 同类对比
与Playwright、Cypress等传统测试框架相比，TestSprite CLI **不需要编写测试脚本**，而是通过自然语言描述行为生成测试计划，并由云端执行。与一般AI测试工具不同，它**专为编程智能体设计**，输出为智能体可直接消费的失败定位包，而非给人看的仪表盘。但它依赖TestSprite云服务，非完全自托管。
