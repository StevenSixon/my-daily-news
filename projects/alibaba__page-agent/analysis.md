## 它是什么
Page Agent 是阿里巴巴开源的纯浏览器端 GUI Agent，运行在用户当前网页内，通过文本 DOM 操作理解页面结构，用自然语言完成点击、填表、导航等任务。它不依赖后端服务、截图或多模态模型，只需注入一段 JavaScript。

## 为什么火
项目 2025 年 9 月发布，已获超 1.9 万星。最大卖点是将复杂的 Web 自动化能力浓缩为一键式前端脚本，极大降低了集成门槛，尤其吸引 SaaS 开发者、无障碍倡导者和自动化爱好者。

## 技术栈
- 核心：TypeScript 编写，编译为可在浏览器运行的 IIFE/ESM 包。
- AI 层：通过兼容 OpenAI 格式的 API 调用 LLM（支持自托管），默认使用通义千问。
- 感知方式：序列化 DOM 树为结构化文本，交由 LLM 决策操作步骤，无需视觉模型。
- 扩展能力：可选的 Chrome 扩展用于多标签页自动化，并提供 MCP Server（Beta）供外部智能体控制浏览器。

## 核心能力
- **文本式 DOM 理解**：将网页元素转化为 LLM 可解析的结构化描述，实现精准操作。
- **可扩展动作**：内置点击、输入、滚动、等待、执行 JavaScript 等工具，且支持 abort 信号。
- **多页面代理**：通过 Chrome 扩展跨标签页工作，已禁用危险的脚本执行工具以提高安全性。
- **生命周期管理**：v1.10.0 重构了运行状态机，支持异步停止、独立的任务状态和错误记录。
- **暗色模式感知**：优化了页面主题检测，避免误判。

## 适用场景
- **SaaS AI Copilot**：为你的产品添加智能助手，用户用自然语言完成复杂操作。
- **表单自动填充**：将多步骤录入浓缩为一句话指令，适合 ERP、CRM 等企业系统。
- **无障碍增强**：让视障用户通过语音或文本命令控制网页，无需特殊适配。
- **跨页面工作流**：配合扩展实现跨标签页的数据抓取或流程自动化。

## 同类对比
- vs. **browser-use**（Python 库）：browser-use 需要 Python 环境和 headless 浏览器，适合服务端自动化；Page Agent 仅需在目标页面嵌入 JS，天然贴合前端增强场景，且无需截图。
- vs. **Playwright + AI**：Playwright 同样需要 Node 环境，无法直接嵌入用户浏览器。Page Agent 部署成本极低，但受限于单页面域（多页面需扩展）。
- vs. **其他浏览器扩展 AI 助手**：Page Agent 提供 NPM 包和 CDN，允许开发者自定义集成，而非封闭的应用。

## 版本动态
最新 v1.10.0（2026-06-15）带来破坏性变更：Agent 运行生命周期重写，`stop()` 变为异步且状态与任务结果解耦，新增 `stopped` 状态和 `agent.lastResult`。功能层面新增可中止的 JavaScript 执行、精简 Agent 提示词以降低 LLM 认知负荷、禁用了 MultiPageAgent 的脚本执行工具以提升安全性。修复了滚动断言、等待上报和文档视频等问题。
