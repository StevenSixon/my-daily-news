## 它是什么
FanBox 是一个本地化 AI coding agent 的集成工作台（“驾驶舱”）。它将文件浏览器、Markdown/代码/图片预览、真实终端（node-pty + xterm.js）融为一体，专为同时跑多个 AI agent 项目的极客设计，解决“AI 快速生成十个项目后迷失在 Finder 里”的痛点。

## 为什么火
AI 辅助开发让项目数量爆炸，文件改动过程不可见，找回和对比困难。FanBox 提供了**活的仪表盘**：agent 每写一个文件，对应卡片即亮起呼吸光效，可跟随模式实时预览 Markdown 渲染、HTML 实时网页，并支持会话回放。加上**零配置、无账号、完全本地**，切中追求透明与控制的技术用户心理，在 GitHub 上迅速获得关注。

## 技术栈
- 桌面端：Electron 框架，原生 Apple Silicon 支持
- 代码编辑器：Monaco（VS Code 内核），Milkdown（Markdown 所见即所得）
- 终端：node-pty + xterm.js（WebGL 渲染，中文宽字符良好）
- 文件改动追踪：影子 git（非侵入式快照）+ 回合安全带（自动存档/回滚）
- 构建/签名：Developer ID 签名 + 苹果公证，免去“未验证开发者”弹窗

## 核心能力
**文件找回与预览**：⌘K 全局模糊搜索，`内容:关键词` 切全文查找；强色实体图标 + 缩略图加速；Markdown 渲染、HTML 实时预览、代码高亮、图片/视频/PDF 内嵌。
**agent 改动透明化**：文件卡片按改动频率发光呼吸，跟随模式跟踪活动文件，会话回放拖拽时间轴重现修改历程，跨项目变更收件箱汇总变动。
**agent 驾驶舱**：内嵌终端直接运行 Claude Code/Codex 等 11 种 agent；截图即喂给终端；项目记忆查看历史会话并一键续接；agent 状态感知与通知。
**编辑与辅助**：Markdown 自动保存、代码编辑器 vs-code 体验、图片标注/压缩；未保存退出拦截。
**发版向导**：一键串联版本号、CHANGELOG、打包、推送、GitHub Release。

## 适用场景
- 使用 Claude Code/Codex 等 AI coding agent 并行处理多个项目，需要集中管理文件与终端
- 需要审查 AI 对本地文件的每一步改动，并快速回退
- 偏好本地工具、反对云服务、注重隐私的开发者
- 需要轻量级文件浏览+预览+agent 指挥，不依赖 VS Code 等重型编辑器

## 同类对比
对比 **VS Code**：FanBox 不拼编辑体验，更侧重“浏览+指挥 agent +改动回溯”一条链路，界面更轻量；VS Code 需装插件、需要打开项目，FanBox 跨项目一站式。
对比 **Finder + iTerm**：FanBox 将两者有机组合，文件与终端联动，增加实时改动高亮和会话回放能力。
对比 **Warp**：Warp 强在终端智能，FanBox 强在文件与 agent 的融合展示和控制。

## 版本动态
最新 v2.5.0 引入**回合安全带**机制，agent 每轮自动存档，一键回到任意历史回合，且不污染 git 仓库；diff 查看扩展到无 git 目录，回溯更安全。持续通过社区反馈迭代，但仅官方支持 macOS，Windows 依靠社区非官方移植。
