## 它是什么
FanBox 是一个 macOS 桌面应用（Electron），专为 vibe coding 场景设计，将文件浏览、agent 终端与实时预览融于一个窗口，提供项目记忆、改动仪表盘、回放等能力，实现“找回→指挥→看清”的闭环。

## 为什么火
Star 数增长迅速，社区积极响应（Windows 移植、微信遥控等），因为精准命中了 AI 辅助编程后“项目爆炸、不可追溯、上下文割裂”的痛点，本地优先、零配置的理念也符合极客偏好。

## 技术栈
- Electron + Monaco Editor（代码编辑）
- node-pty + xterm.js（真实终端）
- Milkdown Crepe（Markdown 所见即所得）
- 集成 Claude Code / Codex CLI
- 多种文件预览工具（图片、视频、PDF、HEIC 等）
- 三套皮肤（Volt、Archive、Index）

## 核心能力
- **文件找回与预览**：全局模糊搜索、强色图标、原地预览 Markdown/HTML/代码/图片/PDF。
- **Agent 改动感知**：文件卡片涟漪发光、跟随模式、会话回放、变更收件箱、Git diff 对比。
- **Agent 驾驶舱**：项目记忆与续接、截图直通车、AI 整理建议、发版向导、Skills 透视、用量监控。
- **实战终端**：真实嵌入式终端，支持拖入文件路径、点击路径跳转、选中发送内容、态势感知通知。
- **编辑与标注**：代码/JSON 用 Monaco、Markdown 所见即所得、图片标注与压缩。

## 适用场景
- 同时管理多个 AI 生成项目，快速找回、预览。
- 配合 Claude Code/Codex 进行高强度 vibe coding，观察 agent 每一步改动。
- 项目复盘与回放，了解 AI 操作历史。
- 多项目并行跑 agent 时统一监控改动。

## 同类对比
- **VS Code**：更重编辑，没有原生项目记忆和 agent 改动仪表盘。
- **iTerm2 + Finder**：分散窗口，无联动。
- **Warp**：终端增强，但缺少文件浏览和 agent 专属功能。
- **其他 AI 编程工具**：多云端绑定，FanBox 完全本地、不读文件内容，隐私优先。

## 版本动态
