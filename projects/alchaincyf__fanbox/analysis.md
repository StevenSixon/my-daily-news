## 它是什么
FanBox 是一款 macOS 原生桌面应用，将文件管理器、真实终端和文件预览整合在一个窗口，为 Claude Code、Codex 等 coding agent 提供可视化操作面板。它不依赖云端服务，无需账号或配置，专注解决“AI 生成大量项目后难以找回和追踪改动”的痛点。

## 为什么火
随着 coding agent 的普及，开发者一个下午可能启动数十个项目，散落在文件夹中，名称无辨识度，改动不可见。FanBox 把“找回 → 运行 agent → 查看改动”的完整循环收敛到单一界面，配合涟漪动画、会话回放、项目记忆等特色功能，大幅降低认知负担。本地优先的设计契合注重隐私的用户，社区已自发移植 Windows 版本，侧面印证其需求刚性和设计吸引力。

## 技术栈
- **桌面框架**：Electron（发布为 Apple Silicon arm64 原生包，已签名并启用 Hardened Runtime）
- **终端**：node-pty 负责伪终端，xterm.js（WebGL 渲染）实现内嵌终端，支持 CJK 宽字符和常见控制序列
- **编辑器**：Monaco Editor（VS Code 内核）用于代码和 JSON 编辑，Milkdown Crepe 实现 Markdown 所见即所得
- **预览**：内建 Markdown 渲染、HTML 实时预览、媒体文件显示（支持 HEIC），缩略图加载优化至 0.1 秒以内
- **截图与测试**：README 中的截图均由 Playwright 从实际应用自动拍摄
- **设计系统**：huashu-design 提供三套整体皮肤，包括配色、字体、图标和终端 ANSI 主题的联动变换

## 核心能力
- **文件找回与预览**：⌘K 全局模糊搜索，支持全文检索；文件按类型显示强色图标和项目徽章；原地预览代码高亮、Markdown 渲染、HTML 实时效果、媒体、PDF 和压缩包
- **Agent 改动可视化**：文件卡片以涟漪和呼吸光效反馈 agent 写入；一键跟随模式实时跟踪 agent 正在编辑的文件；会话回放时间轴查看历史改动；变更收件箱跨项目汇总所有修改
- **Agent 驾驶舱**：项目记忆（历史会话、修改文件列表、一键续接上下文）、截图直通车、AI 基于元数据的整理提案、发版向导、Skills 管理、Agent 用量统计、磁盘占用透视
- **终端集成**：拖拽文件插入路径、输出中的文件路径可点击打开、选中预览内容发送到终端（bracketed paste）、运行状态指示与系统通知
- **编辑与守卫**：三种编辑器的未保存退出统一拦截

## 适用场景
- 用 Claude Code 或 Codex 同时进行多个项目的开发或实验，需要集中管理文件变更和会话
- 需要快速检索一个下午生成的十个项目并识别其类型
- 偏好本地运行、不信任云端文件管理工具的开发者
- 希望减少 IDE 重型功能干扰，只关注“找回-预览-指挥 agent”轻量化流程的用户

## 同类对比
- **Finder + iTerm**：日常流程需切换多个窗口，缺少文件改动追踪和会话记忆，FanBox 将三者合一并增加 agent 专属功能
- **VS Code**：强调代码编辑和大型项目管理，FanBox 不做重型编辑，聚焦快速找回和 agent 交互，角色互补
- **Warp Terminal**：现代终端但缺少文件管理与 agent 改动可视化能力，FanBox 的差异化在于文件侧边栏和项目记忆
