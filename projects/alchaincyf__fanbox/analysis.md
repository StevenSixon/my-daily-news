## 它是什么
FanBox 是一个专为 vibe coding（AI 辅助编码）设计的本地桌面应用，充当 Coding Agent（如 Claude Code、Codex）的“驾驶舱”。它将文件浏览、终端、预览整合在一个窗口，并实时追踪 Agent 对文件系统的每次改动，让开发者不再迷失于散落的项目文件和不可见的修改中。

## 为什么值得关注
vibe coding 流程中，开发者常面临“一个下午起十个项目，然后就再找不到了”的痛点。FanBox 通过全局模糊搜索、项目类型徽章（node/web/py等）和会话记忆，快速找回项目；它不像 Finder 那样只有文件操作，也不像 VS Code 侧重编辑，而是聚焦于“找回→预览→轻改→指挥 Agent”的链式体验。与同类工具相比，它提供了**实时文件改动涟漪动画、跟随模式、会话回放**等可视化反馈，让 Agent 的工作从未如此透明。

## 技术栈
- **Electron**：桌面框架，实现本地文件系统访问、内嵌终端和编辑器。
- **node-pty + xterm.js (WebGL)**：真实终端仿真，支持 Claude Code、vim 等复杂应用。
- **Monaco Editor**：嵌入与 VS Code 同源的代码编辑器，提供 Diff 对比。
- **Milkdown Crepe**：WYSIWYG Markdown 编辑器。
- **本地优先**：零云服务、零账号，一切都在本地，适合数据主权意识强的开发者。
- **设计系统**：采用 huashu-design，提供三套完整皮肤（Volt/Archive/Index）。
- **构建**：无需 build，clone 即可通过 `node server.js` 运行网页版（仅浏览部分），桌面版通过 Electron 打包。

## 核心能力
- **文件找回与预览**：⌘K 全局模糊搜索，可按内容搜索；支持 Markdown 渲染、HTML 实时预览、图片/视频/PDF 内嵌预览，缩略图加速。
- **Agent 改动可视化**：Agent 每写一个文件，卡片会发光涟漪；跟随模式自动跟踪 Agent 编辑的文件，并实时刷新预览；会话回放可回溯修改历史；变更收件箱汇总所有被修改文件。
- **Agent 驾驶舱**：项目记忆（会话历史、改动的文件、一键 resume）、截图直通车、AI 整理提案（元数据驱动、可撤销）、发版向导、Skills 透视、用量监控、磁盘占用透视。
- **真实内嵌终端**：支持拖入文件路径、点击终端内路径在 FanBox 打开、选中文本发送到终端，态势感知标签显示 Agent 状态。
- **所见即所得编辑**：Markdown、代码/JSON、图片标注，三者均支持未保存守卫。
- **隐私与本地**：不上传任何数据，签名 DMG 分发，内置更新提醒。

## 适用场景
- **vibe coding 开发者**：频繁使用 Claude Code、Codex 等命令行 Agent，需要集中管理多个生成的项目。
- **全栈快速原型**：一个下午尝试多种技术栈（Node/Python/Go 等），需要快速找回并对比。
- **代码审查 Agent 输出**：需要看清 Agent 具体改了哪些文件的哪些行，支持 HEAD vs 工作区的 diff。
- **多项目并行 Agent**：同时运行多个 Agent 作业，集中监控改动。

## 同类对比
同类工具有 VS Code + 终端、Warp 终端、Hyper 等，但 FanBox 的独特之处在于：
- **VS Code**：重编辑器，文件浏览和终端是附属；FanBox 以文件浏览和 Agent 交互为中心，编辑是辅助。
- **Warp 终端**：强现代终端，但无文件浏览与改动追踪。
- **Hyper 终端**：可扩展但无文件管理。
- **Finder/Terminal 组合**：割裂，缺乏实时改动反馈和会话记忆。
FanBox 不拼编辑体验，而是专注“指挥 Agent 和工作流整合”。
