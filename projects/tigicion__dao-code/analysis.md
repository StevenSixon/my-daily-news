## 它是什么

DAO CODE（命令 `dao`）是一个终端原生的 AI 编码助手，专为 DeepSeek V4（1M 上下文）打造。它在终端内读代码、写代码、跑命令、修 bug，流式展示推理过程并通过审批门安全执行。设计理念是**不靠贵模型堆体验，靠工程把高性价比模型的体验提升上来**，灵感来自 Claude Code，但追求极低成本与国内直接可用。

## 为什么火

- **可用性**：Claude 需要账号和网络门槛，DAO 完全 MIT 开源，DeepSeek 注册即用、国内直连。
- **成本**：利用 DeepSeek 的低价和前缀缓存，实测 7 个真实任务总成本 ¥1.07，输入 token 聚合缓存命中率 95.8%，比 Claude Opus 4.8 省约 30 倍。
- **自我验证记忆**：跨会话记忆在启动时经过确定性校验，过期或改变的自动剔除，避免盲目堆历史。
- **反思层**：遇到卡顿时自动派独立视角怀疑性复核，长任务中每 N 轮纠偏防 scope 蔓延，这些操作复用主前缀缓存的 fork，几乎零额外成本。
- **长任务稳健**：崩溃恢复、影子 git 检查点、任务清单穿越压缩、DoD 验收、卡死检测等机制，支持自主跑很久不跑偏。
- **生态兼容**：兼容 Claude Code 的 settings、skills、hooks、MCP 配置格式，现成的 CC 方案可直接迁移。

## 技术栈

- 语言：TypeScript
- 运行时：Node.js ≥ 20
- AI 模型：DeepSeek V4（Pro 和 Flash 可切换）
- 终端 UI：Ink（React for CLI）
- 测试：Vitest
- 发布格式：npm 包、独立二进制（多平台）
- 版本管理：MIT 开源，仓库使用 GitHub Actions 持续集成

## 核心能力

- **缓存工程**：系统前缀字节稳定吃满 DeepSeek 前缀缓存；接近 1M 上下文时自动反应式压缩（增量摘要 + 旧工具结果清理 + 硬截断兜底），超大输出落盘只留指针。
- **跨会话记忆**：会话结束自动蒸馏用户偏好与项目约定，下次启动时通过对比实时代码确定性校验，淘汰无效记忆。
- **反思层**：挑战者（连续失败时质疑）、纠偏者（每 N 轮检视目标）、reply-challenger（重复提问时介入），均以 fork 子代理运行。
- **长任务支持**：会话日志与崩溃恢复、影子 git 检查点、任务清单穿越压缩、验证驱动完成（DoD）、卡死检测、并行/后台/隔离 worktree 子代理、`--goal` 自主编排模式。
- **扩展系统**：24 个工具（读写、搜索、Shell、子代理、记忆等），分层权限（allow/ask/deny），Skill 渐进披露，Hooks（5 个生命周期），自定义子代理与 slash 命令，MCP（stdio + HTTP/SSE），外挂技能自动适配工具名与模型版本。
- **终端体验**：Ink 富渲染、@ 文件引用、slash Tab 补全、steering（回合中排队输入）、Esc 一键打断、亮暗自适应、行号+语法高亮 diff。

## 适用场景

- 国内开发者在终端内进行代码阅读、修改、重构、调试。
- 需要在成本极敏感的环境下运行自主编码任务（如开源项目维护者、个人开发者）。
- 对长任务有稳健性要求的场景：批量 bug 修复、跨文件重构、多步骤集成测试。
- 希望拥有本地可控、不依赖第三方闭源服务的 AI 编码助手。

## 同类对比

- **Claude Code**：体验成熟但依赖 Anthropic 账号/网络，国内使用门槛高，模型成本显著更高。DAO 在兼容 CC 配置的同时，大幅压低了成本并解决可用性问题。
- **GLM Coding Plan**：名额有限，经常抢不到；DAO 完全开源，无限使用。
- **其他终端编码 Agent**（如 aider）：大多面向 GPT 或 Claude，DAO 专门针对 DeepSeek 的缓存和推理特性进行工程优化，成本优势明显。
