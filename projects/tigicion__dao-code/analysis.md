## 它是什么
DAO CODE（命令`dao`）是一个终端原生的AI编码助手，专为DeepSeek V4设计，MIT开源。它能在你的终端里读代码、写代码、执行shell命令、修复bug，并通过流式界面展示推理与工具调用过程。核心理念是用工程手段将高性价比模型的潜能压榨到极致，以极低的成本提供接近Claude Code的编码体验，同时解决国内用户访问Claude的网络和账号障碍。

## 为什么火
尽管Star数仅547，但它戳中了开发者的三大痛点：① Claude Code等工具国内使用困难；② GLM Coding Plan名额紧俏；③ 现有AI编码Agent使用成本高昂。DAO利用DeepSeek V4仅为主流闭源模型几十分之一的单价，并通过精巧的前缀缓存设计（实测命中率95.8%），在7个真实开源bug修复中总成本仅¥1.07，对比Claude Opus节省约30倍。它还提供了跨会话记忆、自纠偏反思、崩溃恢复等高级特性，且完全自由使用，对注重预算和可用性的极客非常有吸引力。

## 技术栈
- **语言**：TypeScript
- **运行环境**：Node.js ≥20（也提供独立二进制，无需Node）
- **终端UI**：Ink（React风格的声明式终端渲染）
- **核心逻辑**：自研agent循环，流式调用DeepSeek API，审批门权限系统，文件/命令工具集
- **扩展**：支持MCP协议、Skills（兼容Claude Code格式）、Hooks、自定义子代理
- **测试**：Vitest单元测试 + SWE-bench风格端到端评测（真实模型调用）
- **版本控制**：独立影子git检查点（不污染用户.git）

## 核心能力
1. **极致成本与缓存工程**：系统前缀、工具表、记忆均按字节稳定排列以最大化DeepSeek前缀缓存命中，实测命中率95.8%，长任务成本极低。提供`/cost`、`/audit cache`等命令实时监控。
2. **自校验记忆**：会话结束自动蒸馏关键事实，下次启动时结合当前代码确定性校验，剔除过期信息，防止“记忆幻觉”。
3. **反思与纠偏**：独立反思子代理在卡住或跑偏时触发，复用缓存分支，几乎不增加额外成本。
4. **长任务稳健性**：接近上下文上限自动压缩；todo清单穿越压缩防目标漂移；卡死检测止损；崩溃后`dao -c`恢复；影子git检查点支持`/restore`回退。
5. **扩展与兼容**：支持Claude Code风格的settings.json、SKILL.md、hooks.json、mcp.json，工具名自动映射，外来技能可直接安装。提供分层权限控制（allow/ask/deny），支持YOLO模式。
6. **人性化交互**：Ink渲染的道家美学界面，流式推理展示，`@`文件引用与Tab补全，ESC一键打断，steering排队输入，自动适应亮暗终端。

## 适用场景
- **日常编码辅助**：终端内修改代码、运行测试、分析bug，适合习惯命令行操作的开发者。
- **长任务自动推进**：通过`--goal`模式让agent自主完成复杂功能实现或大型重构，中途只需少量干预。
- **成本受限的环境**：个人开发者、小团队或CI/CD流水线中，需要大量调用AI编码但预算有限。
- **国内网络环境**：DeepSeek国内直连，无需特殊网络，注册即用，免除访问国外服务的烦恼。

## 同类对比
与Claude Code相比，DAO在功能覆盖上（工具、权限、记忆、扩展）全面对标，且成本仅为后者的1/30左右，在中国大陆可用性更佳。但其模型底座锁定DeepSeek V4，代码理解和生成能力可能弱于Claude Opus，对于极端复杂的编程任务可能力不从心。与其他开源终端编码agent（如Aider）相比，DAO更强调缓存优化和长任务稳定性，自带的记忆与反思机制是差异化优势。不足之处在于模型选择单一（仅DeepSeek V4系列），社区和插件生态刚起步，且文档以中文为主，海外推广受限。
