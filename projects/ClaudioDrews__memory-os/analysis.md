## 它是什么
Memory OS 是一套完整的本地记忆基础设施，为 Hermes Agent 提供 7 层持久化记忆体系。它不只是一个插件，而是一个操作系统的概念：从工作区文件、全文本会话搜索、结构化事实库、跨会话 fabric 回溯、混合向量检索引擎、自管理知识 Wiki，到确保记忆真正被使用的 Ground Truth 身份层，全部运行在你的机器上，支持任意 LLM 提供商。

## 为什么火
上线仅十余天即获 1171 Star，源于击中严肃 Hermes 用户的长期痛点：智能体在每一次新会话中丢失之前的推理、决策和个人偏好。Memory OS 提供了一个本地优先、无月费、不被供应商锁定的终极方案，尤其因其“Ground Truth 层级”解决了传统记忆方案“注入但不被使用”的难题，引发社区强烈共鸣。

## 技术栈
- **语言**：Python 3.11+
- **向量数据库**：Qdrant（4096维稠密+BM25稀疏混合）
- **关系型存储**：SQLite（含 FTS5 全文检索）
- **任务队列**：ARQ Worker + Redis
- **容器化**：Docker
- **LLM**：无供应商锁定，可对接 OpenRouter、OpenAI、Anthropic、Ollama 或本地模型

## 核心能力
1. **7 层记忆架构**：
   - 工作区（MEMORY.md 等）注入系统提示
   - 会话全文搜索（FTS5）
   - 结构化事实库（实体解析+信任评分反馈）
   - 跨会话 Fabric（16 种工具，多源注入）
   - Qdrant 向量库（4 级回退：混合→稠密→词汇→SQLite）
   - 自管理 LLM Wiki（持续抽取、语义链接）
   - Ground Truth 层级（SOUL.md + rulebook.md，强制智能体使用注入上下文）
2. **精准注入**：上下文甄选、会话级去重、无效消息过滤，最大化 token 效率。
3. **自动清理**：周期性衰减扫描与语义去重（相似度>0.92 合并）。
4. **一键安装**：单个 curl 命令自动部署全栈（Docker、SQLite、插件、环境）。

## 适用场景
- 长期使用 Hermes Agent 处理复杂项目、决策追踪的个人开发者。
- 重视数据隐私，要求所有记忆数据留存本地，拒绝云订阅的企业用户。
- 需要在不同 LLM 提供商间自由切换，又希望记忆一致的混合环境。
- 期望智能体逐步演进、形成稳定“人格”和知识积累的深度代理场景。
