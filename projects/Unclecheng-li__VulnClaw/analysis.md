## 它是什么
VulnClaw 是一个 AI 驱动的渗透测试 CLI 工具，用户输入自然语言目标描述（如“对 target.com 进行渗透测试”），系统自动完成信息收集、漏洞发现、漏洞利用和报告生成全流程。它基于 LLM Agent + MCP 工具链，内置了渗透 Skill 编排、13 种国内模型 Provider、Web UI 和 TUI 交互界面。

## 为什么火
Star 数快速增长至 1.5k，关键在于它直面 AI 渗透的痛点：传统的固定轮数循环在弱模型下极易空转，且容易编造虚假成果。VulnClaw 引入 `solve` 引擎，将渗透建模为从起点到目标的有向状态空间搜索（Fact + Intent），从结构上杜绝原地打转；同时用“证据级反幻觉闸门”要求结论必须出现在真实工具输出中，拒绝凭空捏造的 flag。这是当前 AI 安全自动化领域少见的务实设计。

## 技术栈
- **语言**：Python 3.10+
- **LLM 接入**：OpenAI 兼容协议，支持 MiniMax、DeepSeek、智谱、千问等 13 个 Provider
- **工具链**：基于 MCP（Model Context Protocol），内置 fetch、memory 本地服务，可对接 chrome-devtools、Burp Suite
- **引擎核心**：`solve` 引擎（黑板图 + OODA 循环）、adaptive reflexion（L0-L4 payload 绕过升级）、漏洞检测插件体系
- **前端**：CLI REPL、TUI 终端工作台、Web UI（http://127.0.0.1:7788）
- **辅助能力**：Python 代码执行、编解码工具、知识库模块

## 核心能力
- **目标驱动求解**：抛弃固定轮数，以目标达成/探索前沿耗尽/安全预算为终止条件，避免无效循环
- **证据级反幻觉**：所有 flag/结论必须在工具输出中逐字符匹配，杜绝 LLM 幻觉导致的假胜利
- **结构化推理与自适应反思**：已知事实、约束、攻击链沉淀；失败自动归类并渐进升级绕过策略
- **丰富的渗透 Skill**：21 个内置 Skill（含 CTF 专项），覆盖 web、crypto、misc 及信息收集等
- **MCP 工具生态**：4 个 MCP 服务（fetch/memory/chrome-devtools/burp），支持浏览器自动化和流量分析
- **持续性测试**：支持周期性循环渗透（默认 100 轮/周期），自动生成报告和 PoC 脚本
- **多 UI 交互**：REPL、TUI、Web 三种方式，方便不同场景使用
- **插件体系**：低耦合漏洞检测插件，可扩展并汇入报告

## 适用场景
- 已授权的渗透测试、红队演练
- CTF 竞赛中的自动化侦察和漏洞利用
- 安全教学和实验，用于演示 AI 自动化攻击链
- 内部安全团队的周期性安全巡检

## 同类对比
VulnClaw 在 README 中未直接列出竞品，但其设计明显区别于两类工具：
- **传统固定轮数的 AI 渗透脚本**：相比那些只按轮数执行、易陷入重复请求的助手，VulnClaw 的状态空间搜索从根本上避免了循环空转，且用证据门防假成果。
- **通用 LLM Agent 框架**（如 AutoGPT 等）：VulnClaw 内置了渗透测试专项的 Skill、MCP 服务和安全约束（如范围限制、禁止未授权动作），而不仅是通过 prompt 让通用 Agent 去“试试”。
其弱点目前尚不明确，安全沙箱（python_execute）标注为高危实验能力，使用者需注意隔离。
