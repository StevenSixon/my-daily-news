## 它是什么
一个 MCP 服务器，让你的 AI 编程助手（如 Claude、Copilot、Cursor）可以直接操控和检查实时 Chrome 浏览器。它把 Chrome DevTools 的全部能力（性能分析、网络监控、控制台检查、截图等）暴露给编码代理，实现可靠的自动化、深度调试和性能分析。提供 CLI 模式，也可脱离 MCP 使用。

## 为什么火
- **痛点直击**：AI 编码代理在生成代码后很难现场验证，尤其涉及前端交互、性能优化和网络请求时。该项目弥补了“写代码”与“看结果”之间的鸿沟。
- **官方血统**：由 Chrome DevTools 团队维护，基于 Puppeteer 和 DevTools 前端，质量与兼容性有保障。
- **生态整合**：深度适配 Claude Code、VS Code、Cursor、Gemini CLI 等主流工具，插件市场一键安装。
- **轻量与全量可选**：提供 `--slim` 模式只做基本浏览器任务，适合快速上手。

## 技术栈
- 核心语言：TypeScript
- 浏览器控制：Puppeteer（自动化 Chrome 操作）
- 开发工具集成：直接调用 Chrome DevTools 前端（性能追踪、源码映射堆栈等）
- 协议：MCP（模型上下文协议），用于与 AI 客户端通信
- 性能数据：可选 CrUX API 获取真实用户性能数据

## 核心能力
- **性能深度洞察**：记录 Trace，提取可操作的性能建议，结合实验室数据和现场数据。
- **高级调试**：分析网络请求、截屏、检查浏览器控制台消息（带源码映射的堆栈跟踪）。
- **可靠自动化**：基于 Puppeteer 的自动等待和操作，降低 AI 代理直接操作浏览器的不稳定性。
- **灵活部署**：支持仅 MCP 服务器、CLI 独立使用，以及作为 Claude/VSCode 插件（含技能提示）。

## 适用场景
- **前端开发与调试**：AI 助手自动发现布局问题、报错原因，并现场修正。
- **端到端测试**：Agent 自动执行操作，抓取页面状态，断言网络请求与性能指标。
- **性能审计**：自动运行 Lighthouse 级分析，定位拥堵点。
- **数据抓取与监控**：构建需要真实浏览器环境的爬虫或监控脚本。

## 同类对比
- vs. Playwright MCP： Playwright 更偏跨浏览器自动化测试，DevTools 集成不如本项目原生；chrome-devtools-mcp 直接使用 DevTools 前端，性能分析能力更强，但仅支持 Chrome。
- vs. Puppeteer 直接调用：本项目已封装 MCP 协议，AI 代理开箱即用，无需自行构造工具链。
- vs. Selenium/Browserless：这些更偏向无头自动化，而本项目强项在于与 AI 助手无缝连接，并提供丰富的调试信息格式。
