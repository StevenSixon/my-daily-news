## 它是什么
lift 是一个从 PDF 和图片中提取结构化数据的 Python 工具，核心是一个 9B 参数的视觉语言模型。你只需传入一个 JSON Schema，它便返回严格符合该 Schema 的 JSON，并通过 Schema 约束解码保证输出永远有效。支持多页文档单次处理，能够提取跨页字段、列表和嵌套对象。

## 为什么火
开源模型在文档字段级提取上长期落后于闭源 API，lift 以 90.2% 的字段准确率（同类开源最高）和 9.5s 的中位延迟吸引了大量关注。它允许开发者本地部署，避免了数据外泄和 API 成本，同时提供了比 NuExtract3、Qwen3.5 等模型更高的单字段精度。

## 技术栈
- 推理引擎：vLLM（推荐）或 HuggingFace Transformers
- 基础框架：基于 Qwen3.5 系列视觉模型微调（感谢 Qwen 3.5）
- 语言：Python，提供 CLI 和 Python API
- 前端工具：Schema Studio（Streamlit 应用）用于交互式构建和测试 Schema
- 部署：Docker 一键启动 vLLM 服务，针对多种 GPU 自动优化

## 核心能力
- **Schema 约束解码**：输出永远符合给定的 JSON Schema，避免格式错误
- **高准确率**：在 225 份文档、约 11000 个字段的基准测试中，字段正确率 90.2%，远超 NuExtract3（81.5%）和 Qwen3.5-9B（76.3%）
- **多页支持**：一次处理多页 PDF，自动聚合跨页值
- **灵活性**：支持任意 JSON Schema（字符串、数字、布尔、数组、嵌套对象）
- **两种推理模式**：本地 HF 模型（需 GPU）或通过 vLLM 服务栈（高性能）
- **批量处理**：CLI 可处理整个目录，适合大规模文档解析

## 适用场景
- 发票、合同、表单、报告等文档的自动化信息提取
- 金融、物流、法律、医疗等需要从非结构化文档中获取结构化数据的领域
- 注重数据隐私、需要本地部署的中小团队或初创公司（<$5M 资金/收入可免费商用）
- 任何希望以较低延迟（9.5s/doc）获得高字段级提取精度的场景

## 同类对比
与基准中的主要模型对比：
| 模型 | 字段准确率 | 全文档准确率 | 延迟 | 备注 |
|---|---|---|---|---|
| **lift** | 90.2% | 20.9% | 9.5s | 开源，需商业许可 |
| Datalab API | 95.9% | 44.4% | 30.8s | 闭源，含验证与引用 |
| Gemini Flash 3.5 | 91.3% | 40.0% | 28.1s | 闭源 API |
| Qwen3.5-9B | 76.3% | 24.0% | 16.8s | 开源，通用模型 |
| NuExtract3-4B | 81.5% | 8.4% | 8.3s | 开源，轻量但全文档精度低 |
lift 在字段级准确率上领先所有开源模型，且速度优于同等规模模型。全文档准确率（所有字段完全正确）较低，说明在复杂文档的完整提取上仍有提升空间，但相比 NuExtract3 已是巨大进步。与闭源 API 的差距主要在于缺少验证和引用机制。
