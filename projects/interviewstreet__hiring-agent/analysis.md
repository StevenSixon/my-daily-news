## 它是什么
一个基于LLM的Resume-to-Score管道，输入PDF简历，输出结构化评价（分项得分、证据、加分/减分项）。核心价值是**用大模型理解简历语义而非简单关键词**，并结合GitHub公开活动（项目参与、技术栈）客观评估候选人。

## 为什么火
- 招聘初筛长期依赖人工或规则匹配，主观且耗时；该项目用LLM提供**有解释的量化评分**，能降低面试官偏见。
- Star数短时间增长至2.7k，说明技术招聘领域对“AI辅助评估”需求旺盛。
- 支持完全本地运行（Ollama），保护简历隐私；同时可选Gemini云端模型，灵活性高。

## 技术栈
- **语言**: Python 3.11+
- **PDF解析**: PyMuPDF (pymupdf4llm风格) 转Markdown
- **LLM后端**: Ollama (本地Gemma/其他)、Google Gemini
- **模板系统**: Jinja2，定义各解析环节的Prompt
- **数据模型**: Pydantic (JSONResume风格)
- **GitHub集成**: PyGithub / GitHub API v3，获取profile、repos并做项目分类与选择
- **输出**: 开发模式缓存中间JSON、附加CSV评估记录

## 核心能力
1. **PDF → Markdown**：保留标题、链接、表格。
2. **分节LLM提取**：基础信息、工作经历、教育、技能、项目、奖项，每个部分用专用模板严格约束输出格式。
3. **GitHub富化**：自动识别简历中的GitHub账号，获取repos，LLM挑选前7个代表项目，排除低贡献仓库。
4. **公平评分引擎**：评估维度含开源贡献、个人项目、生产级经验、技术技能，并按规则加减分；生成解释性证据。
5. **离线/在线双模式**：无API Key可用Ollama本地模型；有Key可切换到Gemini获得更高精度。

## 适用场景
- 技术招聘初筛，尤其是需要评估开源贡献的岗位。
- 简历分析工具原型，可集成到ATS或人才库系统。
- 开发者想自托管招聘辅助代理，保护候选人隐私。

## 同类对比
- vs 传统ATS（如Greenhouse、Lever）：规则匹配为主，缺少语义理解和GitHub信号。
- vs 纯LLM prompt评价（如ChatGPT简历打分）：该项目提供**结构化流水线**，分区提取、校验、缓存、可复现，且支持本地部署。
- vs 其他AI招聘产品（如HireVue、Pymetrics）：该项目**开源、透明、可定制**，但缺少视频分析等模态。
