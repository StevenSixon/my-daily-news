## 它是什么

Hiring Agent 是由 HackerRank（通过 interviewstreet 仓库）开源的一条**简历到评分流水线**。它从 PDF 简历中提取结构化数据，可选地引入候选人的 GitHub 活动信号，最终输出附带解释的公平评估结果。流水线支持**完全本地运行**（通过 Ollama）或使用 Google Gemini 云端模型，核心是在保护隐私的前提下自动化技术简历筛选。

## 为什么火

- 招聘市场一直需要自动化筛选工具，而结合实际代码贡献（GitHub）的评估方案更贴近技术岗位的真实需求。
- 本地优先设计避免了将敏感简历数据上传第三方服务，受到注重合规与隐私团队的青睐。
- 采用模板化提示词（Jinja），评分标准透明、可定制，适合研究或二次开发。
- 以 MIT 许可证开放，降低了企业试用的门槛。

## 技术栈

- **Python 3.11+**，使用 PyMuPDF 将 PDF 转为 Markdown
- **LLM 后端**：Ollama（支持 gemma3:4b/12b/1b 等）或 Google Gemini（如 gemini-2.0-flash）
- **Pydantic** 数据模型，Jinja2 模板管理提示词
- **GitHub API** 抓取用户 profile 和仓库数据
- **本地缓存**（JSON 文件）与 CSV 导出（开发模式）

## 核心能力

1. **PDF 提取**：通过 PyMuPDF 将简历转为 Markdown 文本，保留标题、链接、表格等结构。
2. **分节解析**：将简历拆分为基本信息、工作经历、教育、技能、项目、奖项等节，分别调用 LLM 提取为 JSON Resume 结构化数据。
3. **GitHub 增强**：从简历中识别 GitHub 用户名，抓取个人资料与仓库，由 LLM 精选 7 个最具代表性的项目（倾向有真实贡献的仓库）。
4. **量化评分**：基于开源贡献、个人项目、生产级项目、技术能力四个维度打分，并设有加分与扣分机制，输出证据和解释。
5. **开发模式辅助**：可缓存中间结果、追加 CSV 记录，便于迭代和调试。

## 适用场景

- 技术招聘初筛：快速过滤无法体现实际工程能力的简历。
- 内部人才盘点：结合代码资产量化评估工程师。
- LLM 招聘工具研究：作为模板化评分流水线的参考实现。
- 隐私敏感环境：全本地运行，无需上传简历至任何外部服务。

## 同类对比

- **传统简历解析 API**（如 Affinda、Sovren）：仅提取文本结构化，不包含代码贡献评估，且多为云的付费服务。
- **AI 简历排名工具**（如一些 ATS 内置评分）：通常是黑箱云端，无公开评分逻辑，且不强制整合 GitHub。
- Hiring Agent 的优势：开源、本地可运行、评分逻辑透明可调、支持自托管、将代码贡献作为定量信号。
