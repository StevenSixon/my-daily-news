## 它是什么
PixelRAG 是一个以**截图而非文本**进行文档检索与生成增强（RAG）的框架。它把网页、PDF、图片等渲染成图像瓦片，用微调后的 Qwen3-VL-Embedding 模型生成视觉嵌入，再通过 FAISS 索引实现“所见即所得”的搜索。项目提供了预建的 828 万维基百科页面索引和即时可用 API，同时支持用户自建索引。

## 为什么火
- **保留视觉信息**：传统 RAG 将 HTML 解析为纯文本，会丢失表格、图表、排版等结构；PixelRAG 直接使用页面截图，让检索与答案生成能利用完整视觉上下文。
- **零门槛体验**：官方提供托管 API（无需 API 密钥）和预建索引，一行 curl 即可搜索维基百科；本地使用亦只需 pip install。
- **生态延伸**：附带 Claude Code 插件（pixelbrowse），让 LLM 通过截图“阅读”网页，无须额外服务；训练代码与数据集公开，便于复现和拓展。

## 技术栈
- 渲染引擎：基于 Playwright/CDP 的 `pixelshot`，支持网页、PDF（需 poppler）以及本地图片。
- 嵌入模型：`Qwen3-VL-Embedding-2B`，LoRA 微调于网页截图数据（权重已发布）。
- 索引与搜索：FAISS（支持 CPU/GPU），内置分片构建与服务。
- 服务层：FastAPI 提供 REST API；配置通过 YAML 驱动。
- 训练工具链：独立 uv 项目，使用 PyTorch 2.9.1 + CUDA 12.9，LoRA 微调流程完整文档化。

## 核心能力
- **截图级文档解析**：`pixelshot` 一行命令即可将任意网页或 PDF 转为图像瓦片。
- **视觉语义搜索**：直接以图像 tile 为检索单元，查询可以是文本或图像（视觉搜索）。
- **预建大规模索引**：828 万维基百科页面索引，可从 Hugging Face 下载或直接使用官方 API。
- **端到端自建流水线**：`pixelrag index build` 一条命令从原始文档到可查询索引。
- **生态集成**：Claude Code 插件让 AI 助手获得“阅读”网页原貌的能力。

## 适用场景
- **复杂网页信息抽取**：需要理解表格、图表、排版布局的问答或分析任务。
- **RAG 系统升级**：为现有 LLM 应用增加视觉感知，避免因解析丢失导致答案错误。
- **文档数字化与搜索**：对 PDF 报告、扫描件等非纯粹文本内容建立视觉索引。
- **AI Agent 增强**：为浏览器自动化、研究辅助等 Agent 提供更准确的页面理解。

## 同类对比
与传统文本 RAG（如 LangChain 文本拆分+向量检索）相比，PixelRAG 的独特优势在于保留视觉结构，回答“表格里某个数字是多少”“图表 X 轴代表什么”之类的问题时不再失效。相较于其他视觉嵌入方案（如 ColPali 使用 ViT 编码器），PixelRAG 采用了完整的多模态 VLM（Qwen3-VL）并针对网页截图微调，更贴合真实网页内容。项目还提供从渲染到索引的全套工程化工具，比纯学术框架更易落地。
