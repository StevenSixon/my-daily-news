## 它是什么
PixelRAG 是一种基于视觉的检索增强生成框架——它不解析 HTML/文本，而是将文档（网页、PDF、图片）渲染成一系列截图块，用视觉嵌入模型编码后存入向量索引，检索时返回最相关的截图块，最终让视觉语言模型（VLM）直接“看”图回答。由 Berkeley SkyLab、BAIR 和 Berkeley NLP 团队开发，对应论文已发布。

## 为什么火
- **痛点直击**：传统 RAG 的 HTML 解析或文本提取会丢失表格、图表、复杂排版信息，PixelRAG 则完整保留原始视觉布局，特别适合需要理解图表、财务报表、论文架构的姿态。
- **开箱即用**：官方托管了 8.28M 维基百科页面的预建索引，无需搭建即可通过 API 进行视觉搜索，极大降低体验门槛。
- **Claude Code 插件**：作为 pixelbrowse 技能直接赋予 Claude “阅读网页截图”的能力，开发者社区反响热烈。
- **完全开源可自托管**：Apache-2.0 许可，支持本地构建和部署，满足隐私与定制需求。

## 技术栈
- **渲染**：Playwright / CDP 驱动的 Chromium 无头浏览器渲染网页；PDF 支持通过 poppler 转为截图。
- **嵌入模型**：Qwen3-VL-Embedding-2B，用 LoRA 在网页截图数据上微调，实现对视觉内容的语义检索；支持 CUDA 与 Apple Silicon MPS。
- **索引与搜索**：FAISS 向量索引，提供 FastAPI 搜索服务。
- **训练**：独立的 uv 项目，使用固定版本 torch/transformers，数据与适配器权重已公开。
- **跨平台**：Linux（CUDA）、macOS（MPS）、Windows（系统 Chrome）均提供基础支持。

## 核心能力
1. **pixelshot 渲染器**：一行命令将 URL/PDF/图片转为统一大小的截图块，支持分页、宽页2D分块。
2. **完整索引流水线**：从源文档到可用索引全流程可编排（chunk → embed → build-index），并支持分阶段单独操作。
3. **多模态检索**：支持文本查询和图像查询，返回最相关的截图块（及来源信息）。
4. **预置大规模索引**：8.28M 维基页面索引已在 API 和 Hugging Face 提供，可直接搜索或下载部署。
5. **Claude Code 集成**：无需 MCP 服务器，本地 pixelshot 即可为 Claude 提供页面截图浏览能力。
6. **自托管服务**：支持从任意文档构建私有索引并对外暴露兼容的搜索 API。

## 适用场景
- 学术文献检索（保留图表、公式布局）
- 金融/法律文档问答（表格与格式敏感）
- 为 LLM/Agent 提供“视觉浏览”能力（替代易出错的文本抓取）
- 构建可理解布局的企业知识库
- 视觉驱动的网页内容监控与变化检测

## 同类对比
- **vs 文本 RAG 框架**（LangChain/LlamaIndex/Haystack）：传统框架依赖文本分块，丢失视觉结构；PixelRAG 直接操作截图，天然适合多版式文档。
- **vs 多模态 RAG 方案**（如 ColPali）：PixelRAG 提供了完整的渲染→嵌入→检索→服务工具链，集成度高，且已发布经大规模预训练的 LoRA 嵌入模型和数据集，上手更快。
- **vs 商业闭源方案**：完全开源，可本地部署，数据不出域；易于定制训练其他视觉嵌入骨干。
