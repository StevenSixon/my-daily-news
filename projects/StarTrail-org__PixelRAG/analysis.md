## 它是什么
PixelRAG 是一个将网页、PDF、图片等文档渲染为截图，并直接在图像上进行向量检索与生成回答的系统。它保留了传统文本解析会丢弃的表格、图表、布局等视觉结构。

## 为什么火
2026 年 5 月发布后迅速获得 1.7k+ 星，原因在于它用截图替代了文本分块这一 RAG 的核心范式，提供了 828 万维基百科页面的预建索引和开箱即用的公共搜索 API，并内置 Claude Code 插件让 AI 直接“看图”。

## 技术栈
- 渲染：Playwright CDP + 定制 headless shell
- 嵌入：基于 Qwen3‑VL‑Embedding‑2B 的 LoRA 微调模型，专用于网页截图检索
- 索引与搜索：FAISS（支持 CPU/GPU）+ FastAPI
- 管道：Python CLI（pixelrag） + 独立训练环境（uv + PyTorch）
- 分布：PyPI 包 `pixelrag`，预训练权重与数据集托管在 HuggingFace

## 核心能力
- **pixelshot**：将任意 URL 或文档渲染为截图瓦片
- **视觉搜索**：通过文本或图片查询 FAISS 索引，返回最相关的页面截图
- **Claude Code 插件**：`pixelbrowse` 让 Claude 直接截图并阅读页面，无需解析 HTML
- **全流程自建**：支持从本地文档源 → 切片 → 嵌入 → 索引 → 服务的完整构建
- **训练可复现**：开放 LoRA 训练适配器和训练集，可基于其他视觉骨干模型微调

## 适用场景
- 需要理解图表、表格、布局的多模态 RAG 应用
- 面向网页视觉信息的 AI Agent 或问答系统
- 以“页面长相”为线索的语义搜索系统

## 同类对比
- 相比 **文本块 RAG**（如 LangChain + 分块）：文本 RAG 丢失表格等可视化结构；PixelRAG 直接保留完整视觉信息
- 相比 **多模态检索方案**（如 ColPali）：PixelRAG 更强调开箱即用的网页渲染管道和庞大的预建索引，并提供 Claude Code 集成
- **独特优势**：免费的 828 万维基百科搜索 API，极低上手门槛
