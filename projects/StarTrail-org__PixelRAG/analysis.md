## 它是什么
PixelRAG 将文档（网页、PDF、图片）渲染为截图，直接在图像嵌入上做检索增强生成，不打散成文字。表格、图表、排版等视觉结构被完整保留。

## 为什么火
传统 RAG 把网页解析为文本，会丢失表格与复杂布局，导致无法回答依赖图表的问题。PixelRAG 用截图绕过解析，直接对视觉内容建立索引并检索。开源即提供 828 万 Wikipedia 页面的在线搜索 API 和可下载的 FAISS 索引，零门槛体验。

## 技术栈
Python 生态：Playwright/CDP 截图，Qwen3-VL-Embedding-2B 模型（LoRA 微调）生成视觉向量，FAISS 构建索引，FastAPI 提供搜索服务。渲染和索引分别作为独立命令 `pixelshot` 与 `pixelrag`。

## 核心能力
- 通用文档渲染：网页、PDF、本地图片→标准截图瓦片
- 视觉索引管道：截图分组、嵌入、构建 FAISS 索引
- 多模态检索：支持文本或图像查询，返回相关截图瓦片
- Claude Code 插件：让 Claude 直接「看」网页截图，无需解析 HTML
- 预建大规模索引：Wikipedia 8.28M 页面索引可下载或直接 API 调用
- 本地自建索引：支持 Linux (CUDA) 与 macOS (Apple Silicon)，自动选设备

## 适用场景
- 需要理解表格、图表、信息图的知识型问答
- 对 HTML 解析不友好的网站内容归档与检索
- 多模态 Agent 的视觉“眼睛”（如 Claude 插件场景）
- 企业内部文档（PDF、网页抓取）的视觉搜索

## 同类对比
相对于 LlamaIndex、Unstructured 等纯文本 RAG 管道，保留布局信息，能直接回答“图中第几行”之类的问题。相比用通用 VLM 做全图检索，PixelRAG 使用专门微调的视觉嵌入模型，效率更高且可做大规模索引。
