## 它是什么
一个面向 GPT-image-2 的 AI 图像生成与编辑的本地 WebUI 工作台，同时提供 CLI 支持自动化。内置图库管理、提示词模板、多种 chip 快捷引用、多任务并发队列、分页历史库，并支持 Codex Image 通道、Codex Responses 兼容通道以及 OpenAI 兼容 API 三种后端接入。

## 为什么火
GitHub 近期才发布 v0.5.1 却已有 539 star，主要因为 GPT-image-2 的用户需要一个易用的本地 GUI，而现有方案要么是在线 Colab，要么是通用图像生成界面，缺少对 Codex 通道、批量并发、提示词模板、图库 chip 等深度整合。该项目采用免安装一键包分发，降低了使用门槛。

## 技术栈
后端：Python 3.11+、FastAPI、Uvicorn，数据用 SQLite 存储任务历史和元数据。前端：TypeScript 静态资源，运行时资源版本化（runtime-346）。跨平台打包：Windows x64、macOS Apple Silicon 和 Intel 三种便携包，内嵌 CPython。

## 核心能力
- 多后端接入：默认 Codex Image 通道、Codex Responses 兼容通道、OpenAI 兼容 API 三种模式，供应商卡片式管理、显式编辑和二次确认删除。
- 参考图与图库：支持上传、拖拽、粘贴、公用图库复用；公用图库可分类、命名、引用备注，并可通过 `@` 在提示词中快速插入。
- 三种快捷 chip：`@` 图库 chip、`#` 颜色 chip、`~` 提示词片段 chip，支持搜索、展开、编辑，方便组合复杂提示。
- 提示词模板：完整的保存、分类、插入/替换、历史缩略图关联，比简单片段更结构化。
- 任务与历史：多任务并发，本地队列状态，分页历史库，网格/列表视图，搜索、筛选、懒加载详情，支持失败重试。
- 系统设置整合：统一窗口管理 API 设置、Codex 通道、存储与通知，存储目录变更需重启确认。
- 免安装一键包与手动更新：启动脚本纯本地化，更新器校验 SHA256、保留 data/ 并备份替换文件，降低安全误报风险。

## 适用场景
- 个人创作者需要稳定、解压即用的本地 AI 图像生成工具，且经常用参考图和颜色约束。
- 需要管理大量提示词片段和模板的生成工作流，如角色设定图、产品图批量生成。
- 希望避免在线服务的网络延迟和不稳定性，又不想折腾 Python 环境配置的用户。

## 同类对比
与 ComfyUI 或 Stable Diffusion WebUI 相比，本工具专门面向 GPT-2 图像模型，更聚焦于 API 调用式的生成与编辑，而非本地模型推理；与普通 OpenAI 聊天/图像客户端相比，它提供了完善的图库 chip、模板管理和多供应商并发队列，更工程化。不足之处在于强依赖 GPT-image-2 的 API，不支持加载本地 Stable Diffusion 模型。
