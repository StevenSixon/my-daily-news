## 它是什么

iLab GPT Conjure 是一个专为 GPT‑image‑2 模型打造的 AI 图像生成 WebUI 工作台。它提供类似 A1111/ComfyUI 的本地操作界面，但完全围绕 OpenAI 的 Codex Image / Responses 通道设计，同时兼容 OpenAI‑compatible API 模式。内置公用图库、三种原子 Chip（图片引用、颜色、提示词片段）、提示词模板、多任务并发队列和历史管理。

## 为什么火

- **填补空白**：GPT‑image‑2 没有官方桌面 GUI，该项目为个人和团队提供了标准化的生图、参考图、编辑工作流。
- **本地资产管理**：公用图库、Chip 引用和模板解决了多人共享固定角色、产品主图等长期素材的需求，比在聊天界面反复粘贴高效得多。
- **双模式接入**：既支持利用本地 OAuth 直接调用 ChatGPT 后端（高级模式），也支持标准的 OpenAI API 密钥接入，适合从个人实验到团队部署的不同场景。
- **开箱即用的安装包**：提供 macOS DMG、Windows App ZIP 和免安装 portable 包，启动器内置 Rust 托盘，自动打开浏览器，对非技术用户友好。

## 技术栈

- **后端**：Python 3.11+ / FastAPI，提供 WebUI 和 CLI，SQLite 存储任务历史。
- **前端**：TypeScript + 原生 Web Components / Konva（图像编辑器），预构建静态资源。
- **分发**：PyInstaller 打包的 portable 包，macOS 标准 .app 和 DMG（未签名），Windows App ZIP；Rust 编写的托盘启动器。
- **自动更新**：portable 包使用 Ed25519 签名 + SHA256 校验的更新器。

## 核心能力

- **文生图、参考图与编辑**：支持上传/拖拽参考图，通过图像编辑器进行图层组合、擦除、等比缩放。
- **智能 Chip 系统**：`@` 搜索图库插入参考图，`#` 插入十六进制颜色，`~` 插入常用提示词片段，提交时自动展开。
- **提示词模板**：保存和重用结构化生成框架，支持分类、筛选、导入导出。
- **并发与队列**：可配置并发上限，任务在本地排队，实时显示状态，支持失败重试。
- **历史与搜索**：独立 `/history` 页面，支持 SQLite 分页、按提示词/任务ID搜索、网格/列表视图。
- **多语言**：WebUI 支持 13 种语言，自动跟随浏览器语言或手动选择。
- **认证模式**：API 供应商卡片管理，可配置多组 Base URL / API Key / 模型，支持 Codex Image 和 Responses 通道切换。

## 适用场景

- 个人创作者需要稳定的 GPT‑image‑2 生图流程，希望保存和管理角色设定、风格参考。
- 设计师或团队需要共享公用图库和提示词模板，批量生成产品图片或素材。
- 本地自动化批处理，通过 CLI 集成到其他工具链。
- 对 OpenAI API 有访问权限的开发者，希望用一个轻量本地 GUI 替代在 playground 中手动调用。

## 同类对比

- **vs. ComfyUI / A1111**：那些主要面向 Stable Diffusion 生态，对 GPT‑image‑2 没有原生支持。iLab GPT Conjure 直接使用 OpenAI 接口，无需本地 GPU，但依赖网络和 API 费用。
- **vs. 直接在 ChatGPT 中生成**：缺少图库管理、批量队列、历史搜索和团队共享能力。
- **vs. 其他 GPT‑4o 图像工具**：多数是简易封装或演示，缺乏 Chip 引用、模板和本地数据持久化。

## 版本动态
