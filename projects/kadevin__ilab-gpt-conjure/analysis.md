## 它是什么
iLab GPT Conjure 是一个面向 OpenAI GPT-image-2 模型的本地 WebUI 工作台，提供完整的图片生成、编辑与管理流程。它通过浏览器界面包装了 Codex Image、Codex Responses 和通用 OpenAI 兼容 API 三种后端接入方式，并内置了图库、颜色/片段 Chip 快速引用、提示词模板系统、多任务并发调度和历史库搜索等实用功能。

## 为什么火
- **多 API 源统一接入**：默认支持 Codex 通道和个人 OAuth，同时推荐稳定环境使用 OpenAI-compatible API，避免单一服务中断。
- **提示词工程友好**：Chip 机制允许用 `@` 引用图库、`#` 插入颜色、`~` 调用提示词片段，结合模板管理可大幅提升复杂生成任务的可复用性。
- **生产级任务体验**：多任务并发、本地队列、分页历史、失败重试、单任务多图输出等细节，明显优于简单的 API 调用脚本。
- **零门槛部署**：提供 Windows / macOS 免安装一键包，包含打包好的 Python、依赖和前端静态资源，类似 ComfyUI 的“解压即用”哲学。
- **575+ Star 且持续迭代**：社区积极，v0.5.4 在任务栏、历史库等体验细节上做了大量优化，说明项目正走向成熟。

## 技术栈
- 后端：Python 3.11 + FastAPI，提供 WebUI 和 CLI，SQLite 存储任务数据库
- 前端：TypeScript + 浏览器端构建（打包后作为静态资源），包含图层编辑器（基于 Konva）
- 打包：GitHub CI 自动构建便携包，内含 CPython、依赖，启动脚本自动处理 macOS 隔离标
- 认证：支持 API Key、高级 OAuth（复用 ChatGPT 桌面端登录态）

## 核心能力
1. **文生图 / 图生图 / 图像编辑**：基础生成外，支持参考图、mask 编辑和局部擦除。
2. **多供应商切换**：Codex Image、Codex Responses、OpenAI 兼容 API 三种通道，可在系统设置中统一管理 API 卡片、Base URL、Key、模型名和并发上限。
3. **公用图库与 Chip 引用**：本地持久化收藏常用素材，通过 `@` 快捷插入提示词并自动加载参考图；颜色 Chip 和片段 Chip 进一步解耦提示词结构。
4. **提示词模板系统**：支持分类、收藏、导入导出，可从历史结果选择缩略图，适合团队或长期项目。
5. **任务管理**：多任务并发队列、进度计时、失败重试、部分失败处理；独立历史页面支持搜索（按提示词/任务ID）、筛选、网格/列表视图、懒加载详情。
6. **离线与隐私**：所有数据（设置、图库、任务库）保存在本地 `data/` 目录，无云端依赖。

## 适用场景
- 个人极客：想在本地用 GUI 管理 GPT-image-2 的多次尝试，并积累自己的提示词和图库资产。
- 小型团队：通过 OpenAI 兼容 API 接入统一接口，共享模板与图库（需各自本地维护或通过文件共享）。
- 快速原型：需要反复调整提示词、参考图的创意工作，图库和模板可大幅加速迭代。

## 同类对比
- **vs ComfyUI / Stable Diffusion WebUI**：后者是模型本地推理工作台，而本项目是 GPT-image-2 这类闭源 API 的“遥控器”，不涉及本地模型运行，重在流程编排和资产管理。
- **vs 各平台自带 Playground**：OpenAI 官方 Playground 功能有限，本项目补充了图库、模板、历史管理和多任务并发，更适合连续生产。
- **vs 简单 Python 脚本**：CLI 模式已经比裸 API 方便，WebUI 更是把状态同步、失败处理和 UI 交互做到足够顺手。
