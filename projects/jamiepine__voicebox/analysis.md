## 它是什么
Voicebox 是一个**本地优先**的开源AI语音工作室，把语音克隆、多引擎文语转换（TTS）、全局听写（dictation）、语音转文字（STT）、以及为MCP智能体提供语音输出整合在一个桌面应用里。它试图在本地机器上覆盖 ElevenLabs（语音输出）与 WisprFlow（语音输入）两个云服务的能力，并通过内置的本地LLM实现转录精炼和语音人格化。

## 为什么火
- **填补“本地整循环”空白**：很少有工具同时做到高质量的零样本声音克隆、跨平台全局热键听写、以及让AI代理用克隆声音回应。
- **隐私友好**：模型、音频数据永不出机，适合重视数据主权的个人开发者、企业或需要离线场景的用户。
- **多引擎灵活性**：内含7个TTS引擎（Qwen3-TTS、Chatterbox、HumeAI、Kokoro等），覆盖23种语言，从快速CPU模型到高拟真度1000M+参数模型都支持，用户可按需切换。
- **与Coding Agent生态融合**：通过MCP协议，Claude Code、Cursor等可直接调用`voicebox.speak`用克隆好的声音“说话”，实现真正的人机语音对话。

## 技术栈
- **桌面端**：Tauri (Rust) + 前端 (TypeScript)，非Electron，轻量高性能。
- **推理后端**：macOS 使用 MLX（Metal加速），Windows/Linux 使用 CUDA/ROCm/DirectML，也支持 Docker CPU 推理。
- **TTS引擎集成**：Qwen3-TTS (0.6B/1.7B)、Chatterbox、LuxTTS、HumeAI TADA (1B/3B)、Kokoro (82M) 等，均为本地运行模型。
- **STT**：OpenAI Whisper，通过MLX或PyTorch后端运行。
- **效果处理**：基于 Spotify `pedalboard` 的音频后处理链（混响、延迟、压缩等）。
- **LLM**：内建本地LLM用于录音文字精炼和按角色人格改写/回答。
- **APIs**：REST API + MCP server，方便被外部应用或智能体调用。

## 核心能力
1. **多引擎语音克隆与生成**  
   - 零样本克隆：从几秒参考音频克隆声音。  
   - 7引擎自由切换：Qwen3-TTS 支持自然语言交付指令（“说慢一点”、“低声说”），Chatterbox Turbo 支持拟声标签（`[laugh]`等），Kokoro提供50+预置音色，LuxTTS 仅需~1GB VRAM。  
   - 不限长度生成：自动分句、交叉淡化拼接，支持最多50,000字符输入。
2. **全局听写**  
   - 自定义热键（macOS默认右Cmd+右Option，Windows默认右Ctrl+右Shift），支持按住说话松开粘贴，也可双击Space切换到免提对话模式。  
   - macOS 下通过Accessibility API精准粘贴到聚焦文本框，剪辑板自动保存恢复。  
   - 可选的本地LLM精炼：去掉嗯啊、重复、口误。
3. **代理语音输出**  
   - 一个工具调用 `voicebox.speak`，任何支持MCP的AI代理（Claude Code、Cursor等）即可通过浮动气泡用克隆语音回应。  
   - 可给声音配置“人格描述”，代理调用时经过本地LLM根据人格改写要说的话再朗读。
4. **语音人格**  
   - 每份声音档案可附加一段自由文本人格描述，生成或代理发言时可选择“撰写/改写/回应”模式，让输出文本贴合该角色性格。
5. **音频后期与故事编辑器**  
   - 8种效果（移调、混响、延迟、压缩等），4个内置预设，可建自定义预设并设为默认。  
   - 多轨“故事编辑器”：拖拽多个声音片段、内联剪辑、同步播放，适合制作对话、播客。
6. **API与集成**  
   - 提供REST API和内置MCP服务器，开发者可将语音I/O能力接入自己的应用，也可与MCP生态的工具配合。

## 适用场景
- **隐私敏感的语音交互**：个人日常笔记、会议记录转录后自动粘贴，不经过云服务。
- **辅助开发者的便捷输入**：写代码时用全局热键口述文字，自动输入到编辑器、终端或聊天框。
- **构建本地智能助手**：配合Code Agent实现“说出问题-代理思考-用声音回答”的闭环。
- **内容创作**：快速生成多语言语音旁白、播客片段，借助故事编辑器合成对话。

## 同类对比
- **vs ElevenLabs**：ElevenLabs 云服务，需联网，按字符计费；Voicebox 完全本地、免费无限量，支持更多引擎和语音人格化，但不具备 ElevenLabs 的开放音效库和商业声音市场。
- **vs WisprFlow**：WisprFlow 专注实时语音输入，以云 API 形式提供；Voicebox 则把听写与 TTS 整合，本地LLM精炼，无延迟上传问题。
- **其他开源TTS工具（如XTTS、Coqui.ai）**：Voicebox 集成了多引擎，并补齐了听写与代理输出，构成端到端工作室而非单点模型；其桌面应用体验和全局热键是差异化优势。
