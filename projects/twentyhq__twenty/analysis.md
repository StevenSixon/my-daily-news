## 它是什么
Twenty 是一个开源的CRM平台，定位为Salesforce的替代品，专为技术团队设计。它允许开发者通过代码定义业务对象、视图、工作流和AI代理，并像管理应用代码一样对CRM配置进行版本控制、构建和发布。

## 为什么火
- **开发体验优先**：用TypeScript定义领域模型，取代低效的点击配置，支持Monorepo、Nx构建，贴合现代技术栈。
- **AI原生集成**：内置AI代理和聊天界面，直接与CRM数据交互，适应智能化销售流程。
- **开源与自托管**：避免供应商锁定，数据自主可控，已有5万+ Star，社区活跃。
- **版本化CRM**：提供CLI工具和发布流水线，让业务配置变更可追溯、可回滚，解决传统CRM「配置漂移」痛点。

## 技术栈
- 前端：React 19、TypeScript、Jotai 状态管理、Linaria CSS-in-JS、Lingui 国际化。
- 后端：NestJS、GraphQL API、BullMQ 消息队列、PostgreSQL、Redis。
- 工程化：Nx Monorepo 管理多包，依赖 TypeScript 全栈。
- 部署：支持 Docker Compose 自托管，也提供官方云服务。

## 核心能力
- **代码定义对象**：使用 `defineObject` 描述实体、字段、关系，可版本控制。
- **视图与布局**：灵活配置列表、看板、图表等视图，前端通过Jotai驱动响应式布局。
- **AI 代理**：聊天式AI交互，能基于CRM数据执行任务（具体实现待探索）。
- **应用构建器**：通过CLI创建、发布独立应用，支持私有发布。
- **工作流自动化**：基于事件的自动化（细节在文档中）。
- **扩展与集成**：GraphQL API 便于集成外部系统。

## 适用场景
- 需要高度定制的B2B销售、客户管理场景，尤其是技术驱动型团队。
- 希望将CRM纳入CI/CD流程，实现基础设施即代码（IaC）的组织。
- 需要AI增强销售流程，例如自动记录上下文、建议下一步动作。
- 数据敏感型企业，必须自托管以满足合规要求。

## 同类对比
- **vs Salesforce**：开源、可自托管，无按用户收费的许可成本，定制方式更贴近开发者习惯，但生态和集成广度不如Salesforce成熟。
- **vs EspoCRM / SuiteCRM**：Twenty 更强调现代开发体验和AI能力，技术栈更现代（NestJS/React），但传统功能覆盖（如营销自动化）可能尚在追赶。
- **vs 低代码CRUD工具（如Retool）**：Twenty专注CRM领域，预置销售管道、联系人管理等业务原语，而非通用低代码。
