## 它是什么
由 Nous Research 打造的自我进化的 AI 代理，内置闭环学习引擎——能从对话中创建技能、自动改善技能、主动沉淀记忆，并支持跨会话搜索和用户建模。可在 $5 VPS、GPU 集群或按需无服务器基础设施上运行，通过 CLI 或 Telegram / Discord / Slack / WhatsApp / Signal 等平台交互，实现真正的随时随地个人助手。

## 为什么火
星数超 20 万，社区贡献活跃（v0.17.0 有 245 位贡献者）。它切中了高级用户的痛点：**聊天 AI 不该被锁在某个聊天应用的网页或 App 里**。Hermes 提供统一的网关和终端界面，允许自由切换 OpenAI、Anthropic、Nous Portal、OpenRouter 等 200+ 模型，同时提供记忆、技能、定时任务、子代理等生产级特性，远超普通聊天壳。

## 技术栈
- 核心语言：Python
- 终端界面：全功能 TUI（多行编辑、自动补全、流式工具输出）
- 网关：多平台自托管网关，支持 Telegram、Discord 等，支持语音转写
- 后端：本地、Docker、SSH、Singularity、Modal（无服务器休眠/唤醒）、Daytona
- 记忆系统：FTS5 全文搜索 + LLM 总结，配合 Honcho 辩证法用户建模
- 技能系统：符合 agentskills.io 开放标准，自主创建与自我完善
- 工具网关：可通过 Nous Portal 使用 Firecrawl、FAL、OpenAI TTS 等，也可自带密钥
- 安装：shell 脚本一键部署 Python 3.11 环境及相关依赖

## 核心能力
1. **自学习循环**：代理自主策划记忆、创建技能，并在使用中自我改进
2. **多模型随意切换**：`hermes model` 一键更换模型，无代码改动
3. **跨平台对话**：同一代理在 Telegram、Discord、WhatsApp、Signal、CLI 等保持连续会话
4. **后台自动化**：内置 cron 调度器，用自然语言设置定时任务
5. **子代理并发**：可派生子代理并行工作，或编写调用工具 RPC 的 Python 脚本压缩上下文成本
6. **无服务器省钱**：Modal/Daytona 后端使代理在空闲时休眠，几乎零费用
7. **研究就绪**：支持批量轨迹生成和轨迹压缩，用于训练工具调用模型

## 适用场景
- 需要跨平台统一个人助理的极客和远程工作者
- 希望完全控制数据、又能低成本自托管的高级用户
- 利用模型无关性做 A/B 测试和 prompt 工程的研究人员
- 需要定时报告、监控或备份并通知到聊天软件的 DevOps 场景
- 想用自然语言调度任务并观察执行的家庭自动化控

## 同类对比
- **ChatGPT / Claude 官方客户端**：封闭生态，模型锁定，无法自托管，无跨平台网关
- **LangChain / Langroid 等 Agent 框架**：偏向开发者构建，缺乏开箱即用的终端界面和多平台网关
- **OpenClaw / Clawdbot**：Hermes 提供迁移命令，表明其前身为 OpenClaw，但 Hermes 强化了记忆与技能的自进化，并扩展了平台覆盖（iMessage、Raft 网络等）
- **Home Assistant 语音助手**：侧重智能家居，非通用代理
