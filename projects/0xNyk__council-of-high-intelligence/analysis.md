## 它是什么
一个基于技能（skill）的命令行决策审议系统，集成 18 个预定义的 AI 人格（从亚里斯多德到卡尼曼、费曼、托瓦兹等），在多个 LLM 提供商（Claude、OpenAI、Gemini、Ollama、NVIDIA NIM、Cursor）之间自动分发成员，进行结构化的多轮审议。用户只需在 Claude Code 或 Codex 中输入 `/council 问题` 即可启动。

## 为什么火
- 真实模型多样性：自动检测可用的 CLI 工具，将不同人格分配给不同后端，避免“同一模型换皮肤”的伪多样性。
- 强制异议与反偏见机制：引入极性配对、反对配额、新颖性闸门和反事实提示，阻止过早共识，确保不同的推理路径被彻底探讨。
- 低门槛高天花板：一条命令三种模式（完整/快速/双人），20 种预定义三人小组，适合从快速决策到深度战略审议的各种场景。
- 透明异步输出：决议中包含未解决问题、建议下一步、可行动作条款，而非一味输出表面共识。

## 技术栈
- 核心：Shell 脚本 + Claude Code / Codex 的 skill 系统
- 提供商适配层：通过 `subagent`（Anthropic 原生）、`codex exec`（OpenAI）、`gemini -p`（Google）、`ollama run`、NVIDIA NIM 的 OpenAI 兼容 API、`cursor-agent -p` 来调用不同模型。
- 安装器：`install.sh` 支持 `--codex` 参数，将技能注册到对应平台。

## 核心能力
- **多提供商自动路由**：运行时探测已安装的 CLI 和 API 密钥，自主分配人格到不同模型，无需手动配置席位。
- **三种审议模式**：全模式（3 轮辩论）、快速模式（2 轮无交叉质询）、双人模式（极性对辨证）。
- **20 个预定义领域三人组**：如架构、伦理、调试、AI 安全等，每个小组由互补/对立人格组成。
- **审议协议**：问题重述门控（必须先行），反对匿名化（第二轮质询时用稳定标签隐藏对手身份），反从众指令（要求成员先指明自己先前论证的缺陷），主席模型合成最终裁决。
- **行动导向输出**：所有裁决强制包含行动条款（决策步骤、风险、时间线）、未解决问题、下一步建议，并附加会话元数据。

## 适用场景
- 架构决策（微服务 vs 单体、技术选型）
- 产品/策略决策（发布安排、定价模型、创业竞争）
- 复杂调试或系统设计（逻辑验证、反馈循环分析）
- 需要多视角风险评估的议题（如开源策略、AI 安全）

## 同类对比
与直接询问单一 LLM 或使用简单的 multi-agent 框架相比，该工具强调结构化的对抗性审议和模型输出源的真正异质性。不同于仅靠提示词变化实现的“多角色”，这里通过不同提供商和模型家族（及本地模型）来产生认知多样性。类似项目如 AutoGen 更多面向自动化工作流，而本项目是一个轻量的、即时可用的决策命令。
