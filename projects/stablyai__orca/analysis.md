## 它是什么
Orca 是一个桌面与移动端的 AI 集成开发环境（ADE），专为同时编排和管理多个 CLI AI 编码代理而设计。每个代理运行在隔离的 Git worktree 中，你可以在统一的界面上并行监督、比较和合并输出。

## 为什么火
它抓住了“多代理并行”这个生产力跃升点。相比单代理工具，允许一键将同一任务分派给 Claude Code、Codex、OpenCode 等多个代理，快速对比结果并保留最佳方案。内置终端（Ghostty 级 WebGL 渲染）、浏览器设计模式、原生 GitHub/Linear 集成和移动通知进一步减少上下文切换，吸引追求效率的超级个体和团队。

## 技术栈
- 桌面端：TypeScript（Electron 或类似框架），WebGL 加速终端渲染。
- 移动端：原生 iOS（Swift/AppKit）与 Android（APK 分发）。
- 通信：本地运行时与远程通过 SSH 或 Tailscale。

## 核心能力
- **并行工作树**：将提示广播至多个代理，每个代理拥有独立的 git worktree，相互隔离。
- **终端分割**：无限分割、滚动历史持久化、WebGL 渲染。
- **设计模式**：点击内嵌 Chromium 页面元素，将 HTML/CSS/截图直接喂入代理。
- **原生 GitHub & Linear**：不离开用户界面处理 PR、Issue、项目看板。
- **移动伴侣**：手机监控代理状态，接收完成通知并发送后续指令。
- **SSH 远程工作树**：在远程机器上运行代理，自动重连和端口转发。
- **AI 差异注释**：在差异行上评论并回传给代理，实现审查闭环。
- **Orca CLI**：用 CLI 脚本化工作流（创建 worktree、截图、点击等）。

## 适用场景
- 需要同时评估多个 AI 代码生成方案，挑选最优解。
- 同时进行多个独立任务（如修复 Bug、开发新功能、重构），不想切换终端。
- 希望在手机上随时查看代理进度，而非一直盯着屏幕。
- 在远程重型机器上运行计算密集型代理，本地仅做控制面板。

## 同类对比
- **VS Code + Copilot/Agent 插件**：多为单代理、单会话，缺少原生并行比较和隔离 worktree。
- **Cursor/Codex CLI**：单代理终端模式，无统一 UI 展示多路结果。
- **Ghostty/Terminal 模拟器 + tmux**：提供多窗口但无代理编排层，需手动管理工作树。
