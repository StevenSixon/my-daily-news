## 它是什么

Orca 是一款面向 100x 开发者的 AI 代理开发环境（ADE），支持在独立 git 工作树中并行运行多个编程代理（如 Claude Code、OpenAI Codex、Pi 等），并从桌面端或移动端统一监控、比较和合并结果。不托管模型，只编排已有 CLI 代理，完全使用用户自有订阅。

## 为什么火

- **并行编排刚需**：成熟工程师常需在不同代理间对比输出，Orca 的隔离 worktree + 并行执行 + 结果对比直接命中痛点
- **移动端监控**：支持手机接收代理完成通知并补充指令，填补了桌面代理运行时“离开即失控”的空白
- **生态中立**：不绑定特定代理或模型，可接入任何终端代理，降低迁移成本
- **YC 背书**：YC 投资且快速迭代（每日发版），社区活跃度高

## 技术栈

- **语言**：TypeScript（仓库主语言）
- **终端渲染**：自研基于 WebGL 的高性能终端（类似 Ghostty），支持无限分屏与重启后会话持久化
- **跨平台**：macOS/Windows/Linux 桌面端（Electron 或类似框架）以及 iOS/Android 移动端
- **Git 集成**：深度依赖 git worktree 实现隔离，无服务端部署，本地离线操作
- **通信**：SSH 远程工作树、自动重连、端口转发

## 核心能力

- **并行工作树**：同一条指令分发至多个代理，各自在不同 worktree 执行，可随时切换查看 diff，选择最优提交
- **移动伴侣**：通过 iOS/Android 应用接收代理完成推送，支持追加指令，实现全程掌控
- **Design Mode**：在嵌入式 Chromium 中点击 UI 元素，自动将其 HTML/CSS/截图注入代理提示词，加速前端迭代
- **终端分屏**：WebGL 渲染的高性能终端，无限分屏，滚动历史支持重启恢复
- **GitHub & Linear 深度集成**：在 IDE 内直接浏览 PR、Issue、看板，一键从任务创建 worktree
- **SSH 远程代理**：将代理运行在远程机器上，本地仅保留交互界面，支持断线重连
- **AI Diff 注释**：在生成的差异行上直接写反馈回传给代理，形成审查闭环
- **文件拖拽交互**：从文件浏览器拖入文件或图片至代理对话区，自动引用
- **CLI 可编程**：通过 `orca` 命令脚本化工作流，实现 agents 驱动 Orca
- **多账号切换**：可热切换不同代理的 API 账号，支持用量和限频监控

## 适用场景

- 需要对比多个 AI 编码助手生成质量的资深开发者
- 管理复杂任务时希望拆解并行处理且保留各方案历史的团队 Lead
- 移动办公场景下想随时给后台运行的代理发送指令的用户
- 前端 UI 开发中需反复与代理沟通修改细节的设计/前端工程师
- 希望利用远程高配机器跑代理而本地保持轻量的开发者

## 同类对比

- **vs. 单代理 CLI** (aider、claude-code standalone) ：Orca 不替代代理，而是通过工作树和统一界面解决多代理管理、并行对比、移动通知等问题，是编排层。
- **vs. IDE 内 AI 插件** (Cursor、Copilot)：Orca 更像终端代理的 Docker Compose，支持任意 CLI 代理，而不仅仅是编辑器内补全；移动端和并行 worktree 是其独特优势。
- **vs. 任务编排工具** (LangChain、AutoGPT)：Orca 面向开发实操，聚焦于代码编写和版本管理，而非通用自动化流程。
