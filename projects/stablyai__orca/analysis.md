## 它是什么
Orca 是一个 AI 代理开发环境（ADE），让开发者在桌面和移动端同时管理多个 CLI 编码代理（如 Claude Code、Codex 等），每个代理运行在独立的 git worktree 中，支持终端分屏、内置浏览器、设计模式以及远程 SSH 连接。

## 为什么火
当前 AI 编码工具多为单代理命令行，多代理协作、结果比较、上下文管理极其繁琐。Orca 提供“发散-收敛”工作流：一次提示广播给多个代理，生成代码后可直接在 UI 中标注 diff 并合并，极大地提升了多模型驱动的开发效率。YC 背书，社区活跃，Star 快速增长。

## 技术栈
- 语言：TypeScript
- 终端：集成 Ghostty 级 WebGL 渲染终端
- 核心能力依赖：git worktree、SSH、Chromium 嵌入
- 桌面应用跨平台（macOS/Windows/Linux），移动端（iOS/Android）
- 宿主应用，通过调用用户本地安装的代理 CLI 工作

## 核心能力
- 并行 worktree：一键创建多个独立工作区，每个运行不同代理，可比较结果并合并最优解
- 高性能终端：无限分屏、滚动历史持久化
- 设计模式：在嵌入式 Chromium 中点击元素，直接将 HTML/CSS 和截图发入代理提示
- GitHub/Linear 原生集成：从议题或 PR 直接启动工作树进行修复和评审
- SSH 远程 worktree：在远程服务器上运行代理，自动重连和端口转发
- AI Diff 标注：在生成的 diff 行上评论，反馈给代理进行修改
- 文件拖放：拖拽文件或图片到代理提示
- CLI 控制：`orca` 命令可脚本化整个工作流，实现自动化
- 移动伴侣：手机端监控代理状态，发送后续指令
- 支持 20+ 种 CLI 代理，包括 Claude Code、Codex、Grok、Cursor、OpenCode 等

## 适用场景
- 需要比较多个 AI 代理代码生成质量的开发者
- 远程服务器上运行代理，本地轻量监控的 DevOps 场景
- 团队从 GitHub Issue 直接启动代理修复，并在线审查
- UI 调优中使用设计模式快速反馈
- 构建持续集成流水线调用 Orca CLI 自动生成代码

## 同类对比
没有直接竞品。现有 AI IDE（如 Cursor、Windsurf）仅支持单一代理或内置模型，缺乏多代理并行和 worktree 隔离。Orca 是唯一将多代理编排、终端、浏览器和移动伴侣融为一体的工具，填补了“代理操作系统”的空白。
