## 它是什么
Orca 是一个面向 AI 编程代理的「编排器」——一个跨平台的桌面应用（macOS/Windows/Linux），配套移动端 App。它让你在同一仓库中并行运行多个 CLI 编码代理（如 Claude Code、Codex、OpenCode、Pi 等），每个代理在独立的 git worktree 中工作，互不干扰，最终你可以对比输出、合并最佳结果。Orca 还提供分屏终端、内置浏览器设计模式、GitHub/Linear 原生集成、SSH 远程工作区、AI Diff 注释与移动端监控等全套功能。

## 为什么火
- **痛点尖锐**：开发者希望用多个模型同时解决同一问题，但手动管理多个终端、分支和文件极易混乱。Orca 通过 git worktree 隔离，把并行实验变成一键操作。
- **移动端创新**：能通过手机实时查看代理进度、发出后续指令，这在同类工具中罕见。
- **生态友好**：支持几乎所有主流 CLI 代理，不绑定单一供应商，MIT 开源，YC 背书。
- **快速迭代**：3 个月积累 9k stars，几乎每日发布新版本，社区活跃。

## 技术栈
- 语言：TypeScript
- 桌面框架：Electron（推测，因跨平台桌面应用）
- 终端渲染：基于 WebGL 的 Ghostty 级终端
- 编辑器：Monaco（VS Code 内核）
- 移动端：iOS 原生 / Android APK
- 集成：GitHub API、Linear API、SSH 远程连接
- 构建：Vite、esbuild 等（release notes 中提及依赖更新）

## 核心能力
- **并行 Worktree 编排**：一键将一条 prompt 分发给多个代理，每个代理在独立的 git worktree 中运行，互不干扰，事后可对比差异并合并。
- **移动伴侣**：iOS/Android 应用，可远程监控代理状态、接收通知、发送新 prompt。
- **终端分屏**：类 Ghostty 的高性能终端，支持无限分割、重启后恢复滚动缓冲区。
- **Design Mode**：内嵌 Chromium 窗口，点击任意 UI 元素即可将 HTML/CSS 和截图直接送入代理 prompt。
- **原生 GitHub/Linear 集成**：在应用内浏览 PR、Issues、项目板，一键创建工作区评审代码，无需切换上下文。
- **SSH 远程工作区**：在远程机器上运行代理，提供完整的文件编辑、git 操作和终端，带自动重连和端口转发。
- **AI Diff 注释**：在生成代码的差异行上直接留言，反馈给代理修改。
- **文件拖放**：将文件或图片直接拖入代理 prompt。
- **CLI 脚本化**：可通过 `orca` 命令行实现工作流自动化，代理也能驱动 Orca。
- **其它**：快速搜索、账户切换与用量追踪、Markdown 预览、通知等。

## 适用场景
- **多模型对比开发**：同一任务用 Claude、Codex、Pi 等并行生成，选出最佳方案。
- **代码审查与迭代**：审查 AI 生成的 diff，添加线内注释，让代理根据反馈修改。
- **移动办公**：在手机上监控和调度代理，适合碎片化时间。
- **远程重型计算**：利用 SSH 工作区在算力更强的服务器上运行代理。
- **自动化工作流**：通过 CLI 脚本批量创建任务，用于持续集成、测试等。

## 同类对比
- **与终端多标签页对比**：普通终端可手动开多个代理，但缺少 git 隔离、diff 对比、统一管理和移动端支持。Orca 提供完整 UI 层和编排能力。
- **与 Continue.dev 等 IDE 插件对比**：Continue 是编辑器内嵌的聊天助手，主要面向单代理；Orca 是独立环境，侧重并行多代理和 git 隔离。
- **与 AgentOps 平台对比**：AgentOps 专注于 monitoring/logging，Orca 是开发执行环境，并带编排特性。
- **独特性**：集 worktree 隔离、移动监控、SSH 远程、设计模式于一体，目前无直接竞品。
