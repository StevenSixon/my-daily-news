## 它是什么
Orca 是一个专为开发者打造的 Agent 开发环境（ADE），让你在统一界面中并行运行、管理和对比多个 CLI AI 编码 Agent（如 Claude Code、Codex、OpenCode、Pi 等）。每个 Agent 运行在独立的 git 工作树中，互不干扰，支持本地和远程（SSH）。桌面端提供类 VS Code 编辑器、终端分裂、Chromium 内嵌浏览器等能力，移动端可远程监控与交互。

## 为什么火
1. **多 Agent 对比**：可以同时向多个 Agent 发同一个 prompt，比较结果，合并最佳代码，极大提升实验和选型效率。
2. **工作区隔离**：为每个 Agent 创建独立 git worktree，避免文件冲突，事后可分别 diff 或丢弃，保持主分支干净。
3. **终端一等公民**：基于 WebGL 的高性能终端，分割操作及滚动持久化，尤其适合长时间运行的 Agent 任务。
4. **移动端陪伴**：当 Agent 跑在桌面时，手机可接收完成通知、查看输出、发送后续指令，适合离开电脑但又想跟踪进度的场景。
5. **成熟的集成生态**：内置 GitHub/Li near 任务管理，可右键任务创建 worktree，无缝审查 PR/issue；支持设计模式选中 UI 元素直接注入 Agent 提示。

## 技术栈
- 主要语言：TypeScript
- 桌面框架：Electron（推测，因跨平台桌面应用）
- 终端渲染：提及 Ghostty 级终端，可能使用 WebGL 加速；README 提到“Ghostty-class terminals with WebGL rendering”。
- CLI 工具：orca CLI 用于脚本化操作
- 移动端：iOS App Store 上线，Android 提供 APK
- 许可：MIT

## 核心能力
- **并行 Agent 编排**：单个提示扇出至多个 Agent，每个在独立 worktree 中运行。
- **隔离的工作树**：基于 git worktree，可合并、对比、废弃。
- **终端分裂与持久化**：无限分割，滚动历史重启亦保留。
- **内嵌浏览器 & 设计模式**：点击 UI 元素自动抓取 HTML/CSS/截图发送给 Agent。
- **GitHub/Li near 原生集成**：在应用内浏览任务，从任务打开工作树，无需切换上下文。
- **AI Diff 注释**：在 diff 行上直接添加注释返回给 Agent 迭代。
- **SSH 远程工作树**：在远程强算力机器上运行 Agent，自动重连和端口转发。
- **CLI 驱动**：Agent 亦可通过 orca 命令创建 worktree, snapshot, click 等，实现自动化流水线。
- **移动端监控**：手机接收通知并可发送跟进指令。

## 适用场景
- 需要同时试验多种 AI 编程工具的开发者，希望快速横向对比输出。
- 追求一键启动多个 Agent 并行解决问题的懒人型极客。
- 长期运行 Agent 任务（如大规模重构）时希望离开桌面但保留监控能力的场景。
- 团队使用不同 AI 工具，需要统一管理变更和审核流程。

## 同类对比
- **Terminal + tmux + 多个 Agent CLI**：手动管理窗口和上下文，缺乏结构化 worktree、通知整合和移动端能力。
- **VS Code + Copilot Chat + 其他 Agent 插件**：通常是单 Agent 交互，没有原生多 Agent 对比和隔离的工作树。
- **Aider / Continue**：侧重于单一 Agent 的交互优化，不提供并行编排的统一环境。
Orca 的优势在于将所有 Agent 作为一等公民在同一时空下互不干扰地运行，且融合了 IDE 的编辑、版本控制、任务管理和移动端通知。
