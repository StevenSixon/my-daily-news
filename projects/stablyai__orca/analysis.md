## 它是什么
Orca 是一款桌面+移动端的 AI 开发环境（ADE），能够并行运行多个命令行 AI 编码代理（Claude Code、Codex、OpenCode 等），每个代理运行在独立的 Git worktree 中，所有操作在统一界面内监控、对比和管理。

## 为什么火
满足“同时多路 AI 助手干活”的需求，避免在多终端间反复切换，且通过 Git worktree 天然隔离代码变更，可以快速评估不同代理的输出并选择最佳结果，提升实验与交付速度。

## 技术栈
- 桌面应用（macOS/Windows/Linux）和移动端（iOS/Android），推测基于 Electron 或类似跨平台框架。
- 终端渲染采用 WebGL 技术，具备高性能端口分割和持久化滚动。
- 语言为 TypeScript，支持 CLI 脚本化工作流。
- 集成 Git 操作、GitHub/Linear API、SSH 远程连接等。

## 核心能力
- **并行工作树**：一条 prompt 广播给多个代理，在隔离环境中并发执行，可对比 diff 合并胜者。
- **终端分割**：Ghostty 级终端，支持无限分割，重启后保留历史。
- **设计模式**：点击浏览器中的 UI 元素，自动提取 HTML/CSS 及截图并注入到代理 prompt。
- **GitHub/Linear 集成**：在 app 内浏览 PR、Issue、项目看板，一键创建关联工作树。
- **SSH 工作树**：将代理运行在远程机器上，自动重连和端口转发。
- **AI 差异注释**：在 diff 上直接评论并反馈给代理，完成审阅-修改闭环。
- **拖放文件到代理**：使用内置编辑器，支持拖放文件和图片进代理 prompt。
- **Orca CLI**：可通过命令行脚本化工作流，例如创建工作树、截图、点击、填表等。
- **移动伴侣**：从手机监控代理、发送后续指令。

## 适用场景
- 需要对比多个 AI 编码工具效果的选型或实验。
- 高并发开发任务：同时解决多个 Issue 或并行探索不同实现方案。
- 需要隔离环境但不想手动创建大量分支的日常编码。
- 远程开发，利用强大服务器运行代理。

## 同类对比
与单个 IDE 插件（如 Cursor、Copilot）不同，Orca 定位在编排层，不替代代理本身，而是用统一界面管理多个代理和任务。相对于直接使用命令行或 tmux 管理多代理，Orca 自动处理 Git worktree 隔离和结果合并，并提供了移动监控、SSH 集成等增值能力。
