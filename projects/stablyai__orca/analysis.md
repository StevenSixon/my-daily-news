## 它是什么
Orca 是一个面向 100x 开发者的 AI 编排器。它不提供 AI 模型，而是让你在独立的 git worktree 中并行运行多个 CLI 编码代理（如 Claude Code、Codex、OpenCode 等），对比生成结果，手动或自动合并最优方案。桌面端和移动端均可监控、干预代理行为。

## 为什么火
- **多代理对比**：同一个任务分派给多个代理，结果择优合并，直接提升代码质量。
- **完全掌控**：使用你自己的模型订阅和 API key，不锁定供应商，不产生额外 SaaS 费用。
- **移动端支持**：首创的移动伴侣，可在手机上接收通知、发送后续指令，适合异步长时间任务。
- **开发者体验**：集成 GitHub/Linear 问题追踪、终端分屏、设计模式、SSH 远程开发等实用功能，降低切换成本。

## 技术栈
- 语言：TypeScript
- 终端：基于 WebGL 的高性能渲染，Ghostty 级别体验
- 架构：桌面应用（macOS/Windows/Linux）+ 移动应用（iOS/Android）
- 集成：任意 CLI 代理通过终端运行，由 Orca 管理 git 工作树隔离
- 目标：本地优先，远程可通过 SSH 工作树连接强算力机器

## 核心能力
1. **并行工作树**：一份 prompt 分发给多个代理，各在独立 git worktree 中运行，互不干扰。
2. **多代理支持**：开箱支持 Claude Code、Codex、Cursor、Grok、Copilot 等 20+ 种 CLI 代理，也兼容任何终端命令。
3. **移动伴侣**：iOS/Android 端可随时查看进度、推送新任务，任务完成时收到通知。
4. **设计模式**：在嵌入式 Chromium 窗口中点击任意 UI 元素，自动提取 HTML/CSS/截图注入代理 prompt。
5. **AI 差异注释**：直接在 diff 视图上给代理反馈，无需离开 IDE 即可完成 review-edit-commit 循环。
6. **原生集成**：内置 GitHub 和 Linear 面板，可浏览任务、开启工作树、对比评审。
7. **SSH 工作树**：代理可运行在远程机器上，支持自动重连和端口转发。
8. **终端分屏**：无限分屏、持久化滚动缓冲区，可在此处单独操作各代理的终端。
9. **CLI 自动化**：提供 `orca` 命令，脚本化创建工作树、截图、点击、填充等操作。

## 适用场景
- **代码生成对比**：需求较复杂时，同时让 3-5 个代理各自实现，选取最佳方案组合。
- **跨平台开发**：代理在本地与远端灵活调度，充分利用各自算力。
- **异步自动化**：通过移动端随时分配任务，尤其在通勤或离开电脑时跟踪进度。
- **代码审查**：利用 AI 差异注释直接驱动代理修改，形成紧密反馈循环。

## 同类对比
- **vs. Cursor/VS Code Copilot**：后者是 AI 辅助编码，Orca 是代理编排平台；Cursor 聚焦于单个代理的内联补全，Orca 让多个独立代理协同工作。
- **vs. Aider/Claude 终端**：Aider 是单一会话的编码代理，Orca 提供多会话隔离和结果对比，并加入项目管理、移动端等更高维度能力。
- **vs. Windsurf/Antigravity**：两者本质仍是传统 IDE 的 AI 增强，Orca 定位在代理间的“调度者”，更贴近 DevOps 编排思想。
- **优势**：本地自主运行，无额外订阅成本；并行隔离机制天然适合多代理策略；移动端生产力延伸是其他工具缺失的。
