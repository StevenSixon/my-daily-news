## 它是什么
Orca 是一个面向 100x 构建者的 AI 编排器（ADE）。它允许你在隔离的 Git 工作树中并行运行多个 CLI 编码代理（如 Claude Code、Codex、OpenCode、Pi 等），并在统一的界面中跟踪进度。提供桌面应用和移动伴侣。

## 为什么火
多代理开发模式渐成刚需，但手动管理终端窗口、对比输出、切换上下文极其低效。Orca 提供“一人同时指挥多个代理”的体验，配合 Git 工作树、设计模式、SSH 远程等深度集成，大幅降低并行开发的摩擦。YC 背书和每日快速迭代进一步推高社区热度。

## 技术栈
- 语言: TypeScript
- 终端: WebGL 渲染（Ghostty 级），基于 xterm.js beta
- 桌面应用: 推测基于 Electron（未明确声明）
- 移动端: iOS (App Store / TestFlight) 和 Android (APK)
- CLI: 提供 `orca` 命令行工具，可脚本化代理操作

## 核心能力
- **并行工作树**: 可为每个代理创建独立 Git worktree，结果可比较合并。
- **移动伴侣**: 手机端监控代理状态、接收通知、发送跟进指令。
- **终端分割**: 无限分割、WebGL 加速、重启后保留回滚。
- **设计模式**: 在内嵌 Chromium 窗口中点击 UI 元素，自动捕获 HTML/CSS/截图注入代理提示。
- **GitHub & Linear 原生集成**: 浏览 PR/议题，从任务直接打开工作树，行内 AI diff 注释。
- **SSH 工作树**: 在远程机器上运行代理，支持文件编辑、Git、终端及端口转发。
- **拖拽文件到代理**: 从文件浏览器拖放文件/图片到代理提示。
- **Orca CLI**: 允许代理驱动 Orca 本身（如 `worktree create`, `snapshot`, `click` 等）。
- **账户/用量追踪**: 监控 Claude/Codex 用量，热切换账户。

## 适用场景
- 需要同时评估多个编码代理（如对比 Claude Code vs Codex）的开发者。
- 希望用同一套工具管理 GitHub/Linear 任务与 AI 代理的团队。
- 远程服务器上运行代理并希望本地/移动端监控的 DevOps 场景。
- 利用“计算机使用”让代理操作桌面 GUI 的自动化流程。

## 同类对比
- **Cline / Continue / Cursor**: 均为编辑器插件或 AI-IDE，侧重单一代理在编辑器内的辅助；Orca 定位为“代理的 IDE”，核心在于并行编排多代理并提供全套终端、工作树隔离及移动监控，而非单纯代码补全。
- **直接终端使用 CLI 代理**: 手动管理多个终端和 Git worktree 繁琐易错，Orca 将其自动化并提供可视化比较、通知与远程能力。
