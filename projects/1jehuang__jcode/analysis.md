## 它是什么
jcode 是一个用 Rust 打造的终端 AI 编码代理框架（Harness），提供高性能的 TUI 界面、多会话工作流和可无限定制的 AI 编程体验。它不是单一模型，而是一个连接多种 LLM（OpenAI、Claude、AWS Bedrock 等）的指挥器，强调资源效率与自动化记忆管理。

## 为什么火
- **极致性能**：启动仅需 14ms，内存占用 27MB（关闭本地嵌入），单会话到 10 会话额外开销仅约 10MB，碾压 Claude Code、Cursor Agent 等（动辄数百 MB、启动超 1 秒）。
- **内置语义记忆**：自动将对话转换为向量存入图数据库，后台提取、整理，无需用户显式调用记忆工具，实现“人类式”回忆。
- **全端覆盖**：文档显示桌面版（DESKTOP_*）和移动端（iOS 客户端 / 模拟器）正在开发，未来可跨平台。
- **极客友好**：Rust 项目、MIT 许可、清晰的 crate 边界、编译性能优化计划，源码级可控。

## 技术栈
- 语言：Rust
- 界面：终端 TUI（自定义渲染，非 Electron）
- 记忆：本地向量嵌入 + 图式存储（余弦相似度检索），支持镜像图重整理
- 集成：MCP（多通道协议）、LLM 提供商（OpenAI, Anthropic, AWS Bedrock, 浏览器代理等）
- 构建：静态链接二进制，支持 curl 一键安装

## 核心能力
1. **多会话内存开销恒定** — 每个额外会话只增加 ~10MB，适合长期运行多个项目。
2. **实时语义记忆** — 自动抽取会话记忆，在后续对话中被动注入相关信息，无 token 燃烧。
3. **高性能 UI** — 侧面板可实时显示文件、diff，内嵌 mermaid 图渲染速度是常规方案的 1800 倍。
4. **原生版本控制意识**（AGENT_NATIVE_VCS_CORE_BEHAVIOR.md）— 可直接理解 git 状态并纳入 agent 行为。
5. **钩子系统（HOOKS.md）** — 允许自定义事件触发动作。

## 适用场景
- 日常开发中需要快速切换多个项目，且不想要 IDE 式内存杀手。
- 长生命周期的 AI 编程会话，需要跨会话记忆积累（如大型重构、持续调试）。
- 在低配 VPS 或老旧机器上运行 AI 编码助手。
- 喜欢终端操作并希望完全控制 AI 交互流程的极客。

## 同类对比
- **Claude Code / Cursor Agent / Codex CLI**：启动慢、内存高，jcode 在资源效率上优势明显。
- **pi / OpenCode**：功能相似但多会话内存膨胀严重（pi 单个新增会话约 76MB，OpenCode 约 318MB），jcode 仅 10MB。
- **GitHub Copilot CLI**：同样终端式，但性能不到 jcode 的零头。
- 所有竞品均无内置语义记忆系统，需用户手动管理上下文。
