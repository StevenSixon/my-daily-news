## 它是什么
umadev 是一个用 Rust 编写的编码代理（coding agent），它驱动用户已有的 AI 编码 CLI（Claude Code、Codex 或 OpenCode）作为持续会话的“大脑”，自身不绑定任何模型。它将一次构建请求拆分为产品经理、架构师、UI/UX 设计师、前后端工程师、QA、安全、DevOps 共八个专家角色，由一名技术协调者调度，形成类真实团队的结构化交付流程，产出代码、文档、质量报告和交付证明。

## 为什么火
- **解决 AI 编码的典型痛点**：AI 直接写代码无规划、前后端契约不一致、UI 千篇一律、交付物无质量证据。umadev 将“随手写代码”提升为“带 PRD、架构、UI/UX、执行计划、质量门禁的完整交付”。
- **复用已有工具和账户**：不需要额外的 API 密钥，直接用用户已登录的 Claude Code/Codex/OpenCode，降低接入门槛。
- **本地化、离线 RAG 与自我进化**：内置 459 个工程标准知识和双通道检索（BM25+本地嵌入模型），不依赖网络和外部 API；还能记录每次运行的错误，形成反思记忆，越用越聪明。
- **治理透明可审计**：基于 UMADEV_HOST_SPEC_V1 规范，约 112 条治理规则，每一步都有结构化文档和合规映射，适合团队或需交付证明的场景。

## 技术栈
- 语言：Rust（单二进制），通过 npm 发布外壳
- 依赖：用户需安装一个 AI 编码 CLI（Claude Code / Codex / OpenCode）
- 本地嵌入模型：`multilingual-e5-small`（f16，~224 MB），基于 candle 推理，可选启用向量通道
- 检索：纯 Rust BM25 + 向量模型融合（RRF），带 HyDE 查询扩展
- 规范驱动：遵循 `UMADEV_HOST_SPEC_V1` 规格文件
- 工作分支隔离：所有构建在 `umadev/<slug>` 分支上进行，不污染当前工作分支

## 核心能力
1. **多角色团队模拟**：八个专家角色按需调度，协调者制定执行计划并强制门禁。
2. **全流程交付**：需求路由、研究、PRD/架构/UI设计、前端实现、预览、后端实现、质量门禁、交付证明。
3. **双通道本地 RAG**：工程规范 + 项目代码地图的组合检索，提升提示质量。
4. **自我进化记忆**：记录错误和频率，触发反思纠正，形成项目级经验。
5. **治理与合规**：质量门禁包括构建、测试、lint、契约检查、运行时探针、安全扫描。
6. **交互与无头两种模式**：chat UI 或 `umadev run` 非交互式构建，均可得到同等的规划与交付质量。

## 适用场景
- **快速原型到交付**：一句话描述即可生成包含前后端、文档的完整小型应用。
- **存量项目功能迭代**：在已有代码库上添加特性，工具自动映射现有代码并遵守团队规范。
- **团队协作与审计**：需要 PRD、架构图、验收标准、测试证据的正式交付场景。
- **离线或受限网络环境**：本地 RAG 和嵌入模型使其在无网或内网环境也能提供工程标准辅助。

## 同类对比
与单纯的 AI 编码 CLI（如 Claude Code 单独使用）相比，umadev 带来了结构化的团队流程和治理；与一些 multi-agent 框架（如 CrewAI、AutoGen）相比，umadev 强调“不拥有模型”，而是驱动已有 CLI 作为大脑，且内置了工程标准 RAG 和交付证明。它更偏向“工程交付规程”而非灵活的自主代理编排。但目前版本相对较新，生态和社区规模较小。
