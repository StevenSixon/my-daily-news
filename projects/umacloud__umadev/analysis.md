## 它是什么
umadev 是一个 Rust 编写的命令行 agent，它模拟一个真实的软件开发团队（产品经理、架构师、UI/UX 设计师、前后端工程师、QA、安全、DevOps 加一个协调者），驱动你本机已登录的 AI 编码 CLI（Claude Code/Codex/OpenCode）完成从需求到交付的全流程。它不直接调用模型 API，而是把现有 CLI 当作“大脑”和“双手”，自身负责编排、知识检索和质量把关。

## 为什么火
当前 AI 编码工具普遍缺乏工程规范：直接写代码、无需求文档、前端后端 API 不匹配、UI 千篇一律、遗留大量 TODO 和假数据、无法交付可审计的产物。umadev 解决了这些痛点：每个任务都有路由、研究、PRD/架构/UIUX 文档、执行计划、分步骤实现、质量门和交付证明，确保产出像真实团队交付一样完整、可审查。它还自带 459 个工程标准知识文件，通过纯本地 BM25+向量混合检索注入提示词，并具备从错误中学习的记忆机制，越用越贴合项目。该项目从“super-dev”演化而来，从单纯代码治理升级为全流程 agent，完全开源 MIT 协议，社区关注度快速上升。

## 技术栈
- 核心语言：Rust (1.87+)
- 分发方式：npm 全局安装（作为下载器，实际运行 Rust 二进制），同时提供源码构建
- 外部依赖：需已有 Claude Code、Codex 或 OpenCode 已登录
- 本地 RAG：纯 Rust 实现 BM25 + Candle 运行 multilingual-e5-small 嵌入模型（f16，约 224MB），零网络、零 API 密钥
- 存储：本地文件系统（知识库、记忆、项目事实）、Git 分支管理
- 架构：spec 驱动，基于 UMADEV_HOST_SPEC_V1 规范，包含 32 条规则约 112 项治理检查

## 核心能力
1. **全流程开发团队模拟**：八个角色分工协作，协调者规划调度，不写代码只控流程。
2. **结构化交付流**：需求路由 → 澄清 → 研究 → 产出 PRD/架构/UIUX 文档 → 生成执行计划 → 前端实现并预览 → 后端实现 → 质量门（构建、测试、lint、契约检查、运行时探测、治理扫描） → 交付证明包（评分卡、合规映射等）。
3. **自适应规模**：根据任务复杂度决定流程深度，小修改不跑全流程。
4. **本地知识检索**：内置 459 项工程标准，结合项目现有代码映射，通过双通道混合检索（BM25+矢量，RRF 融合，HyDE 查询扩展）为每个任务提供上下文，无需联网。
5. **自我改进记忆**：记录生成过程中的错误频率，触发反思并注入后续提示，同类错误不再重复。
6. **项目事实记忆**：自动捕获并存储构建命令、环境路径等事实，保持一致性。
7. **分支隔离**：所有构建在独立 `umadev/<slug>` 分支进行，不污染工作区，不自动合并或推送。
8. **多基地支持**：可在 Claude Code、Codex、OpenCode 之间切换，灵活使用不同底层模型。

## 适用场景
- 需要快速从一句话需求得到可运行、可审查的完整项目（SaaS、管理后台、工具应用等）。
- 团队或个人希望为 AI 生成代码加入工程约束和知识库，避免“无脑输出”。
- 已有 Claude Code/Codex/OpenCode 使用经验，希望升级为更结构化的开发流程。
- 注重交付审计、合规证明的安全或企业场景。
- 本地开发，不希望将代码和标准泄露到外部 API。

## 同类对比
- **Devin/Cognition**：全托管 agent，需云端环境和 API 密钥，封闭性强；umadev 完全本地，利用已有 CLI 和登录态，成本更低。
- **GPT-Pilot/OpenDevin**：部分流程设计，但多依赖 API 调用且治理较松散；umadev 有明确规范文件和门控机制。
- **aider**：专注于单会话代码修改，无团队角色和交付物概念；umadev 强调全流程管理和交付物。
- **Cline/Continue**：VS Code 插件，侧重补全和交互；umadev 是独立的命令行工具，专注项目级构建。
综上，umadev 的差异化在于“**Agent as Team**”理念，将工程规范前置，且对基础设施依赖极低。
