## 它是什么
Herdr 是一个运行在终端内的代理多路复用器。它以单个 Rust 二进制 (~10MB) 提供类似 tmux 的持久会话、分屏和标签，但专门为 AI 编码代理 (Claude Code、Codex、Pi、Devin CLI 等) 构建。它可以自动检测每个代理的状态（阻塞/工作中/完成/空闲），并通过一个实时面板展示全局视图，无需任何钩子或配置。

## 为什么火
随着终端驱动的代理激增，开发者需要同时运行多个代理，却苦于 tmux 无代理感知、GUI 管理工具平台锁定且不支持远程。Herdr 填补了这一空白：它把代理当作一等公民，提供统一的终端 UI，支持任意 SSH 连接、会话持久、鼠标操作和脚本化控制。发布仅几个月即获 8.8k+ Star，反映了开发者对「可编程的、轻量级代理工作台」的强烈需求。

## 技术栈
- 语言：Rust
- 终端渲染：直接操作伪终端 (PTY)，提供真实终端视图
- 进程模型：客户端-服务端，后台 Daemon 管理 panes/代理，客户端可随时连接或断开
- API：本地 Unix Socket API 供代理自行编排工作区
- 跨平台：Linux、macOS 完全支持，Windows beta
- 构建工具：Cargo，提供 Homebrew、mise、Nix 等安装方式

## 核心能力
1. **真实终端**：每个代理在一个独立 PTY 中运行，全屏 TUI 能正确渲染。
2. **代理状态自动检测**：基于进程名和终端输出启发式判定代理的阻塞/工作中/完成/空闲状态，零配置。
3. **工作区/标签/面板**：鼠标原生拖拽分割，可自由组织按仓库或任务的视图。
4. **会话持久与远程连接**：后台服务保持所有代理存活，支持 detach/reattach（包括手机 SSH），甚至可通过 `--remote` 模式在远程服务器运行并本地访问。
5. **代理可编排**：任何代理通过 Socket API 可以创建/分割/缩放面板、读取输出、订阅状态变化，实现代理之间的协作。
6. **插件与集成**：官方集成提供会话恢复、语义状态上报等增强功能，可用任何语言编写插件。

## 适用场景
- 单个开发者同时运行多个编码代理、需要实时查看哪个代理在等你输入、哪个在工作或者被阻塞。
- 远程开发：在 VPS 上启动 herdr 运行多个代理，然后通过 SSH 从任何地方连接，保持工作流不被中断。
- 自动化流水线：代理本身可以调用 herdr 的 API 动态创建新终端、分割视图、启动子任务，形成自我协调的代理集群。
- CI/CD 或临时环境中的代理调试，而不需要安装图形界面或浏览器。

## 同类对比
- **tmux / Zellij**：经典终端复用器，但没有代理意识，无法自动报告代理状态，需要手动写钩子。Herdr 提供即时的代理面板和零配置检测。
- **GUI 代理管理器 (Conductor, Cmux, Emdash)**：多为 Mac 专用、闭源、将终端包裹在 GUI 中；不支持远程头显、无法通过 SSH 使用。Herdr 是纯粹的终端程序，运行在任何平台、任何环境下。
- **Warp / Hyper**：现代终端应用，但不专注代理复用、无持久会话。Herdr 专注于代理工作流的一站式管理。

Herdr 的独特价值在于把 tmux 的会话持久性与代理的语义理解结合起来，同时保持极简运行和完全自托管的特性。

## 版本动态
根据 `v0.7.1` (2026-06-24) 的 Release 说明，项目仍在快速迭代：
- 新增更新检查和代理检测清单的独立禁用选项
- 增强 Linux 前台进程识别，支持被 VM/Bubblewrap 等封装工具隐藏的代理
- 用户自定义键绑定现在能正确覆盖默认绑定，提高可配性
- 工作树操作（worktree）修复了分支已存在时的检出现，并支持长时间 Git 操作异步执行
- 多人会话/钩子的可靠性提升，减少了跨进程会话错误
- Windows 下多行文本粘贴修复

这些改进显示了团队在易用性、稳定性和跨平台方面的持续投入。
