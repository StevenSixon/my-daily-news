## 它是什么
Hermes Browser Extension 是一个社区开发的 Chrome/Edge/Chromium 侧面板扩展，专为 Nous Research 的 Hermes Agent 设计。它通过浏览器内置的 Side Panel API，将当前标签页的网页上下文（标题、URL、正文、表单等）注入到托管在本地或远程的 Hermes Agent 会话中，并流式返回响应。扩展不内嵌任何 AI 模型，完全依赖已有的 Hermes 实例，实现本地优先、只读的浏览器集成。

## 为什么火
项目在发布后几天内获得 276 星，反映了 Hermes Agent 社区对浏览器集成需求的迫切。相比常规 AI 浏览器助手，它不依赖第三方 API，所有处理在用户信任域内完成，且 v0.1.6 新增快速命令、标签隔离和敏感信息自动红化，让极客用户可以安全地在日常浏览中利用 AI 能力，无需担心银行、密码等标签泄露。这种“本地优先+隐私感”的设计击中了本地 AI 用户的痛点。

## 技术栈
- **扩展框架**：Chrome Extension Manifest V3，使用 Side Panel API、service worker 等。
- **前端**：纯 JavaScript，无重型框架，自主构建脚本（`npm run build`）生成 dist 目录。
- **通信**：通过 fetch 与 Hermes API 服务器交互，支持 WebSocket（Dashboard 模式），自动处理会话同步。
- **Hermes 集成**：调用 `/v1/models`、`/api/sessions`、`/v1/skills` 等端点，兼容 Hermes Agent 的工具、技能、MCP 服务器生态。

## 核心能力
1. **上下文注入**：自动捕获活动标签页的标题、URL、可读文本、选中内容、链接、表单等，封装为“不可信上下文”发送给 Hermes。
2. **标签控制**：支持跟随活动标签或固定到特定标签，固定标签下保持独立的对话历史与 Hermes 会话绑定。
3. **隐私保护**：自动识别银行、密码、支付等敏感标签，红化其标题和 URL，防止泄露；扩展无 `debugger`、`cookies` 等侵入性权限。
4. **快速命令**：内置 `/summarize`、`/explain`、`/rewrite`、`/tabs`、`/action-items` 等快捷操作。
5. **上下文收据**：每轮对话后展示“What Hermes saw”，方便调试与信任。
6. **多标签选择**：可选择多个打开标签注入上下文。
7. **语音输入**：优先使用 Hermes 音频转录，降级为浏览器语音。
8. **主题与配置**：提供多套主题，支持本地代理端口切换。

## 适用场景
- **研究者/开发者**：拥有本地 Hermes Agent 环境，希望快速总结、解释网页内容，或基于多标签调研生成报告。
- **隐私敏感用户**：需要在涉及敏感信息的浏览过程中使用 AI 助手，但不愿将数据发送到外部服务。
- **本地 AI 工作流**：将浏览器作为 Hermes 的感知窗口，驱动无头分析或知识提取。

## 同类对比
常见浏览器 AI 扩展（如 ChatGPT Sidebar、Monica）依赖云端 API，数据会离开浏览器，且无法与本地 Agent 工具链整合。Hermes Browser Extension 完全自托管，可重用 Hermes 已配置的模型、技能和 MCP 服务器，实现端到端数据控制。其敏感信息红化和标签隔离是专为本地 Agent 设计的隐私增益，而多数竞品缺乏此类细粒度控制。

## 版本动态
项目仍处于公共 alpha（未上架商店），最新 v0.1.6 于 2026-06-28 发布，重点强化了上下文控制和隐私保护，添加快速命令、标签隔离和敏感信息红化，权限模型保持最小化。后续可能放宽可选主机权限。社区贡献活跃，已有两位贡献者推动多标签选择与快速命令功能。
