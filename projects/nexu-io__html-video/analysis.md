## 它是什么
html-video 是一个程序化视频的元框架，由 Open Design 团队开发。它让用户的本地编码代理（Claude Code、Cursor、Windsurf CLI 等）通过自然语言描述或粘贴文章/仓库链接，自动生成包含多帧动画的 HTML 故事板，并调用可插拔渲染引擎在本地输出 MP4 视频。核心思想是在现有的多种视频渲染引擎之上提供一个代理友好的抽象层，避免用户去学习 Remotion、Motion Canvas 等各自不同的作者范式。

## 为什么火
* **零费用、无锁定**：Apache-2.0 协议，本地渲染，不按次数收费，没有云依赖，对个人开发者和希望完全控制工具的团队极具吸引力。
* **代理驱动**：无缝接入当前最流行的 14 种 AI 编码助手，将“对话即视频”变为现实，大幅降低制作程序化视频的门槛。
* **多引擎抽象**：不再受单一渲染引擎的限制，未来可随时切换后端，保护生成的内容资产。
* **链接直达视频**：一键将微信公众号文章、GitHub 仓库内容转化为解说视频，直击内容营销和开发者教育场景的痛点。

## 技术栈
* **语言**：TypeScript（Monorepo，使用 pnpm）
* **运行时**：Node.js ≥20，浏览器相关组件使用 Playwright + Chromium
* **视频编码**：ffmpeg（libx264）
* **默认引擎**：Hyperframes（通过无头 Chromium 录制 HTML 动画帧 → WebM → MP4）
* **AI 音轨**：MiniMax（可选背景音乐和旁白）
* **工程化**：提供浏览器端 Studio 和 CLI 两种使用方式

## 核心能力
1. **代理驱动生成**：支持 14 种本地 CLI 代理，能自动检测路径并切换。
2. **多源输入**：从提示词、网页文章（包括需要服务端渲染的 WeChat 文章）、GitHub 仓库提取内容，生成结构化故事板。
3. **21 个预置模板**：覆盖标题卡、数据图表、电影感画面、终端效果、产品宣传、品牌尾板等多种风格，且均为真实渲染，非模型图。
4. **多帧故事板编辑**：生成内容图（实体/数据/文本节点及其关系），转化为多场景顺序；支持内联文本修改和拖拽排序。
5. **AI 音轨混入**：导出时可混合背景音乐和旁白。
6. **极简部署**：完全本地运行，只需 Node、ffmpeg 和 Chromium，一条命令启动 Studio。

## 适用场景
* **内容创作者**：将博客文章、社交媒体帖子快速转为短视频。
* **开源项目维护者**：为 GitHub 仓库生成介绍视频。
* **产品团队**：制作产品发布、功能演示视频。
* **数据分析师**：将数据报告转化为动画图表视频。
* **AI 应用开发者**：作为视频生成的模块集成到自己的 Agent 工作流中。

## 同类对比
| 工具 | 方式 | 限制 | html-video 的优势 |
|------|------|------|------------------|
| Remotion | React 组件式 | 商业使用需付费，学习曲线陡 | 免费，通过代理屏蔽引擎细节，未来也能适配 Remotion |
| Motion Canvas / Revideo | TypeScript 生成器 | 适合特定类型，需手动编码 | 提供上层抽象和模板，并计划适配这些引擎 |
| Hyperframes | HTML+CSS+GSAP | 单一引擎 | 已作为默认引擎集成，但还提供多引擎切换可能 |
| 其他 AI 视频工具（如云端服务） | 黑箱生成 | 按量付费，可控性差 | 完全本地，代码可控，不按次付费 |
