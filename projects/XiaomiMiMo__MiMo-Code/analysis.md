## 它是什么
MiMoCode 是一个终端原生的 AI 编程助手，可读写代码、运行命令、管理 Git，核心特色是拥有跨会话的持久记忆系统，能够自我改进。它从 OpenCode 分叉而来，保留了多提供商支持、TUI、LSP、MCP 等能力，并增加了智能上下文管理、多代理编排和自主目标驱动的开发循环。

## 为什么火
在众多终端 AI 助手中，MiMoCode 通过“记忆”与“自进化”形成鲜明差异，解决了长期项目中 AI 反复遗忘上下文的问题。内置免费体验通道（MiMo Auto）大幅降低上手门槛，且 Star 超过 1 万，反应出强烈的社区兴趣。

## 技术栈
- 主语言：TypeScript
- 运行环境：Node.js/Bun（bun install/dev/turbo typecheck）
- 持久记忆：SQLite FTS5 全文搜索
- 界面：终端 TUI
- 语音：TenVAD 音频活动检测 + 小米 MiMo ASR
- 基础框架：OpenCode（开源终端 AI 助手）

## 核心能力
- **多 Agent 协作**：内置 build（全权限）、plan（只读分析）、compose（规格驱动）三种主 Agent，可创建子 Agent 并行工作。
- **持久记忆系统**：跨会话的项目记忆（MEMORY.md）、自动检查点（checkpoint.md）、临时笔记、任务进度，通过 FTS5 检索智能注入上下文。
- **智能上下文管理**：自动保存检查点、上下文接近窗口上限时从检查点重建，并使用 token 预算控制记忆注入量。
- **任务跟踪**：树状任务 ID（T1, T1.1…）与检查点集成，任务进度在会话重启后保留。
- **Goal/Stop 判定**：独立 judge 模型评估目标是否真正达成，防止自主工作过早停止。
- **Compose 模式**：规格驱动的结构化开发工作流，内置规划、执行、代码审查、TDD、调试、验证等技能。
- **Dream/Distill 自进化**：/dream 从历史会话提取知识到项目记忆；/distill 将重复手动流程封装为可复用技能或子 Agent。
- **语音输入**：/voice 实时流式语音转录，需 sox 环境。
- **零配置启动**：MiMo Auto 匿名通道免费使用，也可接入 OpenAI 兼容 API。

## 适用场景
- 需要长期维护同一项目、希望 AI 助手记住架构和习惯的开发者。
- 复杂多步骤开发任务，需多代理分工和状态追踪。
- 想从会话中自动提炼经验、减少重复机械工作的团队。
- 偏好终端工作流、不喜欢离开命令行的极客。

## 同类对比
- vs Claude Code：Claude Code 也有子代理和类似技能，但未强调显式的跨会话记忆进化机制，MiMoCode 的持久记忆和 dream/distill 是独特优势。
- vs OpenCode：MiMoCode 的直接基座，缺少记忆系统、自主循环、compose 工作流和自我改进，MiMoCode 全面增强。
- vs Aider：专注于 inline 代码编辑，没有记忆系统和多代理编排。
- vs Cursor/Copilot：编辑器集成而非终端原生，不具备项目级持久记忆和自进化。
