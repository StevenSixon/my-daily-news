## 它是什么
MiMoCode 是小米推出的终端原生 AI 编程助手，基于 OpenCode 分叉，增加跨会话持久记忆、多代理（build/plan/compose）、目标导向自主循环和语音输入等功能。可通过免费限时通道零配置启动，也支持连接主流 LLM 提供商。

## 为什么火
发布不足一个月即获 10k+ Star，免费使用期降低了尝试门槛，记忆与子代理协同设计直击 AI 编程工具长期上下文丢失痛点。

## 技术栈
- 语言：TypeScript
- 运行时：Bun（开发）
- 终端 UI：自研 TUI
- 记忆存储：SQLite FTS5
- 集成：LSP、MCP、多提供商 API
- 语音：TenVAD + MiMo ASR，可选 OpenRouter 等

## 核心能力
- **多代理切换**：build（全权限）、plan（只读分析）、compose（规格驱动流程）
- **持久记忆**：项目记忆文件（MEMORY.md）、自动检查点、笔记、任务进度，通过 FTS5 全文搜索注入上下文
- **智能上下文管理**：自动检查点、接近窗口限制时重建上下文、基于 token 预算的带重要性排序内容注入
- **任务追踪**：树形任务系统，与检查点联动，会话恢复时保留进度
- **子代理系统**：主代理可创建子代理并行工作，支持生命周期跟踪和取消
- **目标/停止条件**：可设置会话停止条件，由独立裁判模型验证，防止过早终止
- **Compose 工作流**：内置规划、执行、代码审查、TDD、调试、验证、合并等技能，覆盖从 spec 到交付的全生命周期
- **语音输入**：实时流式语音转文本，支持米默登录用户
- **Dream & Distill**：从对话轨迹提取项目记忆，去除过时内容；发现重复工作流并封装为技能或命令
- **其他**：支持 Claude Code 配置导入，MCP 服务器连接，自定义快捷键和主题，实验性 Max 模式

## 适用场景
复杂中大型项目开发，需要跨多天的上下文持续累积；自主多步骤任务（如重构、跨文件更改）；团队希望沉淀项目级规则和知识；希望减少重复解释、加速 AI 协作效率的开发者。

## 同类对比
与 Claude Code 等相比，MiMoCode 的核心差异化在于持久记忆系统和子代理编排，能够自主保存和利用项目上下文，避免每次对话重新学习。它还继承了 OpenCode 的多提供商、LSP、MCP 等能力，并加入了自我蒸馏的改进闭环。相比通用的 IDE 插件形式 AI，它更偏向终端原生的强大可控性。
