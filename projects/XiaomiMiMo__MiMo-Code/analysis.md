## 它是什么
MiMoCode 是一个终端原生的 AI 编程助手，定位为“模型与智能体共同进化”的编码工具。它基于 OpenCode 分叉开发，在保留其多提供商支持、TUI、LSP、MCP 等能力的基础上，重点构建了持久记忆系统、多 Agent 协作、目标驱动的自主循环、组合工作流以及自我改进机制（dream/distill）。

## 为什么火
项目在发布短短几天内获得近万星，核心吸引力在于它直接回应了现有 AI 编码助手的通病：上下文断裂。它通过 SQLite FTS5 实现项目记忆的持久化，并借助自动检查点和智能上下文重建，让 Agent 在会话重启后无需重新理解项目，大幅提升连续开发体验。加之免费限时通道、零配置启动和一键从 Claude Code 迁移，降低了上手门槛。

## 技术栈
- 语言：TypeScript
- 运行时：Node.js（npm 全局安装）或 Bun（开发）
- 数据库：SQLite（FTS5 全文搜索）
- 协议：MCP（工具扩展）、LSP（语言感知）
- 语音：TenVAD 暂停检测 + MiMo ASR 模型
- 安装方式：curl 一键脚本或 npm

## 核心能力
- **多 Agent 架构**：内置 build（全权限开发）、plan（只读分析）、compose（规格驱动的编排模式）三种主 Agent，可按 Tab 切换；系统可按需生成子 Agent，支持并行、后台执行和生命周期管理
- **持久记忆系统**：项目记忆文件（MEMORY.md）、会话检查点（checkpoint.md）、便签和任务进度，通过 FTS5 全文索引自动注入会话上下文
- **智能上下文管理**：根据上下文窗口自动触发检查点保存，接近限制时从检查点、记忆和最近消息重建上下文，确保长任务不中断
- **任务追踪**：树形任务 ID（如 T1、T1.2），与检查点系统集成，保留跨会话进度
- **目标/停止条件**：/goal 设定停止条件，由独立评判模型验证是否真正满足，防止“乐观停止”
- **Compose 模式**：从规格到交付的完整工作流，包含规划、执行、代码审查、TDD、调试、验证和合并等内建技能
- **语音输入**：/voice 激活实时流式语音，由 TenVAD 分割音频并增量转写（需 sox，仅限 MiMo 登录用户）
- **自我进化**：/dream 从最近对话中提取持久知识到项目记忆，/distill 发现重复工作流并打包为可复用技能或命令

## 适用场景
- 需要长期维护同一项目、频繁切换开发任务的个人开发者
- 团队协作中希望通过项目记忆统一代码风格和规则的场景
- 希望用终端轻量级助手替代笨重 IDE 插件的 Vim/Emacs 用户
- 探索 Agent 自主规划与记忆结合的研究者

## 同类对比
- **对比 OpenCode**：MiMoCode 作为其分叉，增加了持久记忆、自动检查点、子代理调度、目标驱动循环、Compose 工作流等，而 OpenCode 侧重多提供商和插件生态
- **对比 Claude Code**：MiMoCode 提供一键导入认证，且在持久化记忆和自主规划方面更主动，Claude Code 则更依赖会话内对话历史
- **对比 Aider/Cline**：Aider 强在批量编辑和地图式仓库概览，Cline 重编辑器集成；MiMoCode 在记忆和多 Agent 协同上形成差异
- 无公开基准测试数据，无法量化性能或效果提升
