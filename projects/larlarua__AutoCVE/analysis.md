## 它是什么
AutoCVE 是一个面向 CVE 挖掘的全流程自动化平台。用户只需配置模型并导入目标仓库，平台通过 Orchestrator 调度 Recon、Scan、Triage、Finding、Verification 等多个 Agent，自动完成源码审计、漏洞验证和 CVE 报告生成，支持中文和英文报告。

## 为什么火
项目在短短 7 天测试期内发现并提交了 30 个 CVE，覆盖 14 个开源项目，最高 CVSS 达 9.9，证明了其自动化挖掘的实际产出能力。对于安全研究人员，它把耗时的手工审计转化为一键式操作，显著降低了漏洞挖掘成本。

## 技术栈
- 后端：Python 3.11+，FastAPI
- 前端：React 18
- 数据：PostgreSQL 15
- 部署：Docker Compose，预构建镜像托管在 GHCR
- 模型层：需用户自行配置 LLM（OpenAI 兼容接口）

## 核心能力
- **一键 CVE 挖掘**：自动筛选项目、拉取仓库、创建审计任务、运行 Agent、生成 CVE 申报报告。
- **Multi-Agent 协同**：ReAct loop 驱动的 Finding Agent 可深度分析源码，结合工具调用与结构化终止机制产出证据。
- **三种审计模式**：增强扫描（快速）、智能审计（深度）、综合审计（全覆盖）。
- **交互式审计追踪**：支持用户追问，可视化审计过程。
- **Skills 扩展**：可为不同 Agent 配置专属技能。

## 适用场景
- 漏洞赏金猎人批量审计开源项目
- 企业安全团队自检内部源码
- 安全研究者复现或挖掘 CVE

## 同类对比
相比于手动使用 Semgrep/CodeQL 等 SAST 工具，AutoCVE 通过 LLM Agent 实现了更高层次的自动化决策与推理，能够过滤误报并动态验证，避免传统扫描的海量噪声。与单纯的 LLM 聊天式审计相比，它融合了工具编排和沙箱验证，产出可直接用于 CVE 申请的标准化报告。
