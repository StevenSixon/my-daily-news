## 它是什么
AutoCVE 是一个面向安全研究者的多智能体漏洞挖掘平台，能够对开源项目源码进行自动化审计，从项目发现、代码拉取、Agent 协查、漏洞验证到 CVE 申报报告生成，实现全流程“一键”完成。平台基于 Web 操作界面，支持增强扫描、智能审计（深度挖掘）和综合审计三种模式。

## 为什么火
项目在短短 12 天内吸引 429 颗星，并发布了实际 CVE 挖掘成果：7 天测试内发现 30 个有效漏洞，涵盖 14 个热门开源项目，其中最高 CVSS 达 9.9。它解决了安全研究者手写 POC、人工审计效率低的痛点，让漏洞发现从“手工作坊”进入“自动化产线”。

## 技术栈
- **后端**：Python 3.11+、FastAPI、PostgreSQL
- **前端**：React 18
- **编排**：Docker Compose 一键部署
- **核心架构**：Orchestrator 调度多 Agent（Recon、Scan、Triage、Finding、Verification），ReAct Loop 状态机、工具编排、Nudge 纠偏，最终由 Finding Agent 产出符合 CVE 申报条件的结构化证据

## 核心能力
- **一键 CVE 挖掘**：从 GitHub 仓库筛选到生成 CVE 报告的全自动流水线，并可配置仓库大小限制防止资源耗尽
- **Multi-Agent 协同**：不同 Agent 各司其职，Finding Agent 基于 ReAct 循环深度分析源码，结合工具调用和纠偏机制减少误报
- **三种审计模式**：快速扫描（工具误报过滤）、智能深度挖掘（面向 0day/CVE）、综合审计（全量覆盖）
- **交互式会话**：可对审计结果追加人工追问，补充证据或扩展分析
- **漏洞管理**：自动去重、结构化存库，支持中文/英文/CVE 格式报告导出
- **Skills 扩展**：可为不同 Agent 配置专属技能，灵活扩展能力

## 适用场景
1. **安全研究者 / 白帽子**：批量审计开源项目，快速产出 CVE 编号，提升挖洞产出比
2. **企业安全团队**：自建漏洞挖掘流水线，对内嵌开源组件进行持续性风险评估
3. **漏洞学习与教学**：通过可视化审计过程理解自动化漏洞发现原理

## 同类对比
与单纯的 SAST 工具（如 Semgrep、CodeQL）相比，AutoCVE 不是静态规则匹配，而是利用 LLM 驱动的 Agent 进行上下文理解、证据链构建和动态验证，输出可直接用于 CVE 申请的完整报告。与手动渗透测试工具（Burp、Nessus）相比，它自动化了从源码审计到报告生成的全过程，且内置去重和误报过滤机制。缺陷是目前高度依赖 LLM 能力与 API 成本，未提供离线或本地模型集成说明。
