## 它是什么
AutoCVE 是一款面向 CVE 挖掘的智能体驱动平台，通过 Multi-Agent 协作实现从源码仓库导入、多阶段审计（Recon/Scan/Triage/Finding/Verification）到 CVE 报告生成的全程自动化。项目由 larlarua 开发，以 AGPL-3.0 开源，提供一键部署的 Docker Compose 方案。

## 为什么火
项目以真实成果证明能力：在为期一周的测试中，针对 14 个知名开源项目（如 Chartbrew、Lemmy、JeecgBoot 等）挖掘出 30 个安全漏洞，其中最高 CVSS 达 9.9，并成功分配 CVE 编号。这种“开箱即挖”的效果结合全流程自动化，大幅降低了安全研究人员和渗透测试工程师进行源码级漏洞挖掘的门槛，同时提供了企业级可扩展的多 Agent 架构。

## 技术栈
- **后端**: Python 3.11+, FastAPI, PostgreSQL 15
- **前端**: React 18
- **部署**: Docker Compose (GHCR 预构建镜像)
- **核心机制**: LLM Agent, ReAct Loop, Multi-Agent 调度, 沙箱执行环境
- **文档**: 提供完整的中英文用户指南、架构设计文档和 API 文档

## 核心能力
- **一键 CVE 挖掘**：自动完成项目筛选、仓库导入、任务创建、Agent 审计、CVE 报告生成，用户仅需复制报告提交申请。
- **Multi-Agent 协同**：Orchestrator 调度 Recon（信息收集）、Scan（工具扫描）、Triage（误报过滤）、Finding（深度漏洞挖掘）、Verification（验证）等 Agent。
- **三种审计模式**：增强扫描（快速过滤）、智能审计（Finding Agent 深度挖掘）、综合审计（全量分析），适配不同审计深度需求。
- **Finding Agent 专项优化**：内置 ReAct loop、工具编排、nudge 纠偏、FinalizeFinding 结构化输出，专门产出符合 CVE 申报标准的高价值漏洞。
- **交互式会话**：支持在审计过程中向 Agent 追问，补充证据或扩展分析，所有调用与活动日志可视化。
- **漏洞管理**：去重入库、结构化存储，支持中文/英文报告维护与导出。
- **Skills 扩展**：可为各 Agent 配置专属技能，灵活增强能力边界。

## 适用场景
- 安全研究员批量审计开源项目，快速获取 CVE 漏洞成果。
- 渗透测试团队在红队行动前进行自动化白盒/灰盒扫描。
- 企业安全团队对内部代码库进行常态化脆弱性发现。
- 学生/极客学习 AI+ 安全的实践平台，通过结果报告理解漏洞链。

## 同类对比
- 相比传统 SAST 工具（如 SonarQube、CodeQL），AutoCVE 结合 LLM 的推理能力进行深度上下文分析，不仅能发现签名漏洞，还能挖掘逻辑漏洞，并直接生成可申报 CVE 的报告。
- 相比通用漏洞扫描器（如 Nessus），AutoCVE 聚焦源码审计，更适用于自研或定制化项目。
- 相比纯手工 CVE 挖掘，它提供编排化的 Agent 流程，显著缩短从发现到申报的时间。
