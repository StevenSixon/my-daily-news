## 它是什么
DeerFlow 2.0 是一个从零重写的超级代理编排框架（SuperAgent harness），可协调多个子代理、持久记忆、沙盒化执行环境与可扩展的技能/工具系统，完成从几分钟到数小时的长周期任务。它前身是深度研究框架，现定位为通用“超级代理”平台。

## 为什么火
- 两周内即登 GitHub 趋势榜首，Star 超 7.4万，社区贡献活跃（v2.0 里程碑合并 182 个 PR）。
- 解决了单一 LLM 调用无法处理的复杂多步问题，通过子代理协作、沙盒隔离、长短期记忆实现可靠的任务执行。
- 丰富的集成：支持十余种模型、七种 IM 通道、多种搜索引擎，并提供 TUI 终端界面。

## 技术栈
- 后端：Python 3.12+，基于 LangChain/LangGraph 生态，使用 `uv` 包管理。
- 前端/工具链：Node.js 22+，TypeScript 项目，包含 TUI 客户端。
- 基础设施：Docker 容器化部署，沙盒通过社区 AioSandbox 或本地容器实现；消息网关连接 Slack/Telegram 等。

## 核心能力
- **子代理系统**：可递归调度子代理处理子任务，并实时归集 token 用量。
- **技能与工具**：内置深度研究、播客生成、音乐创作、图片/视频处理等技能；支持 Claude Code 集成及 MCP 服务器。
- **沙盒化执行**：可选远程或本地 Docker 沙盒，安全执行代码与命令。
- **持久记忆**：代理可更新自身 `SOUL.md`，实现跨会话学习与用户隔离。
- **上下文工程**：可配置循环检测、工具频率限制，提升长对话稳定性。
- **多模型与渠道**：支持 OpenAI、vLLM、Codex CLI、Claude Code OAuth 等；IM 通道可绑定用户自有账号。

## 适用场景
- 需要长周期自主研究的任务（市场调研、竞品分析、技术调研）。
- 多步骤编码工程（自动生成项目骨架、调试、文档生成）。
- 企业内部自动化工作流（通过 IM 通道执行复杂指令，如数据分析、报告撰写）。
- 研究者与极客探索 Agent 协作边界。

## 同类对比
与 LangChain/LangGraph 相比，DeerFlow 提供了开箱即用的超级代理模板与沙盒管理；相较于 AutoGPT、CrewAI 等纯多代理框架，它集成了更丰富的 IM 接入和自更新记忆机制；相比 GPT-Engin，它更侧重长时任务的可恢复性与生产级部署（含 Nginx 统一端点）。但缺乏与这些框架在基准任务上的横向对比数据。

## 版本动态
- v2.0.0 为全量重构版本，与 1.x 分支不共享代码（1.x 仍在维护）。
- 主要更新：运行状态持久化与中断恢复、用户自有 IM 帐号绑定、新增多个模型和社区搜索工具、可配置循环检测、子代理 token 流式归因等。
- 当前主线活跃，文档持续扩充。
