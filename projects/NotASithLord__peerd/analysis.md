## 它是什么
peerd 是一个运行在浏览器里的 AI 代理扩展（Chrome/Firefox），没有后端、没有云组件、没有遥测。它利用浏览器自身的沙箱（V8 隔离、WebCrypto、opaque-origin iframe）在本地执行完整的代理循环：读取和操控标签页、启动 JS Notebook 或 WASM Linux 虚拟机、构建客户端应用，并通过支持 P2P WebRTC 的预览通道在代理间共享构建物。密钥由用户自行提供（BYOK），所有数据仅加密存储在本地 vault 中。

## 为什么火
尽管 Star 不多（213），但它的架构理念击中了一个痛点：既想要浏览器自动化能力，又拒绝将 API 密钥或浏览行为交给第三方后端。它把信任边界收缩到浏览器平台本身，无构建步骤、纯原生 JS，安全模型建立在多年打磨的 Web 平台能力之上。对安全型极客、注重隐私的个人和想探索去中心化代理的开发者极具吸引力。

## 技术栈
- 语言：纯 JavaScript（ES2024+），无 TypeScript、无 JSX、无打包工具
- 运行时：Manifest V3（Chrome/Firefox）
- 沙箱：WebAssembly 运行 Linux VM，V8 隔离执行不可信代码
- 安全：WebCrypto 加密本地 vault，WebAuthn passkey 解锁，Subresource Integrity，safeFetch 封装；代理持有密钥的部分从不接触原始页面
- 架构：五个模块（provider/egress/engine/runtime/distributed），每个模块独立 README，跨模块通过 index.js 导入
- 预览功能：P2P WebRTC 网络实现去中心化应用共享

## 核心能力
- **标签页驱动**：AI 可读取、操控任意打开的标签页，执行网页自动化
- **沙箱计算**：在浏览器内启动 JS Notebook、完整的 WASM Linux VM、客户端应用
- **安全模型**：运行不可信代码的“一次性 runner”无密钥无网络，输出结果被标记不受信任；每个动作在生效前需通过实时页面验证
- **本地推理**：支持 Ollama，完全无需联网和账单
- **去中心化共享**：预览通道下的 P2P dwapps，构建物直接在代理间传输

## 适用场景
- 个人浏览器自动化：表单填充、信息批量提取、日常重复任务
- 本地开发与测试：在浏览器内运行轻量级沙箱环境进行代码评估
- 隐私敏感场景：研究需要无后端、无遥测的全栈代理环境
- 探索去中心化 Web 应用：跨浏览器实例共享 AI 生成的组件

## 同类对比
- 与 Selenium/Playwright 等自动化工具相比，peerd 是 AI 原生的代理，可理解自然语言指令，无需编写脚本
- 与基于云端的 AI 浏览器代理（如某些 Chrome 扩展）相比，peerd 完全离线可用（Ollama），密钥和会话零泄露，无供应商绑定
- 与 Copy-as-curl/开发者工具扩展相比，peerd 具备完整的沙箱计算和代理循环，功能更聚合
