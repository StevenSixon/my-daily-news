## 它是什么
peerd 是一个 Chrome/Firefox 浏览器扩展，实现了浏览器原生的 AI 代理循环。它能够利用模型（Anthropic、OpenRouter 或本地 Ollama）来读取和驱动你的标签页，在浏览器沙盒内启动 JS 笔记本、WebAssembly 编译的 Linux VM 以及客户端应用，并（在预览通道）通过 P2P WebRTC 分享构建物。无后端、无遥测，完全本地运行。

## 为什么火
- **隐私与安全设计**：采用浏览器内置安全机制（V8 隔离、WebCrypto 保管库、WebAuthn 解锁），代理持有密钥但不接触原始页面，页面环境由无密钥的子代理操作，防止提示注入扩散。
- **全栈浏览器原语**：直接在浏览器中运行沙盒计算，无需远程服务，一切都在本地完成。
- **BYOK + 无后端**：用户自备密钥，不收集任何数据，符合自托管和隐私极客的偏好。
- **P2P 潜力**：预览版包含去中心化 Web 层，代理间可点对点通信和共享应用。

## 技术栈
- **语言**：纯 JavaScript (ES2024+)，无 TypeScript，无 JSX，无打包器。
- **扩展架构**：Manifest V3，支持 Chrome 和 Firefox（Chrome 为主要开发目标）。
- **模块化**：五个顶层模块（provider, egress, engine, runtime, distributed），每个模块通过 index.js 暴露公共 API。
- **安全基础**：依赖浏览器平台（V8 isolates、WebCrypto、WebAuthn、iframe 沙箱、SRI），不自行实现加密或进程隔离。
- **预览版 P2P**：基于 WebRTC 数据通道。
- **依赖管理**：扩展内无 npm，通过原生模块导入。

## 核心能力
- **代理循环**：协调器委托任务给各个环境专属的 actor 子代理，每个 actor 仅持有对应环境的工具。
- **可视化自动化**：读取和驱动用户已打开的标签页（需 `<all_urls>` 和 `debugger` 权限）。
- **沙盒计算**：支持浏览器内运行的 JS 笔记本、编译为 WASM 的 Linux 虚拟机、个人客户端应用。
- **多模型支持**：Anthropic、OpenRouter、Ollama，可切换。
- **本地保管库**：密钥加密存储，通过指纹识别/Passkey 或恢复口令解锁。
- **安全防护**：动作验证（基于当前页面）、出站请求通过 egress 审查点、审计日志。
- **去中心化网络**（预览通道）：代理间 P2P 共享构建物。

## 适用场景
- 希望用 AI 自动操作浏览器完成复杂任务的开发者（如数据抓取、表单填写、网页测试）。
- 需要在浏览器内进行隔离式计算的原型开发（即时沙盒环境）。
- 研究隐私保护代理的用户（BYOK，无后端）。
- 对去中心化应用感兴趣的早期探索者。

## 同类对比
- **与 browser-use 等工具相比**：browser-use 基于 Playwright 等外部驱动，peerd 内嵌为浏览器扩展，权限更高，可直接访问用户会话和登录状态。
- **与自托管的 Agent 框架（如 AutoGPT）相比**：peerd 直接利用浏览器安全边界，减少攻击面，且无需部署后端服务。
- **与商业 AI 浏览器（如 Arc Max）相比**：peerd 完全开源、本地优先，无遥测，可自定义模型。

## 版本动态
- 当前处于 0.x 实验阶段，`v0.2.1` 为最新预览版，包含去中心化 Web 预览。
- 代码库活跃更新，但存储格式和 Surface 可能发生破坏性变更，无 V1 承诺。
- 商店版本待审核，预览版提供手动安装包。
