## 它是什么
Cognee 是一个开源的 AI 记忆平台，为 AI Agent 提供跨会话的持久长期记忆。它通过自托管的知识图谱引擎，混合向量嵌入与图推理，将各种格式的数据转化为可搜索、可关联的记忆，帮助 Agent 随时回忆、推理并基于完整上下文行动。

## 为什么火
Star 数近 1.8 万，反映了开发者对 Agent 记忆基础设施的强烈需求。传统 LLM 应用缺乏持久记忆，每次会话都丢失上下文，Cognee 填补了这一空白，且采用自托管方式，数据隐私可控，适合构建企业级“公司大脑”。

## 技术栈
- **语言**：Python（支持 3.10～3.14）
- **存储**：同时使用图数据库（如 Neo4j、LanceDB 图模式）和向量数据库（如 LanceDB、Qdrant 等），构建知识图谱与向量索引
- **LLM**：可插拔多种 LLM 提供商（OpenAI 等），用于内容理解、本体生成和问答
- **部署**：支持本地运行、Docker、Modal、Railway、Fly.io、Render 等多种一键部署方式
- **认知科学**：引入本体生成，从认知科学角度组织知识，使记忆更具推理能力

## 核心能力
- **记忆操作**：`remember`（存储）、`recall`（查询）、`forget`（删除）、`improve`（智能蒸馏优化）
- **双存储模式**：持久知识图谱 + 会话级快速缓存，后台异步同步
- **智能蒸馏**：自动合并重复信息、提炼关键知识点，并可视化标记蒸馏节点
- **多源摄取**：支持任意格式文档、对话等数据，自动构建图谱关系
- **Agent 集成**：提供 Claude Code 插件、OpenClaw 插件、MCP 服务器，可直接接入现有 Agent 工作流
- **安全与隔离**：用户/租户级数据隔离、API 认证、审计跟踪（OTEL 采集）

## 适用场景
- **客户支持 Agent**：联合财务、产品、客服等多渠道数据，随时回忆历史案例和解决策略
- **专家知识蒸馏**：将资深员工的查询模式、工作流自动化存储，帮助初级员工达到接近专家的水平
- **企业知识库**：构建公司级别的记忆底座，供各类内部 Agent 共享上下文
- **任何需要长期记忆的 AI 应用**：如个人助理、研究助手、代码助手等

## 同类对比
对比纯粹的向量记忆（如 mem0、LangChain 的 ConversationBufferMemory），Cognee 的核心差异在于：
- **知识图谱 + 向量双引擎**：不仅能语义搜索，还能推理实体关系，生成结构化知识
- **认知驱动本体**：基于认知科学自动生成领域本体，而不仅仅是存储原始文本片段
- **自托管优先**：数据完全留在本地或私有云，适合企业合规要求
- **完整的记忆生命周期**：从摄取、存储到蒸馏优化和遗忘，形成闭环
