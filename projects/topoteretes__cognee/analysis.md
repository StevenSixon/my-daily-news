## 它是什么
Cognee 是一个开源的 AI 记忆平台，为 AI Agent 提供持久化的长期记忆。它能够摄取任意格式的数据，自动构建自托管的知识图谱，让 Agent 跨会话回忆、关联并基于完整上下文行动。项目结合了向量嵌入、图谱推理和认知科学启发的本体生成，使文档既可被语义搜索，也可通过不断演化的关系网络发现深层联系。

## 为什么火
随着 LLM Agent 应用激增，“记忆”成为核心瓶颈。Cognee 填补了开源生态中“结构化、可进化的 Agent 记忆”空白，在 GitHub 上迅速获得 2.4 万星。它不与特定框架绑定，可独立部署，支持从文本到多模态的持续知识累积，并引入了“真理子空间”等创新反馈学习机制，让记忆质量随使用自动提升。

## 技术栈
- 核心语言：Python（3.10-3.14）
- 存储后端：支持 LanceDB（向量） + Neo4j/Postgres（图/向量），可自选组合
- LLM 集成：OpenAI 默认，可通过文档接入其他提供商
- 部署：Docker 镜像开箱即用，支持 Colima 等本地容器运行时
- 客户端：提供 Python SDK、Rust/TypeScript 客户端、MCP 服务器、Claude Code 插件
- 可观测：内置 OTEL collector，支持审计和追踪

## 核心能力
1. **统一记忆管道**：提供 `remember` / `recall` / `forget` / `improve` 四个原语，将摄取、图谱构建、检索、反馈学习浓缩为几行代码。
2. **会话记忆与全局记忆**：支持快速会话缓存，后台同步至永久知识图谱，实现短期与长期记忆分层。
3. **自进化记忆**：v1.2.2 引入“真理子空间”，从已接受的会话学习中蒸馏出紧凑的质心/槽位索引，用于重排搜索结果，使回答更符合已确认的事实。
4. **多模态与多源**：可摄取文本、代码、文档等多种格式，构建跨数据源的统一知识视图。
5. **安全与隔离**：支持用户/租户隔离，签名使用 sha256，提供可验证的审计追踪。

## 适用场景
- 需要跨会话持久记忆的对话 Agent（客服、个人助手）
- 企业内部知识库 Agent，需整合不同系统数据并基于历史交互优化回答
- 研究场景中需要可解释图谱推理与增量学习的 Agent 架构
- 重视数据隐私、希望自托管记忆层的团队

## 同类对比
- **Mem0**：同样提供记忆层，但更侧重纯向量检索，Cognee 强调知识图谱的显式关系构建与反馈驱动的自适应。
- **LangChain Memory**：框架内工具，与 LangChain 生态绑定，Cognee 作为独立平台更通用。
- **Zep**：注重生产级会话记忆与事实提取，Cognee 在认知架构和动态本体生成上更突出，且开源社区更活跃。
- **RAG 数据库**（如 Chroma、Weaviate）：纯存储与检索，缺记忆演化与 Agent 原生接口。

## 版本动态
最新版本 **v1.2.2** (2026-06-26) 是“Truth Subspace & Retrieval Improvements”，核心更新：
- 新增真理子空间构建器，从会话蒸馏的“课程”中生成质心（主题锚点）和槽位（分组示例），用于重排序搜索结果。
- 检索可选用反馈信号提升相关性（默认关闭）。
- 签名升级至 sha256，质心会话过滤更严格以降低噪音。
- 修复 LanceDB S3 问题，新增构建数据集上下文追踪及演示/测试。
整体上该版本保持了无破坏性变更、默认安全的设计原则。
