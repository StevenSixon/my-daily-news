## 它是什么
Cognee 是一个开源的 AI 记忆平台，专为 AI 代理设计。它通过自托管的知识图谱引擎，将任意格式的数据转化为可持久化的长期记忆，让代理在不同会话间保持上下文、建立关联并进行推理。核心思路是把“记忆”变成可查询、可演化的知识结构，而不仅仅是向量搜索。

## 为什么火
- 解决 AI 代理“失忆”痛点：多数 LLM 应用只在会话内保持上下文，Cognee 提供了跨会话的持久记忆层。
- 知识图谱 vs 纯向量：向量检索仅基于语义相似度，而图谱能捕获实体关系、推演事实，记忆更结构化、可解释。
- 自托管、数据私有：企业可在本地部署，确保数据不外泄，适合合规场景。
- 开发体验友好：提供 Python SDK、CLI、Docker 一键启动，以及 Claude Code 等代理的原生插件。

## 技术栈
- 核心语言：Python（3.10-3.14）
- 记忆存储：知识图谱（支持 Neo4j、PGVector 等） + 向量数据库（可插拔）
- LLM 集成：需用户配置 LLM 提供商，通过环境变量 `LLM_API_KEY` 接入，支持 OpenAI 等（可扩展）
- 部署方式：本地 pip 安装、Docker 容器、Docker Compose 多服务编排
- 辅助工具：MCP 服务器、Web UI（索引状态、搜索预览）、Claude Code 插件

## 核心能力
- **记忆操作基本原语**：`remember`（存储）、`recall`（检索，自动选择最佳搜索策略）、`forget`（删除）、`improve`（反馈学习）
- **会话级高速记忆**：支持 `session_id`，先查会话缓存再回退到持久图谱，延迟更低
- **自动图谱构建**：摄取文本后，自动抽取实体、关系，生成本体，并与向量嵌入结合
- **多模态支持**：宣称支持多模态数据（文档未详述具体格式）
- **生产级可靠性**：后台同步、错误恢复、租户/用户隔离、审计追踪（OTEL）
- **代理集成**：即插即用的 Claude Code 插件，自动捕获工具调用、注入记忆上下文

## 适用场景
- **客服代理**：统一客户历史交互、工单、产品信息，让代理理解全貌并推荐解决方案
- **专家知识蒸馏**：如 SQL Copilot，存储专家查询模式、表结构，帮助初级分析师复用经验
- **个人 AI 助理**：记录用户偏好、任务历史，实现个性化长期交互
- **企业知识中台**：连接不同数据源，构建可查询、可推理的统一知识库

## 同类对比
- **LangChain 内存模块**：侧重于会话缓冲和摘要，缺乏结构化图谱和跨会话持久化推理。
- **MemGPT / Letta**：引入“虚拟上下文管理”，但更偏操作系统式内存分页，Cognee 强调图谱构建和本体演化。
- **纯向量数据库（如 Chroma、Pinecone 辅以 RAG）**：只能返回语义相近内容，无法显式建模实体间关系。Cognee 同时提供图+向量双重索引，更适合需要关系推理的场景。
