## 它是什么
sandboxd 是一个开源后端引擎，专为 AI 应用构建产品（如 Lovable、Bolt、Replit）设计。它提供按用户隔离的云开发环境、内置编码代理和实时预览 URL，通过自托管方式实现。一个 HTTP 请求即可创建沙箱、运行 AI 代理、生成可访问的预览链接，底层使用 Docker 容器 + SQLite + Traefik 路由，无需 Kubernetes。

## 为什么火（解决什么痛点）
- 平台构建者常需从零实现多租户隔离、成本控制、代理编排、预览 URL，耗时数月；sandboxd 将这些集成到一键部署中。
- 闲置自动休眠、请求唤醒，极大提高服务器密度，节省云成本。
- 开源 MIT 协议，无供应商锁定，适合需要控制数据和利润的团队。

## 技术栈
- Go 编写的单一控制平面守护进程，通过 Docker CLI 管理容器。
- 数据持久化用 SQLite（WAL），重启后通过协调器恢复状态。
- 网络层用 Traefik v3 自动路由 TLS。
- 沙箱镜像内置 OpenCode/Claude Code 代理 CLI。

## 核心能力
- 一键创建隔离沙箱（独立文件系统、内存/PID 限制）。
- 通过 API 执行命令、写入文件、运行编码代理任务并流式返回进度（SSE）。
- 预览地址自动分配，停止后按需唤醒。
- 持久化工作区，数据在宿主机目录，容器销毁数据可保留。
- 支持 API 密钥注入和环境变量。

## 适用场景
- 构建 AI 应用生成器，如“描述应用→实时预览”。
- 为用户或分支提供临时预览环境（如 SaaS 工厂）。
- 在线编程练习平台、团队协作开发环境。

## 同类对比
- 相较于直接用 `docker run` 脚本：缺乏 URL 路由、空闲回收、状态持久化、API 网关等企业特性，维护成本随规模剧增。
- 相较于 Kubernetes：复杂度低，单机即可运行，适合不想维护集群的团队；但缺少跨主机调度，需扩展到集群时可通过接口替换为 k8s 后端。
- 相较于 Replit、Pinokio：自托管、无需依赖外部服务，完全掌控数据和成本。
