## 它是什么
sandboxd 是开源的 AI 应用构建后端，让你用一条命令在自己的服务器上运行类似 Lovable、Bolt、Replit 的体验：发送一个 HTTP 请求，就能创建一个隔离的 Linux 容器，在其中运行 AI 编码代理（OpenCode/Claude Code），并自动获得一个可分享的预览 URL。

## 为什么值得关注
构建“用自然语言生成应用”产品的真正门槛不是 prompt，而是底层的基础设施：多租户隔离、按需启动/休眠、代理编排、文件持久化、崩溃恢复。sandboxd 把这些一次性打成一个单机可运行的二进制 + Docker 方案，没有 Kubernetes、没有独立的数据库或消息队列，非常适合想快速验证想法或构建自托管 AI 平台的团队。

## 技术栈
- 控制平面：Go 单二进制，通过 Docker CLI 管理容器
- 路由/反向代理：Traefik v3，自动发现容器并生成预览域名
- 数据存储：SQLite（WAL 模式）作为唯一真实状态源，启动时通过 reconciler 与 Docker 实际状态对齐
- 沙箱隔离：基于 Docker + 加固的 runc（cap-drop ALL, no-new-privileges, 只读根文件系统）
- 存储：每个沙箱在数据目录下有独立绑挂的工作区目录，持久化保存

## 核心能力
- **隔离**：每个沙箱拥有独立的文件系统和资源限制，互不可见
- **内置编码代理**：OpenCode 与 Claude Code 预装在基础镜像中，支持带提示词的 agent 任务（可流式获取进度）
- **预览 URL**：沙箱自动向 Traefik 注册路由，生成形如 `s-<id>-<port>.preview.yourdomain.com` 的地址，支持 TLS
- **休眠与唤醒**：空闲沙箱自动停止释放内存，收到 HTTP 请求时透明唤醒（热启等待页 + 就绪探测）
- **持久化**：停止/销毁容器可以保留工作区文件，重建时数据不丢失
- **应用模板**：新版本支持预置可编译运行的脚手架（如 Vite+React+TS），避免代理从空白目录从零开始
- **应用实体**：引入了高于沙箱生命周期的 app 概念（名称、描述、标签），适合用户视角的应用管理

## 适用场景
- 构建 AI 应用生成平台（类似 Lovable/ v0）
- 托管多人使用的在线编码环境或编程练习场
- 为团队提供按需的预览环境（每个分支/PR 一个沙箱）
- 需要完全控制数据与成本的自托管 AI 编码产品

## 同类对比
- **自己写 Docker 脚本**：单人或少数容器足够，但管理预览域名、休眠唤醒、崩溃恢复、API 权限等会迅速变成平台级工程，sandboxd 替代了这一部分
- **基于 Kubernetes 的方案**：功能强大但运维复杂，sandboxd 用 SQLite + Docker 单机即可实现，资源开销低，且代码量少可读
- **SaaS 方案（Lovable/Replit）**：每月付费、数据在外部，sandboxd 自托管让你独享毛利和数据控制

## 版本动态
最新 v0.2.0 于 2026-06-22 发布，主要改进：
- 引入持久化 app 实体与 `/v1/apps` API，应用生命周期超出单个沙箱
- 新增内置的 React 标准模板（启动时自动 seeding），代理立即有可运行的起点
- 增加了端到端 CI 和镜像冒烟测试
- 任务提交加入 `timeout_s` 参数，修复了长任务被误杀的问题
- 沙箱支持 `idle_policy`（sleep/always_on）
- 多个可靠性修复（快照、空模板创建等）
所有改动向下兼容，只需重建基础镜像并让控制平面自动运行迁移即可。
