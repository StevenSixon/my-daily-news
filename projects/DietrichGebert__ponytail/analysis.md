## 它是什么
Ponytail 是一款注入“懒人资深开发者”人格的 AI 代理技能 / 插件。它通过一个 7 阶决策阶梯（YAGNI → 复用 → 标准库 → 原生 API → 已装依赖 → 一行代码 → 最少可行实现）在代理编码前强制思考，逼它写出最精简且安全的代码。它以 Claude Code、Codex、Gemini CLI 等平台的插件或 AGENTS.md 规则形式分发，安装即用。

## 为什么火
因为 AI 编码代理普遍过度工程化——为一个小需求安装重型库、生成长篇包装代码、增加 token 消耗和幻觉风险。Ponytail 用一套简洁规则直击痛点，并用严格基准证明效果：减少 54% 代码行、20% 成本、27% 耗时，同时保持 100% 安全（对比纯提示方案安全度会下降）。6.6 万星和多家平台官方适配证明了它的实用价值。

## 技术栈
核心是纯文本规则集，项目使用 JavaScript 编写轻量生命周期钩子（用于 Claude Code、Codex 等平台），不依赖后端服务。适配层简单：读取规则注入 LLM 上下文，注册斜杠命令。扩展性依赖各代理平台的插件机制。

## 核心能力
- **决策阶梯**：连续 7 层过滤，杜绝无意义代码
- **跨平台**：原生支持 16 个主流 AI 代理工具（Claude Code, Codex, Copilot CLI, Gemini CLI, Hermes Agent, Devin CLI 等）
- **命令增强**：提供 `/ponytail`、`/ponytail-review`、`/ponytail-audit` 等交互命令
- **安全不妥协**：验证、错误处理、安全防护一律保留，只砍冗余
- **自动触发**：基于任务描述自动激活，无需手动调用
- **原生推广**：鼓励使用 HTML `<input>`、标准 API 等替代第三方组件

## 适用场景
- 使用 AI 编码代理的任何项目（前端、全栈、脚本）
- 团队希望降低代码审查负担、减少依赖和文件变更量
- 快速原型开发中抑制代理的“设计过度”倾向
- 教育场景，向开发者展示 YAGNI 和最小化原则

## 同类对比
Ponytail 在同样使用 Haiku 4.5 的代理基准造中与“无技能基线”、“caveman（简洁文本控制）”、“YAGNI+单行提示”对比，各项指标最优：
- 代码量削减 54%（caveman 20%，简单提示 33%）
- 成本降低 20%（caveman 反增 3%，提示降 21%）
- 时间加快 27%（caveman 慢 2%，提示快 30% 但降安全）
- 安全性 100%（caveman 100%，提示仅 95%）
唯一同时斩获所有正向指标且未牺牲安全的方案。
