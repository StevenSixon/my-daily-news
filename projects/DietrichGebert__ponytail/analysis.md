## 它是什么
Ponytail 是一套嵌入 AI 编码代理的提示工程技能/规则集，模拟一个“最懒的资深开发者”的决策逻辑。通过一个预定义的阶梯（先问是否要写、是否已有、是否可用标准库/平台特性/已有依赖，最后才是最小实现），强制代理在产生多余代码前停下。它不是一个代码库，而是一组 `.md` 或 `.json` 规则文件，以及少量 Node.js 生命周期钩子，用于在 Claude Code、Codex、Copilot CLI、OpenCode、Gemini CLI 等超过 14 种代理工具中激活。

## 为什么火
Star 数超过 5 万，因其精准击中开发者痛点：AI 代理常常为简单需求生成庞大实现（如日期选择器引入整页组件）。Ponytail 通过严格的阶梯规则和始终保留的验证/安全/无障碍检查，将平均代码量削减 54%（在真实 FastAPI+React 仓库上实测），Token 用量和成本降低约 20%，同时保持 100% 的安全护栏，远超简单“写最短代码”提示（后者会丢掉安全护栏）。它适配几乎所有主流代理，零学习成本，且提供了 `lite`/`full`/`ultra` 模式来调整懒惰程度。

## 技术栈
- 规则文件：纯文本 Markdown 规则，存储在仓库的 `.cursor/rules/`、`.windsurf/rules/`、`skills/` 等目录，可被各代理直接引用或复制。
- 生命周期钩子：极少的 Node.js 脚本（`hooks/` 目录），用于在 Claude Code 和 Codex 中实现自动激活和模式切换，其余代理通过预定义的规则文件直接工作。
- 平台适配：覆盖 Claude Code、Codex、Copilot CLI、OpenCode、Gemini/Antigravity CLI、Pi、CodeWhale、Swival、OpenClaw 等，对不支持插件的代理提供复制规则文件的方式。

## 核心能力
- **阶梯决策模型**：7 级阶梯（存在？已有？标准库？原生平台功能？已安装依赖？一行？最小实现）在理解上下文后执行，避免毫无理由地跳过必要代码。
- **安全不可妥协**：信任边界校验、数据丢失防护、安全性与无障碍永远不被“精简”掉，区别于简单的 YAGNI 一条龙提示。
- **模式切换**：`/ponytail lite`、`full`、`ultra`、`off` 命令和 `PONYTAIL_DEFAULT_MODE` 环境变量控制规则严格程度。
- **子代理注入**（v4.8.3）：通过 `SubagentStart` 钩子让派生出的子代理也保持懒惰，防止主线程省而子线程滥写。
- **基准量化**：提供可复现的 agentic 和单次生成基准测试脚本，证明在多个维度（LOC、tokens、cost、time、safety）全面优于无技能基线和同类简洁提示。

## 适用场景
- 使用 Claude Code、Codex、Copilot CLI、OpenCode、Cursor、Windsurf 等现代 AI 编程助理的开发者。
- 希望控制 AI 生成成本、提升代码质量、避免过度工程化的团队。
- 对多项目管理要求统一编码风格和安全性底线的组织。
- 任何想在保留 AI 辅助效率的同时抑制其“创造冲动”的极客。

## 同类对比
- **Caveman**（稀疏散文风格）：同样精简，但 Ponytail 在 token、成本和时间上均更优，且保持 100% 安全，Caveman 在部分模型上可能导致 token 使用反升。
- **自定义 YAGNI + one-liner 提示**：代码减少 33%，但安全评分降为 95%，Ponytail 以更少的代码量保持满分。
- **项目自带的 .clinerules 或 cursorrules**：大多缺乏结构性阶梯和安全锚点，Ponytail 从“最小必要”出发，兼顾实效和安全。
