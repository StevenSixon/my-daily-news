## 它是什么
OpenCode 是一个开源的 AI 编码代理（coding agent），可在终端或桌面应用中运行。它提供两种内置代理：**build**（默认、全文件访问、可执行命令）和 **plan**（只读、命令需许可），适合开发与代码探索两种场景。此外还有一个内部使用的 **general** 子代理用于复杂搜索。

## 为什么火
项目在 GitHub 星数超过 18 万，社区活跃（Discord 频道），拥有多语言 README 和丰富的安装方式（curl、npm、Homebrew、Scoop 等）。开源且 MIT 许可降低了使用门槛，终端原生体验吸引了许多习惯命令行的开发者。

## 技术栈
- 主要语言：TypeScript
- 终端 UI：基于控制台的交互界面（可能有自有框架）
- 桌面应用：提供 macOS、Windows、Linux 的原生包（技术栈未详述，推测为 Electron 或 Tauri）
- 协议支持：从发版说明可见 MCP（Model Context Protocol）OAuth 集成
- 包分发：npm、Homebrew、Scoop、Pacman、Nix 等

## 核心能力
- **双代理模式**：Tab 键切换 build（读写）与 plan（只读），满足安全勘探与全自动编码需求
- **会话管理**：支持会话快照与回滚，可回退到之前消息状态并还原文件变更（v1.17.11 新增）
- **桌面增强**：拖拽标签页、Chrome 风格标签切换快捷键（mod+1~9）、TODO 停靠、跳转最新按钮优化
- **跨平台**：CLI 一键安装，桌面应用提供 dmg/exe/deb/rpm
- **配置文档**：详细文档在 opencode.ai/docs（具体内容未在 README 展示）

## 适用场景
- 日常编码任务：通过自然语言指令修改代码、重构、生成代码
- 陌生代码库探索：使用 plan 代理安全阅读和分析代码
- 敏捷团队协作：统一终端编码代理，无需切换工具
- 偏好开源与本地控制：替代闭源商业 AI 编码助手

## 同类对比
与 Claude Code、Cursor、GitHub Copilot Chat、Continue 等相比，OpenCode 的核心差异在于：
- **开源与免费**：无订阅费用，代码可审计和改造
- **终端优先**：为命令行环境设计，同时提供桌面应用
- **双代理设计**：显式区分只读/读写模式，减少误操作风险
- **多模型支持未知**：README 未列出支持的 LLM 提供商，可能需自行配置密钥
